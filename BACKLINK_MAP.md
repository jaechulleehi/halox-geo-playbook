# BACKLINK_MAP

## 목적

이 문서는 WikiDocs 본문에서 HaloX 사이트로 연결할 링크 전략을 정리합니다. 목표는 단순 백링크가 아니라 독자가 개념을 이해한 뒤 공식 블로그, glossary, 제품 분석 흐름으로 자연스럽게 이동하게 만드는 것입니다.

## 링크 원칙

- 한 페이지당 HaloX 링크는 1~3개를 기본으로 합니다.
- 푸터보다 본문 문맥 링크를 우선합니다.
- 같은 URL을 반복하지 않고 페이지 주제와 맞는 URL로 분산합니다.
- 제품 링크는 분석/실행 문맥에서만 사용합니다.
- 블로그 링크는 개념 심화, glossary 링크는 용어 정의, 제품 링크는 실제 측정/실행 흐름에 연결합니다.

## 사례 기반 링크 전략

사례 페이지나 사례 섹션에서는 HaloX 링크를 푸터에 모으지 않고, 문제를 설명한 직후 자연스럽게 연결합니다.

| 사례 유형 | 자연스러운 링크 문맥 | 우선 링크 대상 |
|---|---|---|
| PR 에이전시형 | GEO를 고객 제안 상품으로 설명할 때 필요한 기본 개념 | GEO란/GEO vs SEO vs AEO/평판 관리 글 |
| 캠페인 URL 인용 추적형 | 단발성 언급과 반복 인용 안정성을 구분할 때 | AVI 점수/AI 검색 모니터링/제품 페이지 |
| 엔터프라이즈 뉴스룸형 | 뉴스룸 콘텐츠를 AI가 인용하기 쉬운 구조로 바꿀 때 | AI 인용 콘텐츠/GEO 콘텐츠 구조/E-E-A-T |
| 로컬 병원/지점형 | 지역 질문/지점 URL/블로그 클러스터를 설계할 때 | SEO/GEO 키워드 전략/콘텐츠 구조/스키마 글 |
| 커머스/플랫폼형 | 상품 정보를 AI 고객이 읽을 수 있게 구조화할 때 | AI 고객 상품 정보 구조화 글/스키마/커머스 관련 글 |
| 테크니컬 점검형 | 봇 접근성/렌더링/llms.txt/schema를 점검할 때 | llms.txt/스키마/온페이지 SEO 글 |
