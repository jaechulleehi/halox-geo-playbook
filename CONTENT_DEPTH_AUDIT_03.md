## 03장 콘텐츠 깊이 보강 감사 메모

### 진단

03장은 01장의 질문셋과 02장의 측정 결과를 콘텐츠 실행으로 바꾸는 다리입니다. 기존 본문은 방향은 맞았지만 `fan-out 질문맵`이 왜 필요한지, 어떤 단위로 질문을 펼쳐야 하는지, 질문 유형과 콘텐츠 갭이 어떻게 실행 과제로 이어지는지 설명이 얕았습니다.

### 보강 기준

| 페이지 | 보강 역할 |
|---|---|
| 03 parent | 01장 질문셋과 02장 기준선을 04장 리라이트로 연결하는 실행 모델 제시 |
| 03-01 | seed 질문을 depth 1/2/3 질문 노드로 펼치는 방법, 노드별 콘텐츠 커버리지, 우선순위 기준 |
| 03-02 | 정의/비교/추천/검증/실행뿐 아니라 브랜드형/카테고리형/대안형/리스크형/로컬형/커머스형 질문 포트폴리오로 확장 |
| 03-03 | 질문별로 신규 작성/리라이트/source 보강/기술 점검/제품 설명 보강을 분리하는 판단표 |

### 반영한 소스

- `SOURCE_MAP.md`: fan-out 질문맵 기반 콘텐츠 기획, 질문셋 포션/커버리지, AI 설명 품질 진단 연결
- `BACKLINK_MAP.md`: 03장 HaloX 링크 전략
- Obsidian product note: `기능 아이디어 - fan out 질문맵 기반 콘텐츠 기획 뷰`
- Obsidian feature spec: `기능 설계 - fan out 질문맵 기반 콘텐츠 기획 뷰`
- Obsidian product note: `기능 아이디어 - 인용 사유 fan out 사례집 아카이브 뷰`
- Google Search Central: SEO Starter Guide, Creating helpful content
- HaloX blog: SEO/GEO 키워드 전략 프레임워크, GEO 콘텐츠 구조화 가이드

### 핵심 모델

```text
기준선에서 약한 질문군 선택
→ seed 질문 설정
→ depth 1/2/3 fan-out 질문 노드 확장
→ 질문 유형/구매 단계/답변 재료 표시
→ 현재 콘텐츠 커버리지 확인
→ 신규 작성/리라이트/source 보강/기술 점검/제품 설명 보강으로 분류
→ 04장 Answer-first 리라이트와 05장 source 전략으로 넘김
```

### 남은 후속 후보

- 04장에서는 03장에서 나온 콘텐츠 갭을 Answer-first 본문, FAQ, 표, schema 구조로 바꾸는 부분을 더 깊게 연결해야 합니다.
- 05장에서는 source/citation 쪽으로 넘어가는 갭을 외부 출처/엔티티 전략으로 분리해야 합니다.
- 06장에서는 기술 점검으로 분류된 갭을 schema, canonical, robots, sitemap, 렌더링 점검으로 이어가야 합니다.
