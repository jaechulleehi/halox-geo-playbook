# EBOOK_PLAN

## 목표

WikiDocs 책을 기반으로 판매 가능한 GEO 이북을 만듭니다. WikiDocs는 공개 지식 베이스이고, 이북은 읽기 흐름과 실습 워크북을 강화한 유료 패키지입니다.

## 이북 구성 방향

1. 개념부: GEO/SEO/AEO/AIO, AI 검색 변화, 질문 시장
2. 분석부: GEO 지표, HaloX 분석 프레임, 리포트 해석
3. 실행부: 질문맵, 콘텐츠 구조, 출처 설계, 테크니컬 점검
4. 산업부: 커머스, 플랫폼, 로컬, 글로벌
5. 운영부: 대행사 검증, 30일 실행 계획, 워크시트

## WikiDocs와 다른 점

- 공개 링크 중심 문장을 줄이고 책 문장으로 정리합니다.
- 강의 운영 메모는 제거하거나 워크북으로 재배치합니다.
- 표/체크리스트/템플릿을 PDF 친화적으로 재편집합니다.
- HaloX 제품 언급은 사례와 분석 프레임 중심으로 유지합니다.


## PDF 워크북 패키지

이북 판매본은 본문 교재와 워크북을 분리해서 구성합니다.

| 구성 | 역할 | 파일 |
|---|---|---|
| 본문 교재 | GEO 개념/측정/실행 전략 설명 | WikiDocs 본문 기반 |
| 워크북 | 수강생이 직접 작성하는 표/템플릿 | `PDF_WORKBOOK.md` |
| 강사용 노트 | 강의 진행 멘트/피드백 기준 | `INSTRUCTOR_NOTES.md` |
| 슬라이드 목차 | 4주차별 발표 흐름 | `LECTURE_SLIDES.md` |
| 판매 페이지 문구 | 외부 랜딩/강의 소개 | `COURSE_LANDING_COPY.md` |

## 이북 편집 순서

1. WikiDocs 본문에서 강의 진행 멘트를 줄입니다.
2. 표/체크리스트/FAQ를 PDF 친화적으로 재배치합니다.
3. 각 장 끝에 워크북 연결 안내를 붙입니다.
4. HaloX 링크는 과하게 넣지 않고, 개념/측정/실행 프레임이 필요한 곳에만 둡니다.
5. 최종 부록에 4주 워크북과 30일 액션 플랜 템플릿을 배치합니다.
