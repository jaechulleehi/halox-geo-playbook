# EXPANSION_BACKLOG

## 목적

이 문서는 `halox-geo-playbook`을 단순 GEO 개론서가 아니라 HaloX의 실제 스터디, 고객 미팅, 제품 기능, 블로그 자산이 살아 있는 강의/이북/백링크용 지식 베이스로 보강하기 위한 실행 백로그입니다.

핵심 원칙은 하나입니다. 사례는 많이 넣되, 실제 고객명/계정/URL/비공개 수치가 아니라 `산업별 익명 사례`와 `공개 가능한 기능 흐름`으로 바꿔 넣습니다.

## 사례 반영 원칙

| 원칙 | 기준 |
|---|---|
| 실명보다 유형 | 고객명 대신 `PR 에이전시형`, `엔터프라이즈 뉴스룸형`, `로컬 병원형`, `금융/거래소형`, `커머스/플랫폼형`으로 설명합니다. |
| 기능보다 문제 먼저 | `HaloX 기능`을 먼저 말하지 않고, 고객이 겪은 문제를 먼저 말한 뒤 어떤 분석 기능으로 풀 수 있는지 연결합니다. |
| 수치는 학습용으로 | 실제 내부 수치가 아니라 예시 수치 또는 범위로 바꿉니다. |
| 제안서 언어로 전환 | 미팅 메모를 그대로 옮기지 않고, 강의/이북에서 재사용 가능한 판단 기준으로 바꿉니다. |
| 링크는 자연스럽게 | HaloX 링크는 사례 설명 뒤에 개념 심화/측정/실행 도구 흐름으로 넣습니다. |
