# EXPANSION_REVIEW

## 목적

현재 WikiDocs GEO 책이 강의자료/백링크 지식 베이스/이북으로 충분한지 점검하고, Obsidian의 스터디/고객사 미팅/블로그/제품 노트를 어떤 방식으로 반영할지 정리한다.

## 결론

현재 00~10장 구조는 유지하는 것이 좋다. 다만 지금 구조는 "기본 교재 골격"에 가깝고, HaloX가 가진 실제 강점인 고객사 VOC, 산업별 케이스, 제품 분석 프레임, 한국형 GEO 데이터, 커머스/로컬/PR/엔터프라이즈 운영 사례가 아직 충분히 들어가지 않았다.

따라서 장 수를 크게 늘리기보다 다음 방향이 적합하다.

1. 00~10장 대구조는 유지한다.
2. 각 장 안에 실전 케이스/고객사 익명 사례/제품 관점/체크리스트를 보강한다.
3. 부족한 축은 부록/심화 트랙으로 확장한다.
4. 강의자료는 4주 기본 과정 + 심화 세션 3개로 분리한다.
5. 이북은 공개 WikiDocs보다 사례/워크북/템플릿/작성 예시를 더 깊게 넣어 유료 가치를 만든다.

## 확인한 주요 소스 축

### 1. GEO 스터디 마스터 인덱스

주요 축은 이미 현재 책의 뼈대와 잘 맞는다.

- GEO 측정/KPI: mention/source/citation, 질문셋 포션, 커버리지 가설
- Fan-out/인용 구조: 내부 확장 질문, 문장 단위 confidence, 검증 루프
- 콘텐츠 구조: FAQ, 표, schema, markdown 친화 구조, 연도 업데이트
- 제품/UX: 온보딩, 라이트 모드, 경쟁사 탐색, 인용 매체 가시성
- 오프사이트: 위키, PR 기사, 언드미디어, 네이버 블로그, Reddit, YouTube
- 커머스/AIO: 플랫폼 상세페이지, Merchant Center, 상품 정보 구조화
- 글로벌/영문 GEO: 영문 카테고리 정의, locale/hreflang, 영문 디렉터리
- 시장/대행사: 허상과 실제 차별점, 사례집, 실험 구조
- 테크니컬: robots.txt, AI crawler, sitemap, CSR/SSR, canonical, 대량 발행 리스크
- AX/조직 변화: 엔터프라이즈 내부 운영, 보안, DX 수준, 변화관리

### 2. 고객사/파트너 미팅

공개 책에는 실명 노출을 피하고, 산업별 익명 케이스로 전환해야 한다.

- PR 에이전시형: PR 에이전시/캠페인 모니터링 파트너 미팅
  - 진단/전략/콘텐츠 실행 상품화
  - 네이버 블로그/링크드인/뉴스룸/언론/온드미디어 역할 분리
  - 고객 제안용 단순 진단/점수/전략 언어 필요
  - 캠페인 URL, 인플루언서 URL, 반복 인용 안정성 추적 니즈
- 엔터프라이즈 커뮤니케이션형: 엔터프라이즈 커뮤니케이션/뉴스룸 운영 미팅
  - 뉴스룸/유튜브/그룹 현안/회장님/전략 프롬프트 운영
  - 5월 보고용 초기 증거, 점수화, 자동 체크, 검증 루프
  - 글로벌 기업 뉴스룸 벤치마크 같은 엔티티 허브 벤치마크
- 로컬/병원형: 로컬 병원/지점형 서비스 미팅
  - 지점별 URL/키워드/지역 의도
  - 병원/화장품/해외 고객 유치 트랙 분리
  - 블로그 구축 + 콘텐츠 클러스터 + 컨설팅 조합
- 금융/거래소형: 금융/거래소형 브랜드 미팅
  - 브랜드 검색 결과와 비브랜드 전환 키워드 동시 관리
  - 제3도메인 시도의 authority 한계
  - 기술 SEO/GEO 진단, 프롬프트/키워드 전략, 내부/외부 운영 모델

### 3. HaloX 블로그/콘텐츠

현재 WikiDocs는 HaloX 블로그로 자연스럽게 연결되고 있으나, 링크 수와 문맥은 더 정교화할 수 있다.

- 이미 발행된 GEO 클러스터: GEO란, GEO vs SEO vs AEO, AVI, AI 인용 콘텐츠, GEO 콘텐츠 구조, llms.txt, SEO/GEO 키워드, 평판 관리
- 추가 연결 후보: OpenAI crawl signal, AI preferred content, SEO fundamentals, schema, topical authority, E-E-A-T, unified SEO/GEO strategy
- 공백 키워드: GEO 최적화 도구, AI 검색 모니터링 도구, Perplexity SEO, ChatGPT Search 최적화, Google AI Overviews 노출 전략, 한국 브랜드 GEO 전략

### 4. 제품/기능 노트

제품 노트는 책의 차별화 포인트가 될 수 있지만, 공개 가능한 수준으로 일반화해야 한다.

- mention/source/citation 성과 프레임
- fan-out 질문맵 기반 콘텐츠 기획
- 질문 기반 AI 설명 품질 진단
- 병원 지점형 GEO 운영 설계
- 플랫폼 커머스 상세페이지 GEO 진단
- 글로벌 영문 GEO readiness
- GEO 대행사 검증 체크리스트
- AI 브리핑 인용 안정성 리포트
- 네이버/구글 AI 교차율 모니터링 카드

## 현재 10장 구조 진단

### 유지할 점

- 00~02장이 입문/측정 기반을 잡고 있다.
- 03~06장이 질문/콘텐츠/출처/테크니컬 실행 축을 담당한다.
- 07~08장이 산업별/글로벌 확장성을 담고 있다.
- 09~10장이 시장 검증/강의 워크숍/실행 리포트로 닫힌다.
- 강의자료와 WikiDocs를 동시에 쓰기 좋은 구조다.

### 부족한 점

- 고객사 미팅에서 나온 실제 VOC가 아직 본문에 충분히 녹아 있지 않다.
- 07장 산업별 전략이 너무 압축되어 있다. 병원/로컬, PR/뉴스룸, 금융/거래소, 커머스/플랫폼은 분리 가치가 있다.
- 02장 측정 지표는 더 강해질 여지가 크다. 교차율, 인용 안정성, 질문셋 포션, 브랜드/채널/경쟁사 시계열을 더 넣어야 한다.
- 05장 오프사이트 전략은 PR/언론/위키/UGC/네이버 블로그/Reddit/YouTube까지 채널별 역할표가 필요하다.
- 06장 테크니컬은 robots.txt, AI crawler, llms.txt, CSR/SSR, schema, sitemap, canonical, 대량 발행 리스크를 더 실무형으로 보강해야 한다.
- 09장 시장/대행사 장은 실제 제안서 언어, 진단 상품 구성, 가격/범위/운영 모델까지 더 구체화할 수 있다.
- AX/조직 변화 관점은 현재 별도 장으로 보이지 않는다. 엔터프라이즈 도입 현실을 다루는 심화 섹션이 필요하다.

## 확장 방향

### 옵션 A. 00~10장 유지 + 세부 페이지 보강

가장 추천한다. 현재 WikiDocs 구조 안정성을 유지하면서 내용 밀도를 높일 수 있다.

- 각 세부 페이지에 `현장 인사이트` 섹션 추가
- 각 장에 `익명 케이스` 1~2개 추가
- 각 장에 `HaloX로 보면` 섹션 추가
- 각 장에 `강의 토론 질문` 추가
- 각 장에 `이북 보강 메모` 추가

### 옵션 B. 심화 부록 추가

10장 구조는 유지하되, 부록을 만들어 강의/이북 가치를 높인다.

추천 부록:

- 부록 A. GEO 용어집과 지표 사전
- 부록 B. 산업별 GEO 케이스북
- 부록 C. GEO 리포트/제안서 템플릿 모음
- 부록 D. 테크니컬 GEO 체크리스트
- 부록 E. HaloX 블로그/자료 링크 맵

### 옵션 C. 11~13장 추가

장 수 확장은 당장 1순위는 아니다. 다만 이북을 두껍게 만들거나 심화 강의로 분리한다면 아래 장을 추가할 수 있다.

- 11. 한국형 GEO와 네이버 AI 브리핑 전략
- 12. 엔터프라이즈 GEO 운영 체계와 조직 변화
- 13. GEO 제안서/리포트/상품화 실전

## 1차 작업 제안

바로 다음 작업은 `SOURCE_MAP.md`와 신규 `EXPANSION_BACKLOG.md`를 먼저 만드는 것이 좋다.

이유:

- 지금은 소스가 많아 곧바로 본문을 쓰면 중복/누락이 생긴다.
- 고객사 미팅 내용은 익명화 기준이 필요하다.
- 장별로 무엇을 보강할지 먼저 고정해야 45개 페이지를 안정적으로 확장할 수 있다.
- 이 작업이 끝나면 02장/07장/05장 순서로 본문 보강을 빠르게 진행할 수 있다.
