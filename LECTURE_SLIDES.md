# LECTURE_SLIDES

## 목적

이 문서는 `생성형 엔진 최적화(GEO) 실전 교과서`를 4주 강의 슬라이드로 바꿀 때 쓰는 제작용 목차입니다. WikiDocs 본문은 교재이고, 이 문서는 강사가 수업 흐름을 잡기 위한 슬라이드 뼈대입니다.

강의의 약속은 단순합니다. 수강생은 4주 뒤 자기 브랜드의 `GEO 실행 리포트`와 `30일 액션 플랜`을 가져가야 합니다.

## 전체 강의 구성

| 주차 | 수업 제목 | 핵심 산출물 | 연결 WikiDocs |
|---|---|---|---|
| 1주차 | SEO 키워드를 AI 질문 시장으로 바꾸기 | 브랜드 baseline 리포트 | 00장/01장/02장/10-01 |
| 2주차 | Fan-out 질문맵과 콘텐츠 갭 찾기 | 질문맵/갭 리스트 | 03장/04장/10-02 |
| 3주차 | AI가 인용하는 콘텐츠와 출처 설계 | 리라이트 초안/source map | 04장/05장/10-03 |
| 4주차 | 실행 리포트와 30일 액션 플랜 | GEO 실행 리포트 | 06장/09장/10장 |

## 공통 슬라이드 구조

각 주차는 18~24장 안에서 끝내는 것을 기본으로 합니다. 설명 슬라이드는 줄이고, 실습 안내/작성 예시/피드백 슬라이드를 충분히 둡니다.

| 구간 | 슬라이드 수 | 역할 |
|---|---:|---|
| 오프닝 | 2~3 | 오늘의 문제와 산출물 제시 |
| 핵심 개념 | 4~6 | 실습에 필요한 개념만 설명 |
| 사례/예시 | 3~4 | AcmeGEO 또는 익명 산업 사례로 이해 |
| 실습 안내 | 4~6 | 워크시트 작성 순서 안내 |
| 피드백/제출 | 3~5 | 교차 검토 기준과 과제 정리 |

## 슬라이드 제작 기준

- 한 슬라이드에는 메시지 하나만 둡니다.
- 설명 슬라이드는 `문제 → 기준 → 예시` 순서로 씁니다.
- 실습 슬라이드는 `작성 항목 → 작성 기준 → 제출 형태`가 보여야 합니다.
- HaloX 화면이나 리포트는 기능 소개가 아니라 판단 기준을 보여주는 용도로 넣습니다.
- 마지막 슬라이드는 항상 다음 주차 입력값 또는 30일 실행 기준으로 끝냅니다.
