# WORKSHOP_PACKAGES

## 목적

이 문서는 WikiDocs 세부 페이지를 강의자료/실무 패키지/이북 워크북으로 다시 쓸 때 필요한 공통 구조를 정리합니다. 각 세부 페이지는 사례, 참고 링크, 실습 템플릿, 제출 산출물을 포함해야 합니다.

## 공통 패키지 구조

| 구성 | 역할 | 본문 반영 위치 |
|---|---|---|
| 사례 | 독자가 자기 상황에 대입할 수 있는 장면 | `## 사례로 이해하기` |
| 참고 링크 | HaloX 블로그/공식 사이트/관련 WikiDocs 페이지 연결 | `## 참고 링크 패키지` |
| 실습 워크시트 | 강의 중 바로 작성할 표 | `## 실습 워크시트` |
| 제출 템플릿 | 과제/이북 워크북으로 재사용할 양식 | `## 제출 템플릿` |
| 완료 기준 | 산출물이 충분한지 확인하는 기준 | `## 완료 기준` |

## 장별 패키지 방향

| 장 | 패키지 초점 | 핵심 산출물 |
|---|---|---|
| 03장 | 질문 확장과 콘텐츠 갭 | Fan-out 질문맵/질문 유형 포트폴리오/콘텐츠 갭 리스트 |
| 04장 | AI가 읽기 좋은 콘텐츠 리라이트 | Answer-first 초안/FAQ 표/schema 구조/리라이트 전후안 |
| 05장 | 출처/인용/엔터티 설계 | 출처 후보표/브랜드 합의 신호 맵/오프사이트 실행표 |
| 06장 | 테크니컬 GEO 점검 | 크롤러 접근성 점검표/렌더링 진단표/사이트 이전 리스크 표 |
| 07장 | 산업별 적용 | B2B SaaS/커머스/로컬 서비스별 질문셋과 실행 우선순위 |
| 08장 | 글로벌/영문 확장 | 영문 카테고리 자산표/locale 점검표/글로벌 source map |
| 09장 | 시장/대행사/사업화 검증 | 대행사 평가표/도구 리포트 검증표/사업화 패키지 설계 |
| 10장 | 4주 챌린지 운영 | 주차별 워크시트/실행 리포트/30일 액션 플랜 |

## 강의 운영 기준

- 강의 설명은 20% 이하, 작성/진단/피드백 시간을 80%에 가깝게 둡니다.
- 모든 실습은 한 장짜리 산출물로 끝나야 합니다.
- 산출물은 다음 주차 실습의 입력값이 되어야 합니다.
- HaloX 링크는 개념을 보강하거나 실제 측정 흐름으로 넘어갈 때만 넣습니다.
- 이북 전환 시에는 강의 진행 멘트를 줄이고 워크시트/예시/완료 기준을 남깁니다.

## 작성 예시 운영 기준

작성 예시는 빈 워크시트와 실제 과제 사이를 이어주는 샘플입니다. 모든 예시는 가상의 B2B SaaS 브랜드 `AcmeGEO`를 기준으로 두되, 독자가 자기 브랜드로 쉽게 바꿀 수 있게 항목명을 유지합니다.

| 원칙 | 기준 |
|---|---|
| 가상 브랜드 사용 | 실제 고객명이나 비공개 데이터를 넣지 않습니다. |
| 한 페이지 한 예시 | 페이지 주제와 직접 연결되는 표 하나만 둡니다. |
| 숫자는 학습용 | mention 수, 질문 수, 일정은 실습 이해를 위한 예시로 씁니다. |
| 다음 액션 포함 | 예시 표의 마지막 항목은 반드시 실행 또는 재측정으로 끝냅니다. |
| 이북 전환 가능성 | 강의 멘트 없이도 독자가 혼자 따라 할 수 있어야 합니다. |

## 1주차 입문/Baseline 패키지

00~02장은 4주 챌린지의 출발점입니다. 이 구간은 지식을 많이 설명하는 것보다, 수강생이 자기 브랜드 기준으로 GEO를 정의하고 질문셋과 baseline 표를 만들게 하는 데 집중합니다.

| 구간 | 핵심 산출물 | 강의 활용 |
|---|---|---|
| 00장 | GEO 개념 정의/용어 비교/활용 계획 | 강의 시작 전 사전 과제 또는 오리엔테이션 |
| 01장 | 키워드 10개와 AI 질문 30개 | 1주차 메인 실습 |
| 02장 | mention/source/citation baseline 진단표 | 1주차 제출물과 2주차 fan-out 입력값 |

운영 기준은 간단합니다. 00장은 이해 확인, 01장은 질문 생성, 02장은 측정 기록입니다. 이 세 단계가 연결되어야 03장 이후의 fan-out/콘텐츠 갭/source map 실습이 흔들리지 않습니다.


## 사례 중심 보강 기준

앞으로 각 장을 보강할 때는 `AcmeGEO` 하나의 가상 예시만 반복하지 않고, 산업별 익명 사례를 함께 배치합니다. 단, 실제 고객명/계정/비공개 URL/계약 조건은 넣지 않습니다.

| 사례 유형 | 강의에서 보여줄 장면 | 워크시트로 받을 산출물 |
|---|---|---|
| PR 에이전시형 | GEO를 고객 제안 상품으로 설명해야 하는 상황 | 진단/전략/실행 패키지 구성표 |
| 엔터프라이즈 뉴스룸형 | 뉴스룸/유튜브/온드미디어를 GEO 운영 체계로 묶는 상황 | 전략 프롬프트 세트와 뉴스룸 개선 우선순위 |
| 로컬 병원/지점형 | 지점별 지역 질문과 내원 전환을 연결하는 상황 | 지점/키워드/URL/콘텐츠 클러스터 표 |
| 금융/거래소형 | 브랜드 검색과 비브랜드 전환 키워드를 동시에 관리하는 상황 | 브랜드/비브랜드 질문셋과 기술 점검 우선순위 |
| 커머스/플랫폼형 | AI 고객이 상품 정보를 비교/추천할 수 있게 만드는 상황 | 상품 정보 구조화 체크리스트 |
| 글로벌/영문 확장형 | 한국 브랜드가 영문 source와 카테고리 자산을 준비하는 상황 | 영문 카테고리 정의/source map |

## 기능 설명 운영 기준

HaloX 기능은 `기능 소개`가 아니라 `사례를 해결하는 분석 도구`로 등장해야 합니다.

좋은 흐름은 다음과 같습니다.

1. 현장 사례를 먼저 보여준다.
2. 사례에서 막히는 판단 지점을 짚는다.
3. 필요한 지표나 화면을 설명한다.
4. HaloX 기능/프레임으로 어떻게 읽는지 보여준다.
5. 독자가 자기 브랜드 워크시트로 옮긴다.

예를 들어 `AI 브리핑 인용 안정성 리포트`는 기능명부터 말하지 않습니다. 먼저 “캠페인 URL이 한 번 AI 답변에 들어간 것과 매주 반복 인용되는 것은 다르다”는 문제를 설명한 뒤, 반복 측정/시계열/출처 포함 여부를 보는 도구로 연결합니다.


## 10장 강의자료 전환 기준

10장은 책의 결론이 아니라 실제 강의 운영안입니다. 각 페이지는 `강의 목표 → 90분 진행안 → 실습 워크시트 → 제출 템플릿 → 피드백 기준 → 완료 기준` 순서로 읽히게 구성합니다.

| 주차 | 강의자료로 뽑을 핵심 슬라이드 | 회수할 제출물 |
|---|---|---|
| 1주차 | GEO baseline과 질문셋 기록법 | 브랜드 baseline 표 |
| 2주차 | Fan-out 질문맵과 콘텐츠 갭 분류 | 질문맵/갭 리스트 |
| 3주차 | Answer-first 리라이트와 source map | 리라이트 초안/source map |
| 4주차 | 실행 리포트와 30일 액션 플랜 | GEO 실행 리포트 |

이북으로 전환할 때는 강사용 멘트를 줄이고, 워크시트/작성 예시/완료 기준/FAQ를 남깁니다. 강의 판매 페이지로 전환할 때는 산출물과 변화 약속을 앞세웁니다.
