## SERP 분석과 콘텐츠 갭 찾기

SERP 분석은 상위 노출 페이지를 베끼기 위한 작업이 아닙니다. 검색엔진이 특정 키워드에서 어떤 답변 형식, 어떤 정보 깊이, 어떤 신뢰 신호를 우선 보여주는지 읽는 작업입니다. GEO에서는 이 SERP 분석을 한 단계 확장해 `AI가 답변을 만들 때 빠뜨리면 안 되는 정보 단위`를 찾습니다.

예를 들어 `GEO 도구`라는 키워드를 보면 기존 SEO에서는 상위 문서의 제목, 목차, 비교표, 가격 정보, FAQ를 봅니다. GEO에서는 여기에 더해 AI가 추천 답변을 만들 때 어떤 후보군, 비교 기준, 리포트 예시, 신뢰 근거를 조합하는지까지 봐야 합니다.

## SERP를 볼 때 먼저 나눌 것

| 관찰 항목 | SEO에서 보는 의미 | GEO로 확장할 질문 |
|---|---|---|
| 상위 페이지 유형 | 블로그, 제품 페이지, 비교 글, 가이드, 리뷰 | AI 답변은 어떤 유형의 근거를 섞어 쓰는가? |
| 제목과 첫 문단 | 검색 의도와 핵심 약속 | AI가 한 문장으로 요약할 브랜드/개념 정의가 있는가? |
| 목차와 H2 | 사용자가 기대하는 하위 질문 | fan-out 질문맵으로 바꿀 수 있는가? |
| 표/FAQ/리스트 | 빠르게 비교 가능한 정보 단위 | AI 답변의 판단 기준으로 재사용 가능한가? |
| 외부 인용/출처 | 신뢰 보강 방식 | source/citation 후보로 이어질 수 있는가? |

이 표를 채우면 SERP 분석은 “상위 글이 길다”가 아니라 “AI 답변에 필요한 재료가 무엇인지”를 보여줍니다.

## 콘텐츠 갭을 찾는 5단계

1. 핵심 키워드 5~10개를 고릅니다.
2. 각 키워드의 상위 결과 5~10개를 봅니다.
3. 반복되는 제목/H2/FAQ/표 항목을 모읍니다.
4. 우리 페이지에 없는 답변 요소를 표시합니다.
5. 빠진 요소를 AI 질문셋과 리라이트 후보로 바꿉니다.

이 과정에서 중요한 것은 검색량이 아니라 `반복되는 판단 기준`입니다. AI 답변은 여러 출처에서 반복되는 기준을 조합해 답변을 만들 가능성이 높기 때문입니다.

## SERP 갭을 GEO 질문으로 바꾸기

| SERP에서 발견한 갭 | GEO 질문으로 바꾼 예 | 다음 액션 |
|---|---|---|
| 가격/운영 방식 비교가 없다 | GEO 도구를 고를 때 월간 리포트와 재측정 기능은 어떻게 봐야 하나? | 비교 기준 글 작성 |
| 후보군 분류가 약하다 | AI 검색 모니터링 도구는 SEO 도구와 무엇이 다른가? | 도구 유형 분류표 작성 |
| 실제 사례가 없다 | B2B SaaS가 GEO 기준선을 잡는 예시는? | 케이스/워크시트 추가 |
| 기술 조건 설명이 없다 | schema나 sitemap이 AI 인용에 어떤 영향을 주나? | 06장 기술 점검으로 연결 |

SERP 갭은 곧 AI 답변 갭입니다. 상위 문서가 반복해서 다루는 항목이 우리 페이지에 없다면 SEO에서도 약하고, AI 답변 재료로도 약합니다.

## AcmeGEO SERP 분석 예시

아래는 `AI 검색 모니터링 도구` 키워드를 분석한다고 가정한 예시입니다.

| 상위 SERP에서 반복되는 요소 | 우리 페이지 상태 | 콘텐츠 갭 | GEO 질문으로 바꾼 문장 |
|---|---|---|---|
| AI 검색 노출 확인 방법 | 설명은 있으나 절차가 짧음 | 단계별 측정 순서 부족 | 우리 브랜드가 ChatGPT/Perplexity 답변에 나오는지 어떻게 확인하나? |
| 도구 비교표 | 기능 목록만 있음 | 질문셋/재측정/리포트 기준 부족 | GEO 도구를 고를 때 어떤 리포트 항목을 봐야 하나? |
| 가격/도입 기준 | 없음 | 구매 검토 의도 대응 부족 | GEO 솔루션 도입 비용과 실행 범위는 어떻게 판단하나? |
| source/citation 설명 | 용어 설명은 있음 | 실제 예시 부족 | AI 답변에서 source와 citation은 어떻게 다르게 기록하나? |
| 기술 조건 | 거의 없음 | schema/canonical/sitemap 연결 부족 | AI가 인용할 수 있는 URL 상태는 어떻게 점검하나? |

이 표에서 바로 리라이트 우선순위가 나옵니다. 첫째는 도구 비교 기준, 둘째는 source/citation 예시, 셋째는 기술 점검 연결입니다. SERP 분석은 상위 문서의 제목을 따라 쓰는 작업이 아니라, 독자가 기대하는 답변 단위를 빠뜨리지 않게 만드는 작업입니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 분석 키워드 | 핵심 SEO 키워드 5~10개 |
| 상위 결과 유형 | 가이드/제품/비교/리뷰/뉴스/커뮤니티 |
| 반복 목차 | 상위 결과에서 3번 이상 반복되는 H2/FAQ |
| 우리 페이지에 없는 요소 | 정의, 비교표, 가격, 사례, 기술 기준, 출처 등 |
| GEO 질문 후보 | 갭을 질문으로 바꾼 문장 |
| 리라이트 후보 | 고칠 페이지 또는 새로 만들 페이지 |

```text
키워드 / 상위 결과 유형 / 반복 목차 / 누락 요소 / GEO 질문 후보 / 리라이트 후보
```

## 완료 기준

- 상위 SERP를 순위가 아니라 답변 구조로 읽을 수 있습니다.
- 반복되는 목차/FAQ/표 항목을 fan-out 질문으로 바꿨습니다.
- 우리 콘텐츠에 빠진 정보 단위가 리라이트 후보로 정리됐습니다.
