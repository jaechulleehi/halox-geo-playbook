## 온페이지 SEO와 Answer-first 구조

온페이지 SEO는 title, meta description, heading, 본문, 이미지 설명, 내부 링크처럼 한 페이지 안에서 검색엔진이 이해할 수 있는 신호를 정리하는 작업입니다. GEO에서는 이 작업이 `AI가 답변 재료를 빠르게 찾고 인용할 수 있는 구조`로 확장됩니다.

좋은 GEO 콘텐츠는 SEO 기본기를 버리지 않습니다. 오히려 title과 H2는 더 명확해야 하고, 첫 문단은 더 빨리 답해야 하며, 표와 FAQ는 더 재사용 가능한 정보 단위가 되어야 합니다.

## 온페이지 SEO 요소를 GEO로 다시 읽기

| 요소 | SEO에서의 역할 | GEO에서의 확장 |
|---|---|---|
| title | 검색결과에서 주제와 클릭 이유를 전달 | AI가 페이지 주제를 분류하는 첫 단서 |
| meta description | 검색자에게 요약 제공 | 답변에 필요한 페이지 약속을 압축 |
| H2/H3 | 본문 구조와 하위 의도 정리 | fan-out 질문별 답변 블록 제공 |
| 첫 문단 | 사용자의 즉시 이해를 도움 | AI가 가져갈 핵심 답변 문장 제공 |
| 표 | 비교/조건/절차를 빠르게 전달 | AI 답변의 비교 기준으로 재사용 |
| FAQ | 세부 질문을 직접 처리 | long-tail 질문과 AEO/GEO 접점 제공 |
| 내부 링크 | 관련 문맥과 권위 연결 | 주제 클러스터와 엔티티 관계를 강화 |

이 표를 기준으로 보면 온페이지 SEO는 단순한 체크리스트가 아니라 GEO 콘텐츠 구조의 바닥입니다.

## 첫 문단은 왜 더 중요해졌나

기존 SEO에서도 첫 문단은 중요했지만, GEO에서는 더 직접적이어야 합니다. AI가 긴 문서를 모두 같은 비중으로 읽는다고 가정하면 안 됩니다. 페이지 상단에서 무엇을 답하는지, 누구에게 필요한지, 어떤 기준을 제공하는지 바로 보여줘야 합니다.

나쁜 첫 문단은 보통 이렇게 시작합니다.

```text
이번 글에서는 GEO 도구에 대해 알아보겠습니다.
```

좋은 첫 문단은 이렇게 답합니다.

```text
GEO 도구는 AI 검색에서 브랜드가 어떤 질문에 언급되고, 어떤 출처를 근거로 설명되며, 경쟁사와 어떤 기준으로 비교되는지 측정하는 도구입니다. 기존 SEO 도구가 검색 순위와 클릭을 중심으로 본다면, GEO 도구는 mention/source/citation과 재측정 가능한 질문셋을 함께 봅니다.
```

## 온페이지 SEO 점검 순서

1. title에 핵심 검색 의도와 페이지 약속이 들어 있는지 봅니다.
2. 첫 문단이 질문에 바로 답하는지 확인합니다.
3. H2가 사용자의 하위 질문으로 읽히는지 확인합니다.
4. 표/리스트/FAQ가 판단 기준을 분리해 주는지 봅니다.
5. 내부 링크가 다음에 볼 개념과 실제 실행 페이지로 이어지는지 확인합니다.
6. schema가 본문에 실제로 있는 정보를 구조화하는지 봅니다.

## Before/After 예시

| 항목 | 약한 구조 | 개선 구조 |
|---|---|---|
| 제목 | GEO 도구 정리 | GEO 도구 비교: AI 검색 모니터링에서 봐야 할 7가지 기준 |
| 첫 문단 | GEO 도구가 중요해지고 있습니다 | GEO 도구는 mention/source/citation을 질문셋 단위로 반복 측정하는 도구입니다 |
| H2 | 기능 | 질문셋 관리, source/citation 추적, 리포트 재측정 |
| 표 | 도구 목록 | 도구 유형/측정 지표/운영 적합성 비교 |
| 내부 링크 | 없음 | SEO 도구 비교, GEO 리포트, 기술 점검 페이지로 연결 |

## title/meta/heading 작성 기준

온페이지 SEO는 각 요소가 따로 잘 쓰였는지보다 한 페이지 안에서 같은 질문에 답하는지가 중요합니다.

| 요소 | 약한 작성 | 좋은 작성 | 점검 질문 |
|---|---|---|---|
| title | GEO 도구 정리 | GEO 도구 비교: AI 검색 모니터링에서 봐야 할 7가지 기준 | 검색어와 답변 약속이 함께 있는가? |
| meta description | GEO 도구를 소개합니다 | GEO 도구를 질문셋, mention/source/citation, 리포트 재측정 기준으로 비교합니다 | 클릭 전 요약이 구체적인가? |
| H2 | 기능 | 질문셋 관리 / source 추적 / citation 확인 / 리포트 재측정 | 하위 질문처럼 읽히는가? |
| 첫 문단 | 중요성이 커지고 있습니다 | GEO 도구는 AI 검색에서 브랜드 언급과 답변 근거를 질문셋 단위로 반복 측정하는 도구입니다 | 첫 2문장 안에 답이 있는가? |
| 내부 링크 | 관련 글 보기 | SEO 도구 비교, GEO 리포트, 테크니컬 점검으로 연결 | 다음 판단에 필요한 문맥으로 이어지는가? |

### 리라이트 전후 예시

```text
Before
GEO 도구는 AI 시대에 중요합니다. 이 글에서는 GEO 도구의 기능을 알아보겠습니다.

After
GEO 도구는 ChatGPT, Perplexity, Google AI Overviews에서 브랜드가 어떤 질문에 언급되고 어떤 출처로 인용되는지 반복 측정하는 도구입니다. 좋은 GEO 도구는 검색 순위만 보여주는 것이 아니라 질문셋, mention/source/citation, 경쟁사 비교, 재측정 리포트를 함께 제공해야 합니다.
```

After 문장은 검색자에게도 답이 되고, AI 답변에도 그대로 인용 가능한 설명 단위가 됩니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 대상 페이지 | 리라이트할 URL 또는 문서 |
| 핵심 질문 | 이 페이지가 답해야 할 한 문장 질문 |
| title 후보 | 검색어와 답변 약속을 함께 담은 제목 |
| 첫 문단 | 2~3문장 안에 바로 답하는 문단 |
| H2 목록 | 하위 질문처럼 읽히는 구조 |
| 표/FAQ 후보 | 비교/절차/체크리스트로 만들 정보 |
| 연결 링크 | 앞뒤 개념 페이지와 실행 페이지 |

```text
대상 페이지 / 핵심 질문 / title / 첫 문단 / H2 / 표와 FAQ / 연결 링크
```

## 완료 기준

- title, 첫 문단, H2가 같은 검색 의도를 향합니다.
- 첫 문단이 답을 미루지 않고 바로 말합니다.
- 표/FAQ/schema가 본문에 있는 정보를 구조화합니다.
