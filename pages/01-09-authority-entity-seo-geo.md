## 권위 신호와 엔티티를 GEO로 확장하기

SEO에서 권위 신호는 링크, 브랜드 언급, 전문성, 신뢰성, 콘텐츠 품질처럼 검색엔진이 페이지와 사이트를 신뢰할 때 참고하는 단서입니다. GEO에서는 이 신호가 `AI가 답변 근거로 삼을 수 있는 합의된 설명`과 연결됩니다.

단순히 백링크를 많이 만드는 것이 목표가 아닙니다. AI가 브랜드를 어떤 카테고리에 넣어 이해하는지, 어떤 외부 출처가 같은 설명을 반복하는지, 어떤 문장이 source/citation으로 쓰일 수 있는지 관리해야 합니다.

## SEO 권위 신호를 GEO 신호로 읽기

| SEO 신호 | 기존 의미 | GEO 확장 의미 |
|---|---|---|
| 백링크 | 외부 사이트가 우리 페이지를 참조 | AI가 참고할 수 있는 외부 근거 후보 |
| 브랜드 언급 | 브랜드 인지도와 평판 신호 | mention/consensus의 출발점 |
| E-E-A-T | 경험/전문성/권위/신뢰 | 답변에 포함될 자격과 리스크 판단 기준 |
| 프로필/디렉터리 | 조직/인물/제품 정보 확인 | 엔티티 필드의 일관성 확인 |
| 리뷰/커뮤니티 | 실제 사용자 경험 신호 | 장단점/문제 해결 문맥의 근거 |
| 뉴스룸/PR | 공식 발표와 외부 보도 | source/citation 후보와 최신성 신호 |

GEO에서는 이 신호를 `source`, `citation`, `entity`, `consensus`로 나눠 읽어야 합니다.

## 엔티티는 왜 중요한가

엔티티는 검색엔진과 AI가 사람, 회사, 제품, 장소, 개념을 하나의 식별 가능한 대상으로 이해하는 방식입니다. 브랜드명이 여러 방식으로 쓰이고, 설명 문장이 매번 달라지고, 공식 사이트와 외부 프로필의 정보가 다르면 AI 답변도 흔들릴 수 있습니다.

| 필드 | 확인할 것 |
|---|---|
| 공식 이름 | 한글/영문/약칭 표기가 일관적인가 |
| 카테고리 | 어떤 제품군/서비스군에 속하는가 |
| 핵심 설명 | 한 문장으로 무엇을 하는 브랜드인가 |
| 대표 URL | 공식 사이트/제품/문서 URL이 분명한가 |
| 근거 페이지 | 소개, 기능, 가격, 사례, 리포트 페이지가 연결되는가 |
| 외부 프로필 | 위키/디렉터리/PR/파트너 페이지 정보가 맞는가 |

이 필드가 정리되어야 05장의 source/citation/entity 전략이 실제 운영으로 이어집니다.

## 백링크를 답변 근거로 확장하기

| 약한 접근 | GEO형 접근 |
|---|---|
| 링크 개수만 늘린다 | 질문별로 필요한 출처 후보를 나눈다 |
| 같은 보도자료를 반복 배포한다 | 매체/블로그/디렉터리별로 다른 질문을 맡긴다 |
| 브랜드명만 반복한다 | 카테고리, 사용 사례, 비교 기준을 함께 설명한다 |
| 인위적 후기를 만든다 | 실제 사용 맥락과 리스크를 정직하게 구조화한다 |

권위 신호는 조작 대상이 아니라 설명의 일관성을 만드는 작업입니다.

## E-E-A-T와 백링크를 실행 항목으로 바꾸기

E-E-A-T는 추상적인 품질 구호가 아니라 페이지와 브랜드가 어떤 근거로 믿을 만한지 보여주는 기준입니다. GEO에서는 이 기준이 source/citation/consensus로 이어집니다.

| E-E-A-T 요소 | SEO에서 볼 것 | GEO 실행 항목 |
|---|---|---|
| Experience | 실제 사용 경험, 사례, 전후 비교 | 익명 사례, 워크시트 작성 예시, 리포트 샘플 추가 |
| Expertise | 전문 설명, 기준, 방법론 | 용어 정의, 판단 기준표, 체크리스트 보강 |
| Authoritativeness | 외부 언급, 링크, 업계 맥락 | PR/뉴스룸/디렉터리/파트너 페이지의 일관된 설명 확보 |
| Trustworthiness | 정확성, 최신성, 위험 표현 관리 | 업데이트 날짜, 출처, 제한 조건, 리스크 문장 정리 |

백링크도 같은 방식으로 봐야 합니다. 단순 링크 수보다 `어떤 질문에서 우리를 어떤 문장으로 설명하는가`가 중요합니다.

### 권위 신호 보강 예시

| 발견한 문제 | 원인 | 보강 액션 | 연결되는 GEO 지표 |
|---|---|---|---|
| AI가 브랜드를 후보군에 넣지 않음 | 카테고리 설명이 외부에 부족함 | 공식 소개/디렉터리/비교 글의 카테고리 문장 통일 | mention |
| 인용은 되지만 낡은 설명이 반복됨 | 오래된 PR/블로그가 source로 잡힘 | 최신 기능/리포트 기준을 설명하는 뉴스룸 글 보강 | source/citation |
| 경쟁사 대비 장점이 흐림 | 비교 기준을 외부에서 설명하지 않음 | 파트너/리뷰/사례 페이지에 사용 맥락 추가 | consensus |

이 작업은 링크 빌딩만의 문제가 아닙니다. 브랜드가 어떤 문제의 답으로 등장해야 하는지 외부 신호까지 같은 문장으로 맞추는 작업입니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 브랜드/제품명 | 공식 표기와 약칭 |
| 카테고리 문장 | AI가 이해해야 할 한 문장 설명 |
| 내부 근거 페이지 | 소개/기능/가격/사례/리포트 URL |
| 외부 근거 후보 | 언론, 리뷰, 디렉터리, 파트너, 커뮤니티 |
| 불일치 정보 | 이름, 설명, 가격, 기능, 지역, 대표 URL 오류 |
| 보강 액션 | 수정 요청, 공식 문서 보강, PR/블로그/디렉터리 업데이트 |

```text
브랜드명 / 카테고리 문장 / 내부 근거 페이지 / 외부 근거 후보 / 불일치 정보 / 보강 액션
```

## 완료 기준

- 브랜드/제품/인물/장소의 핵심 엔티티 필드가 정리됐습니다.
- 외부 출처가 맡을 질문과 설명 범위가 나뉘었습니다.
- 백링크를 숫자가 아니라 source/citation/consensus 관점으로 볼 수 있습니다.
