## SEO 기본기와 GEO 확장 전략

![SEO 키워드에서 AI 질문 시장으로 전환되는 흐름](../assets/images/chapter-heroes/halox-geo-keyword-to-question-hero-1600x900.png)

GEO 최적화는 SEO를 버리고 새 용어로 갈아타는 일이 아닙니다. 검색엔진이 페이지를 발견하고, 사용자의 검색 의도를 해석하고, 콘텐츠와 링크와 기술 신호를 평가하는 흐름을 이해해야 AI 검색 최적화도 제대로 설계할 수 있습니다.

이 장은 SEO만 따로 떼어 `검색 유입 구조`를 먼저 잡는 장입니다. 키워드 리서치, 검색 의도, SERP 분석, 온페이지 SEO, 테크니컬 SEO, 권위/엔티티 신호를 하나의 워크플로우로 정리한 뒤, 그 입력값을 GEO 질문셋과 AI 답변 근거로 확장합니다.

## SEO에서 GEO로 이어지는 핵심 모델

```text
SEO 키워드
→ 검색 의도
→ SERP/콘텐츠 갭
→ 온페이지 구조
→ 기술 접근성
→ 권위/엔티티 신호
→ AI 질문 유형
→ 필요한 콘텐츠/source/citation
→ 기준선 측정 질문
```

예를 들어 `GEO 도구`라는 키워드는 그대로 두면 검색어입니다. 하지만 SEO/GEO 워크플로우로 보면 아래처럼 확장됩니다.

| 단계 | 예시 |
|---|---|
| SEO 키워드 | GEO 도구 |
| 검색 의도 | 도구 탐색/비교/구매 검토 |
| SERP 갭 | 후보군, 가격, 리포트 예시, 비교 기준이 부족함 |
| 온페이지 액션 | title/첫 문단/H2/비교표/FAQ 정리 |
| 기술 액션 | 핵심 페이지 sitemap, canonical, schema 점검 |
| 권위 액션 | 외부 비교 글, PR, 디렉터리, 리뷰 신호 정리 |
| GEO 질문 | B2B SaaS 팀이 쓸 GEO 모니터링 도구를 추천해줘 |
| 측정 지표 | mention/source/citation, 경쟁사 비교, 재측정 변화 |

이렇게 바꾸면 SEO는 과거의 검색 순위 관리가 아니라 GEO 실행의 입력값이 됩니다.

## 왜 SEO 장을 따로 봐야 하나

GEO를 바로 시작하면 흔히 AI 질문셋이나 ChatGPT 답변 화면만 보게 됩니다. 하지만 질문셋은 허공에서 나오지 않습니다. 사람들이 이미 검색창에 입력한 키워드, 상위 SERP가 반복해서 다루는 하위 질문, 기존 콘텐츠의 구조와 기술 상태가 모두 질문셋의 재료입니다.

반대로 SEO만 보면 AI 답변 안에서 브랜드가 어떻게 언급되고 어떤 출처로 설명되는지는 놓치기 쉽습니다. 그래서 이 장의 목적은 SEO를 독립적으로 설명하되, 마지막에는 반드시 `AI 질문/답변 근거/측정`으로 연결하는 것입니다.

## SEO 워크플로우를 GEO로 확장하는 법

| SEO 단계 | 먼저 묻는 질문 | GEO로 확장한 질문 |
|---|---|---|
| 키워드 리서치 | 사용자는 어떤 단어로 찾는가? | AI에게 어떤 조건과 맥락으로 묻는가? |
| 검색 의도 분석 | 정보형/비교형/구매형 중 무엇인가? | 답변은 정의/추천/검증/실행 중 무엇을 해야 하는가? |
| SERP 분석 | 상위 문서는 무엇을 공통으로 답하는가? | AI 답변이 조합할 필수 정보 단위는 무엇인가? |
| 온페이지 SEO | 페이지가 주제와 답을 명확히 전달하는가? | AI가 가져갈 첫 답변/표/FAQ가 있는가? |
| 테크니컬 SEO | 크롤링/색인/대표 URL이 정상인가? | AI가 발견하고 읽고 인용할 수 있는 URL인가? |
| 권위 신호 | 외부 링크와 신뢰 신호가 있는가? | source/citation/consensus가 충분한가? |
| 성과 측정 | 노출/클릭/순위가 어떻게 변했는가? | mention/source/citation이 어떻게 변했는가? |

이 표를 기준으로 보면 GEO는 SEO의 대체재가 아니라 확장 레이어입니다.
