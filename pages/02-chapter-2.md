## GEO 측정 지표와 HaloX 분석 프레임

GEO를 실행하려면 먼저 “AI 검색에서 잘되고 있다”는 말을 측정 가능한 문장으로 바꿔야 합니다. 단순히 ChatGPT나 Perplexity 답변에 브랜드가 한 번 나왔는지만 보면 부족합니다. 어떤 질문군에서 언급되는지, 답변의 근거로 어떤 source가 쓰였는지, 사용자가 클릭할 수 있는 citation이 붙었는지, 경쟁사와 어떤 문맥에서 함께 비교되는지까지 함께 봐야 합니다.

이 장은 1장에서 만든 AI 질문셋을 실제 진단표로 바꾸는 단계입니다. 1주차 강의에서는 여기서 브랜드 baseline을 잡습니다. 실무에서는 이 baseline이 콘텐츠 우선순위, 제품 메시지, PR/오프사이트 전략, 테크니컬 점검의 출발점이 됩니다.

HaloX의 [AVI 점수 가이드](https://haloxlabs.ai/ko/blog/avi-score-explained)는 AI 검색 시대의 브랜드 가시성을 점수화해서 보는 관점을 설명합니다. 이 책에서는 그 개념을 강의와 실무 리포트에서 바로 쓸 수 있도록 mention/source/citation/경쟁 문맥/질문 포션 프레임으로 풀어봅니다.

## GEO 측정은 왜 클릭만으로 부족한가

기존 SEO에서는 순위, 노출, 클릭률, 유입, 전환이 핵심 지표였습니다. 이 지표들은 여전히 중요합니다. 문제는 AI 검색에서 사용자가 항상 링크를 클릭하지 않는다는 점입니다. AI가 이미 답변을 요약하고, 후보를 비교하고, 추천 이유까지 제시하면 사용자는 클릭 전에 많은 판단을 끝냅니다.

그래서 GEO에서는 “방문자가 몇 명 왔는가” 이전에 “AI가 우리 브랜드를 어떤 답변 재료로 쓰는가”를 봐야 합니다. 클릭이 없어도 브랜드가 비교 후보에 들어갔다면 수요 형성 단계에서 의미가 있습니다. 반대로 클릭은 있어도 AI 답변에서 경쟁사가 더 신뢰도 높은 선택지로 설명된다면 장기적으로는 위험 신호입니다.

## 다섯 가지 핵심 지표

GEO 측정은 최소 다섯 가지 축으로 나눠야 합니다.

| 지표 | 보는 질문 | 해석 기준 |
|---|---|---|
| Mention | 우리 브랜드가 답변에 등장하는가 | 인지도와 카테고리 포함 여부 |
| Source | AI가 답변을 만들 때 우리 콘텐츠를 근거로 쓰는가 | 콘텐츠가 답변 재료로 채택되는 정도 |
| Citation | 사용자가 볼 수 있는 링크로 인용되는가 | 클릭 가능한 근거와 신뢰 신호 |
| Co-mention | 어떤 경쟁사/대안과 함께 언급되는가 | 시장 내 포지션과 비교 문맥 |
| Answer quality | 어떤 이유로 추천/제외/설명되는가 | 메시지 정확도와 브랜드 이해도 |

여기서 중요한 것은 source와 citation을 분리해서 보는 일입니다. AI가 어떤 페이지를 참고해 답변을 만들었더라도 사용자 화면에 링크가 보이지 않을 수 있습니다. 반대로 citation은 붙었지만 답변의 핵심 문장을 구성하는 데 큰 역할을 하지 못할 수도 있습니다. 실무 리포트에서는 둘을 한 줄로 합치지 말고 따로 기록해야 합니다.

## 사례별로 다르게 읽어야 하는 지표

같은 mention 1회라도 업종과 목적에 따라 의미가 달라집니다. 강의에서는 아래 표를 먼저 보여주면 학습자가 “무엇을 측정해야 하는지”를 빠르게 이해합니다.

| 사례 유형 | 핵심 질문 | 가장 먼저 볼 지표 | HaloX로 설명할 기능 흐름 |
|---|---|---|---|
| PR 에이전시 상품화 | 고객에게 GEO 성과를 어떻게 보여줄까 | source/citation 분리, 1페이지 진단 | source/citation 추적, 대행사 검증 리포트 |
| 캠페인 URL 인용 추적 | 캠페인 URL이 AI 답변 근거로 반복 인용되는가 | citation 안정성, 검색 TOP10과 AI 답변 교차율 | AI 브리핑 인용 안정성, URL별 citation 리포트 |
| 엔터프라이즈 뉴스룸 | 뉴스룸이 브랜드 엔티티 허브로 작동하는가 | answer quality, entity 일관성, 출처 다양성 | 뉴스룸 점수화, 전략 프롬프트 세트 |
| 로컬 병원/지점 | 지역 질문에서 후보로 추천되는가 | 지역 질문 mention, 리뷰/source, 지점 URL citation | 지역 질문셋, 지점/URL/콘텐츠 클러스터 점검 |
| 금융/거래소 | 브랜드 신뢰와 비브랜드 전환 질문을 잡는가 | authority, co-mention, 비브랜드 질문 점유 | authority 진단, 브랜드/비브랜드 시계열 |
| 커머스/플랫폼 | AI가 상품 정보를 정확히 읽는가 | 상품 데이터 파싱, 리뷰/source, schema 상태 | 상품 정보 구조화 점검, merchant feed/llms.txt 체크 |

이 표는 [산업별 GEO 익명 케이스북](https://wikidocs.net/346381)의 요약판입니다. 02장에서는 사례를 길게 설명하기보다, 사례마다 어떤 지표가 달라지는지 먼저 잡습니다.

## 질문 포션을 먼저 정해야 한다

GEO 지표는 질문셋 없이 보면 의미가 흐려집니다. 브랜드가 100개 질문 중 20개에서 언급됐다는 숫자는 좋아 보일 수 있습니다. 하지만 그 20개가 구매 의도와 무관한 정의형 질문이라면 사업적 의미는 약합니다. 반대로 30개 질문 중 8개만 언급됐더라도 그 질문이 비교형/추천형/검증형이라면 우선순위가 높습니다.

그래서 1장에서 만든 질문셋을 다음처럼 포션으로 나눕니다.

| 질문 포션 | 예시 | 측정 목적 |
|---|---|---|
| 정보 탐색형 | GEO란 무엇인가 | 카테고리 이해도 확인 |
| 비교형 | A 도구와 B 도구를 비교해줘 | 경쟁 문맥 확인 |
| 추천형 | B2B SaaS 팀이 쓸 GEO 도구를 추천해줘 | 후보군 포함 여부 확인 |
| 검증형 | GEO 대행사를 고를 때 무엇을 봐야 하나 | 신뢰 기준과 리스크 확인 |
| 실행형 | 우리 브랜드의 AI 검색 노출을 점검하는 방법 | 실무 전환 가능성 확인 |

이 포션을 정해야 baseline 진단이 가능합니다. 어떤 질문은 언급률이 낮아도 괜찮고, 어떤 질문은 반드시 잡아야 합니다. 특히 비교형/추천형/검증형 질문에서 빠지는 것은 단순 노출 문제가 아니라 구매 후보군에서 빠지는 문제입니다.

## HaloX 분석 프레임으로 읽는 순서

GEO 리포트는 숫자를 많이 보여주는 것보다 “무엇을 먼저 고쳐야 하는가”를 알려줘야 합니다. HaloX 같은 AI 검색 모니터링 도구를 볼 때도 같은 순서로 읽으면 좋습니다.

1. 질문셋을 포션별로 나눈다.
2. 각 질문에서 브랜드 mention 여부를 확인한다.
3. mention이 있다면 설명 품질과 추천 이유를 본다.
4. mention이 없다면 어떤 경쟁사/대안이 등장하는지 본다.
5. 답변의 source와 citation을 분리해 기록한다.
6. 반복적으로 등장하는 출처/경쟁사/표현을 묶는다.
7. 콘텐츠 수정, 신규 글, 오프사이트 신호, 테크니컬 점검 중 무엇이 먼저인지 결정한다.

이 흐름에서 [HaloX](https://haloxlabs.ai/)는 단순 순위 확인 도구가 아니라 AI 답변 안에서 브랜드가 어떻게 읽히는지 확인하는 분석 프레임으로 쓰입니다. 제품 화면을 볼 때도 점수 하나만 보지 말고, 질문군/출처/경쟁 문맥/답변 품질을 함께 읽어야 합니다.

## 우리 기능으로 설명하는 리포트 카드

GEO 강의에서는 기능명을 먼저 외우게 하기보다, 리포트 카드가 어떤 질문에 답하는지 보여주는 편이 좋습니다.

| 리포트 카드 | 답하는 질문 | 연결되는 실행 |
|---|---|---|
| AVI/브랜드 가시성 | AI 답변에서 우리 브랜드가 얼마나 보이는가 | 질문군별 baseline과 경쟁사 비교 |
| Mention map | 어느 질문에서 등장하고 빠지는가 | 3장 fan-out 질문맵 보강 |
| Source/Citation map | 어떤 출처가 답변 재료와 화면 링크로 쓰이는가 | 5장 source map과 PR/오프사이트 실행 |
| Co-mention map | 어떤 경쟁사/대안과 함께 비교되는가 | 포지셔닝 문구와 비교 콘텐츠 작성 |
| AI answer quality | AI가 우리 브랜드를 정확히 설명하는가 | 4장 answer-first 콘텐츠 리라이트 |
| Technical readability | AI crawler가 페이지를 읽을 수 있는가 | 6장 sitemap/schema/llms.txt 점검 |
| Campaign URL tracking | 특정 URL이 반복 인용되는가 | 캠페인 성과 리포트와 다음 배포 계획 |

이렇게 설명하면 HaloX 기능이 홍보 문구가 아니라 학습자의 진단 도구가 됩니다. “어떤 버튼이 있다”가 아니라 “이 카드가 어떤 의사결정을 줄여주는가”로 이해시키는 것이 핵심입니다.

## 1주차 baseline 진단표

강의에서는 다음 표를 기본 산출물로 사용합니다. 처음부터 완벽한 자동화를 목표로 하기보다, 브랜드 상태를 한 번 손으로 읽어보는 것이 중요합니다.

| 항목 | 기록할 내용 |
|---|---|
| 질문 | 실제 AI에게 던질 질문 |
| 질문 포션 | 정보 탐색형/비교형/추천형/검증형/실행형 |
| 모델/플랫폼 | ChatGPT/Perplexity/Gemini/Google AI Overviews 등 |
| Mention | 브랜드 언급 여부 |
| Source | 답변 재료로 보이는 출처 |
| Citation | 화면에 표시된 인용 링크 |
| Co-mention | 함께 언급된 경쟁사/대안 |
| Answer quality | 설명이 정확한지, 추천 이유가 타당한지 |
| 다음 액션 | 콘텐츠 보강/출처 확보/기술 점검/메시지 수정 |

이 표를 채우면 “우리는 AI 검색에서 안 보인다”보다 훨씬 구체적인 진단이 나옵니다. 예를 들어 “추천형 질문에서는 경쟁사 A와 B가 반복 등장하지만 우리 브랜드는 source로만 쓰이고 mention은 약하다”처럼 문제를 좁힐 수 있습니다.

## 좋은 리포트와 나쁜 리포트

나쁜 GEO 리포트는 언급 횟수만 보여줍니다. “이번 달 37회 언급” 같은 숫자는 보기에는 좋지만, 어떤 질문에서 어떤 이유로 언급됐는지 없으면 실행으로 이어지지 않습니다.

좋은 GEO 리포트는 다음 질문에 답합니다.

- 어떤 질문군에서 우리 브랜드가 강한가?
- 어떤 질문군에서 경쟁사가 먼저 추천되는가?
- AI가 우리 브랜드를 설명할 때 틀리거나 약하게 말하는 부분은 무엇인가?
- 우리 콘텐츠가 source로 쓰이는가, citation으로 보이는가?
- 다음 30일 동안 고칠 수 있는 콘텐츠/출처/기술 과제는 무엇인가?

이 차이가 GEO를 단순 모니터링이 아니라 운영 전략으로 바꿉니다.

## 우선순위를 정하는 기준

모든 질문에서 한 번에 좋아지려고 하면 실행이 흐려집니다. 1차 진단 후에는 아래 순서로 우선순위를 잡습니다.

1. 구매 의도에 가까운 추천형/비교형 질문에서 빠지는 문제
2. 브랜드 설명이 틀리거나 오래된 문제
3. 경쟁사는 citation을 얻는데 우리는 source조차 되지 못하는 문제
4. 콘텐츠는 있는데 AI가 읽기 어려운 구조 문제
5. 외부 출처/리뷰/PR/엔터티 신호가 약한 문제
6. crawler 접근성, sitemap, schema, 렌더링 같은 기술 문제

이 기준을 쓰면 2장의 진단이 3장 fan-out, 4장 콘텐츠 구조, 5장 source/citation/entity, 6장 테크니컬 GEO, 7장 산업별 전략으로 자연스럽게 이어집니다.

## 흔한 실수

**Q. AI 답변에 브랜드가 한 번 나오면 GEO가 잘되고 있는 건가요?**

아닙니다. 어떤 질문에서 나왔는지, 추천 문맥인지 단순 언급인지, 근거 링크가 붙었는지까지 봐야 합니다.

**Q. Citation만 많으면 충분한가요?**

충분하지 않습니다. citation은 중요하지만, 답변의 핵심 판단에서 브랜드가 어떻게 설명되는지도 함께 봐야 합니다.

**Q. GEO 점수 하나로 의사결정해도 되나요?**

점수는 빠른 상태 파악에는 유용합니다. 하지만 실행 우선순위는 질문 포션, source/citation, 경쟁 문맥, 답변 품질을 함께 봐야 정할 수 있습니다.

**Q. baseline은 얼마나 자주 측정해야 하나요?**

초기에는 주 1회가 적당합니다. 질문셋과 콘텐츠 수정이 빠르게 바뀌는 시기에는 같은 질문으로 반복 측정해야 변화 방향을 읽을 수 있습니다.

## 사례로 이해하기

측정 지표를 표로 남기면 막연한 “AI에서 안 보인다”가 실행 가능한 문제로 바뀝니다. AcmeGEO 사례에서는 추천형 질문에서 mention이 낮고 citation이 없어 4장과 5장 과제가 먼저 보입니다.

실제 강의에서는 한 가지 브랜드만 보지 않고, 산업별 익명 사례를 같이 놓고 비교합니다. PR 에이전시형은 고객 제안서에 들어갈 1페이지 리포트가 중요하고, 로컬 병원형은 지역 질문셋과 리뷰/source가 중요하며, 커머스형은 상품 데이터 구조화가 중요합니다. 같은 GEO라도 측정표의 첫 열이 달라져야 합니다.

| 질문 포션 | 정보/비교/추천/검증/실행 |
| Mention | 브랜드 언급 여부와 문맥 |
| Source/Citation | 답변 재료와 화면 인용 링크 |
| 사례 유형 | PR/캠페인/뉴스룸/로컬/금융/커머스 중 어디에 가까운지 |
| 다음 액션 | 콘텐츠/출처/기술/메시지 중 우선 과제 |

## 제출 템플릿

```text
질문별 측정표 / 지표 요약 / 사례 유형 / 가장 약한 질문군 / 우선 액션 5개 / 재측정 날짜
```

## 작성 예시

예시는 가상의 브랜드를 기준으로 GEO 측정표를 만드는 방식입니다. 실제 운영에서는 같은 질문셋으로 반복 측정할 수 있게 지표와 날짜를 남깁니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 질문 | B2B SaaS 팀이 쓸 GEO 분석 도구를 추천해줘 |
| 질문 포션 | 추천형 |
| Mention | AcmeGEO 미언급, 경쟁사 3곳 반복 등장 |
| Source/Citation | AcmeGEO 콘텐츠 source 없음, citation 0개 |
| 사례 유형 | B2B SaaS/PR 에이전시 상품화형 |
| 다음 액션 | 비교표 콘텐츠 2개 작성, source map 10개 확보, 30일 뒤 같은 질문 재측정 |

## 완료 기준

- 개념 설명에서 끝나지 않고 다음 측정/작성/실행 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 산업별 사례 유형에 따라 우선 지표가 달라졌는지 설명할 수 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.
