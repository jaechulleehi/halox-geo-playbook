## Fan-out 질문맵으로 AI 질문셋 만드는 법

Fan-out 질문맵은 대표 질문 하나를 정의형/비교형/추천형/검증형/실행형 질문으로 펼쳐 AI가 답변을 만들 때 필요한 정보 단위를 찾는 도구입니다. GEO 실무에서는 검색어 하나보다 질문 묶음이 더 중요합니다.

질문을 넓게 펼치지 않으면 AI 답변에서 빠지는 하위 판단을 놓칩니다. 사용자는 한 문장으로 묻지만 AI는 정의, 비교, 조건, 신뢰도, 실행 방법을 함께 확인합니다.

## Fan-out에서 먼저 볼 것

| 하위 질문 유형 | 확인 질문 | 필요한 답변 재료 |
|---|---|---|
| 정의형 | 이 개념은 무엇인가? | 짧은 정의, 적용 범위, 오해 방지 |
| 비교형 | 어떤 선택지와 비교되는가? | 비교 기준, 장단점, 적합한 상황 |
| 추천형 | 어떤 조건에서 추천되는가? | 독자 조건, 추천 이유, 제외 기준 |
| 검증형 | 신뢰할 근거가 있는가? | 공식 문서, 리뷰, 사례, 화면 인용 후보 |
| 실행형 | 무엇부터 하면 되는가? | 순서, 체크리스트, 재측정 기준 |

## 실행 순서

1. Seed 질문 하나를 고릅니다.
2. 정의형/비교형/추천형/검증형/실행형으로 질문을 펼칩니다.
3. 각 질문이 요구하는 정보 단위를 적습니다.
4. 이미 답하는 콘텐츠와 비어 있는 콘텐츠를 분리합니다.
5. 우선 질문 5개와 재측정 날짜를 정합니다.

## 사례로 이해하기

B2B SaaS가 `GEO 모니터링 도구` 한 키워드만 보고 있다면 AI 답변 시장을 좁게 보는 것입니다. `우리 브랜드가 ChatGPT 답변에 나오는지 확인하는 법`, `Perplexity에서 경쟁사와 비교되는 기준`, `AI Overview에 인용되는 콘텐츠 조건`처럼 질문을 펼쳐야 실제 콘텐츠 과제가 보입니다.

## Fan-out이 필요한 배경

AI 답변은 하나의 질문에 대해 여러 하위 판단을 동시에 수행합니다. “GEO 도구 추천”이라는 질문에도 정의, 비교 기준, 가격, 신뢰도, 사용 난이도, 대체재, 실제 사례가 섞입니다. Fan-out은 이 숨은 하위 질문을 밖으로 꺼내는 작업입니다.

이 과정을 거치지 않으면 콘텐츠 과제가 너무 넓게 잡힙니다. “GEO 도구 글을 쓰자”가 아니라 “비교 기준을 설명하는 글이 없다”, “출처 추적 예시가 없다”, “초보자가 리포트를 읽는 법이 없다”처럼 과제가 작아져야 실행할 수 있습니다.

## Seed 질문 하나를 펼친 예

Seed 질문을 `GEO 도구 추천`으로 잡으면 실제 Fan-out은 아래처럼 넓어집니다.

| 질문 유형 | 확장 질문 예시 | 필요한 정보 |
|---|---|---|
| 정의형 | GEO 도구는 SEO 도구와 무엇이 다른가? | GEO/SEO 차이, 측정 대상 |
| 비교형 | GEO 도구와 AI 검색 모니터링 도구는 같은가? | 기능 범위, 리포트 지표 |
| 추천형 | B2B SaaS 팀이 먼저 볼 GEO 도구는 무엇인가? | 팀 규모, 예산, 질문셋 관리 방식 |
| 검증형 | GEO 리포트가 믿을 만한지 어떻게 확인하나? | 기준선, 재측정, 답변 근거/화면 인용 분리 |
| 실행형 | 30일 동안 GEO 도구로 무엇을 개선할 수 있나? | 우선 질문, 콘텐츠 갭, 리라이트 대상 |

이렇게 펼치면 `도구 추천`이라는 한 단어가 도구 비교 글, 리포트 검증 글, 기준선 진단표, 콘텐츠 갭 분석으로 나뉩니다. Fan-out 질문맵의 목적은 질문을 많이 만드는 것이 아니라, 어떤 질문이 어떤 실행 과제로 이어지는지 보이게 하는 것입니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| Seed 질문 | 처음 출발할 대표 질문 1개 |
| Fan-out 질문 | 정의형/비교형/추천형/검증형/실행형으로 확장 |
| 현재 답변 자산 | 이미 답하는 블로그/제품/문서 |
| 빈칸 | AI가 답하려면 필요하지만 우리에게 없는 정보 |
| 다음 액션 | 신규 작성/리라이트/출처 보강 |

## 정리 양식

```text
브랜드명 / Seed 질문 / 확장 질문 20개 / 우선 질문 5개 / 연결할 기존 자산 / 새로 만들 콘텐츠 / 재측정 날짜
```

## 작성 예시

이 예시는 개념을 실제 운영 언어로 바꿔 보는 용도입니다. 그대로 베끼기보다 자기 브랜드의 질문, 페이지, 출처 후보로 바꿔 적습니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 브랜드 | AcmeGEO |
| Seed 질문 | AI 검색에서 우리 브랜드가 보이는지 확인하는 방법은? |
| 확장 질문 | GEO 모니터링 도구 추천 / ChatGPT 브랜드 언급 측정법 / Perplexity 경쟁사 비교 기준 / AI Overview 인용 조건 / GEO 리포트 읽는 법 |
| 우선 질문 | ChatGPT 답변에 우리 브랜드가 빠지는 이유는? |
| 다음 액션 | 02장 기준으로 기준선을 다시 측정하고, 04장 리라이트 후보 3개를 만든다 |

## 완료 기준

- “Fan-out 질문맵으로 AI 질문셋 만드는 법” 실습 결과가 질문맵으로 남습니다.
- 질문별 콘텐츠/출처 빈칸이 분리됩니다.
- 다음 실습에서 처리할 우선 질문이 정해집니다.

## 참고 링크 패키지

Seed 키워드를 질문 묶음으로 확장하는 감각이 필요할 때는 HaloX의 [SEO/GEO 키워드 전략 프레임워크](https://haloxlabs.ai/ko/blog/seo-geo-keyword-strategy-framework)를 함께 보면 좋습니다. 이 페이지의 워크시트로 질문맵을 만들고, HaloX 글에서는 키워드가 AI 질문 시장으로 바뀌는 과정을 더 넓게 확인합니다.

질문을 콘텐츠 구조로 바꿀 때는 Google의 [유용한 콘텐츠 만들기](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)도 함께 봅니다. 질문을 많이 만드는 것보다 실제 독자의 의사결정에 도움이 되는 답변 재료를 갖추는 것이 더 중요하기 때문입니다.

## 체크리스트

- Seed 질문이 너무 넓거나 추상적이지 않은가?
- 하위 질문이 정의/비교/추천/검증/실행으로 나뉘는가?
- 각 질문마다 필요한 콘텐츠와 출처가 보이는가?
- 우선 처리할 질문 3~5개가 남았는가?
- 다음 재측정 기준을 정했는가?

## 흔한 질문

**Q. 이 내용을 이 책에서 바로 실습으로 써도 되나요?**

가능합니다. 다만 먼저 `Fan-out 질문맵으로 AI 질문셋 만드는 법`에서 어떤 판단 기준을 얻는지 설명한 뒤 실습으로 넘어가는 편이 좋습니다. 표를 채우는 것보다 왜 이 기준을 쓰는지 이해시키는 것이 먼저입니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 먼저 매출이나 상담으로 이어질 가능성이 높은 질문군 3~5개만 고르고, 그 질문에서 비어 있는 콘텐츠와 출처부터 처리합니다.
