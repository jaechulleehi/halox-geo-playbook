## Fan-out 질문맵으로 AI 질문셋 만드는 법

![Fan-out 질문맵과 실행 과제 흐름](../assets/images/page-heroes/halox-geo-03-01-fan-out-question-map-hero.png)

Fan-out 질문맵은 seed 질문 하나를 여러 하위 질문 노드로 펼쳐, AI가 답변에 쓰기 위해 확인할 정보 단위를 찾는 도구입니다. GEO 실무에서 중요한 것은 검색어 하나를 여러 표현으로 바꾸는 일이 아니라, AI가 답변을 만들 때 어떤 판단을 함께 수행하는지 보는 일입니다.

예를 들어 사용자가 `GEO 도구 추천`이라고 물어도 AI 답변은 추천 목록만 만들지 않습니다. GEO 도구가 무엇인지, SEO 도구와 무엇이 다른지, 어떤 팀에 적합한지, 어떤 지표를 측정하는지, 리포트가 믿을 만한지, 실제 30일 동안 무엇을 개선할 수 있는지를 함께 봅니다. Fan-out은 이 숨은 질문들을 밖으로 꺼내는 과정입니다.

## Fan-out 질문맵의 4요소

| 요소 | 설명 | 실무에서 적을 것 |
|---|---|---|
| Seed 질문 | 출발점이 되는 대표 질문 | 기준선에서 브랜드가 약했던 질문군 |
| 질문 노드 | seed 질문에서 갈라진 하위 질문 | 정의형/비교형/추천형/검증형/실행형 질문 |
| 커버 상태 | 현재 자산이 답할 수 있는 정도 | 충분/부분/없음 |
| 다음 액션 | 빈칸을 닫는 실행 과제 | 신규 작성/리라이트/source 보강/기술 점검 |

질문 노드마다 `왜 중요한가`를 한 줄로 붙이면 좋습니다. 질문맵은 예쁜 그래프가 아니라 콘텐츠 우선순위를 정하는 판단 도구이기 때문입니다.

## 질문 노드 분류 체계

| 노드 유형 | 확인 질문 | 필요한 답변 재료 | 주로 이어지는 과제 |
|---|---|---|---|
| 정의형 | 이 개념은 무엇인가? | 짧은 정의, 적용 범위, 오해 방지 | 첫 문단/용어 정의 보강 |
| 비교형 | 무엇과 비교되는가? | 비교 기준, 장단점, 선택 조건 | 비교표/대안 페이지 작성 |
| 추천형 | 어떤 조건에서 추천되는가? | 대상별 적합성, 제외 기준, 사례 | 추천 기준/고객 유형 보강 |
| 검증형 | 믿을 근거가 있는가? | 공식 문서, 리뷰, 사례, 외부 출처 | source/citation 보강 |
| 실행형 | 무엇부터 하면 되는가? | 단계, 체크리스트, 완료 기준 | 워크시트/30일 액션 작성 |
| 리스크형 | 무엇을 조심해야 하는가? | 실패 사례, 금지 표현, 정책/규제 | FAQ/주의사항/검수 기준 추가 |

초기에는 다섯 가지 유형만으로도 충분합니다. 다만 병원/금융/커머스처럼 리스크가 큰 업종은 리스크형 질문을 별도로 표시해야 합니다. AI가 잘못된 추천이나 과장된 문장을 만들면 노출 자체가 위험이 될 수 있기 때문입니다.

## 실행 순서

1. 02장에서 기준선이 약했던 질문군을 하나 고릅니다.
2. seed 질문을 사용자 말투로 씁니다.
3. 정의형/비교형/추천형/검증형/실행형/리스크형으로 하위 질문을 펼칩니다.
4. 각 질문이 요구하는 답변 재료를 적습니다.
5. 기존 콘텐츠/블로그/제품 페이지/뉴스룸/외부 출처가 답할 수 있는지 표시합니다.
6. 커버 상태를 `충분/부분/없음`으로 나눕니다.
7. 우선 질문 3~5개와 재측정 날짜를 정합니다.

## Seed 질문 하나를 펼친 예

Seed 질문을 `B2B SaaS 팀에 맞는 GEO 도구는 무엇인가?`로 잡으면 아래처럼 펼칠 수 있습니다.

| 깊이 | 질문 | 필요한 정보 | 커버 상태 | 다음 액션 |
|---|---|---|---|---|
| Seed | B2B SaaS 팀에 맞는 GEO 도구는 무엇인가? | 선택 기준 전체 | 부분 | 비교 기준 페이지 작성 |
| 1 | GEO 도구와 SEO 도구는 무엇이 다른가? | 측정 대상/리포트 차이 | 부분 | 09장 도구 비교와 연결 |
| 1 | 어떤 지표를 봐야 하나? | mention/source/citation/답변 품질 | 충분 | 02장으로 링크 |
| 2 | citation 추적이 왜 필요한가? | source와 citation 차이 | 부분 | 05장 source/citation 보강 |
| 2 | 보안/연동은 무엇을 물어봐야 하나? | 권한/데이터/연동 범위 | 없음 | 도구 검증 체크리스트 추가 |
| 3 | 30일 동안 무엇이 개선되었는지 어떻게 증명하나? | 기준선/재측정/액션 로그 | 부분 | 10장 실행 리포트와 연결 |

이 표를 만들면 `GEO 도구 추천 글 작성`이라는 막연한 과제가 `비교 기준 보강`, `citation 설명 보강`, `도구 검증 체크리스트`, `30일 실행 리포트`로 나뉩니다.

## 우선순위 점수 매기기

모든 질문을 한 번에 처리할 필요는 없습니다. 다음 기준으로 우선순위를 정합니다.

| 기준 | 질문 | 높은 점수의 의미 |
|---|---|---|
| 비즈니스 영향 | 이 질문이 상담/구매/신뢰에 가까운가? | 전환에 가까운 질문 |
| 현재 노출 약점 | 02장 기준선에서 mention/source/citation이 약한가? | 실제로 빠지고 있는 질문 |
| 답변 재료 부족 | 우리 자산이 답하지 못하는가? | 신규 작성 또는 리라이트 필요 |
| 출처 필요도 | 외부 근거가 없으면 믿기 어려운가? | 05장 source 전략으로 연결 |
| 실행 난이도 | 이번 주에 닫을 수 있는가? | 빠르게 개선 가능한 과제 |

점수는 정교한 수식보다 팀 합의용이면 충분합니다. `높음/중간/낮음`으로 표시해도 우선순위는 충분히 보입니다.

## Google 기준과 함께 읽기

Google의 [유용한 콘텐츠 만들기](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)는 콘텐츠가 독자에게 실제 도움이 되는지 묻습니다. Fan-out 질문맵도 같은 관점으로 써야 합니다. 질문을 많이 만들기 위한 표가 아니라, 독자가 실제로 판단하고 실행하는 데 필요한 답변 재료를 찾는 표여야 합니다.

또한 [SEO Starter Guide](https://developers.google.com/search/docs/fundamentals/seo-starter-guide)의 기본 원칙처럼 페이지가 검색엔진에 발견되고 이해될 수 있어야 합니다. GEO에서는 여기에 `이 페이지가 어떤 AI 질문의 답변 재료인가`를 추가로 표시합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| Seed 질문 | 기준선에서 약했던 대표 질문 1개 |
| 확장 질문 | 질문 유형별로 20개 내외 작성 |
| 필요한 답변 재료 | 정의/표/사례/공식 근거/FAQ/체크리스트 |
| 현재 자산 | 이미 답하는 블로그/제품/뉴스룸/외부 출처 |
| 커버 상태 | 충분/부분/없음 |
| 다음 액션 | 신규 작성/리라이트/source 보강/기술 점검 |
| 재측정 기준 | 같은 질문으로 다시 볼 날짜와 지표 |

## 흔한 실수

**Q. 질문을 많이 뽑으면 좋은 질문맵인가요?**

아닙니다. 좋은 질문맵은 질문 수가 아니라 실행 연결이 분명합니다. 각 질문이 어떤 콘텐츠/출처/기술 과제로 이어지는지 보여야 합니다.

**Q. AI에게 질문을 생성시키면 끝인가요?**

초안 생성에는 도움이 됩니다. 하지만 최종 질문맵은 기준선 측정 결과, 실제 고객 질문, 기존 콘텐츠 자산, 업종 리스크를 반영해 사람이 조정해야 합니다.

**Q. 모든 질문을 콘텐츠로 만들어야 하나요?**

아닙니다. 어떤 질문은 기존 페이지 리라이트로 충분하고, 어떤 질문은 외부 출처 보강이 더 중요합니다. 03-03에서 갭 유형을 분리합니다.
