## 질문셋에서 콘텐츠 갭을 찾는 법

![질문셋에서 콘텐츠 갭을 찾는 흐름](../assets/images/page-heroes/halox-geo-03-03-content-gap-from-questions-hero.png)

질문셋은 콘텐츠 아이디어 목록이 아니라 우선순위 판단표입니다. 질문은 있는데 답할 자산이 없으면 신규 콘텐츠, 자산은 있는데 구조가 약하면 리라이트, 설명은 있지만 근거가 약하면 source 보강, 콘텐츠는 있지만 AI가 읽기 어려우면 기술 점검으로 넘깁니다.

GEO에서 콘텐츠 갭은 단순히 `아직 안 쓴 글`이 아닙니다. AI가 답변을 만들 때 필요한 판단 재료가 우리 자산 안에 없거나, 있어도 찾기 어렵거나, 외부 출처로 뒷받침되지 않는 상태입니다. 그래서 콘텐츠 갭은 질문셋을 기준으로 찾아야 합니다.

## 콘텐츠 갭의 5가지 유형

| 갭 유형 | 판단 기준 | 다음 액션 | 연결 장 |
|---|---|---|---|
| 신규 작성 | 질문에 답할 페이지가 없음 | 새 가이드/FAQ/비교글 작성 | 04장 |
| 리라이트 | 페이지는 있지만 첫 답이 약함 | Answer-first 구조로 수정 | 04장 |
| source 보강 | 설명은 있으나 신뢰 근거가 약함 | 뉴스룸/외부 글/사례/리뷰 연결 | 05장 |
| 기술 점검 | 콘텐츠가 있지만 AI가 읽기 어려움 | schema, 내부 링크, 렌더링, sitemap 점검 | 06장 |
| 제품/메시지 보강 | 기능은 있으나 선택 이유가 설명되지 않음 | 비교 기준/대상 고객/제외 기준 정리 | 07장/09장 |

이 분류가 중요한 이유는 갭마다 담당자가 다르기 때문입니다. 신규 작성은 콘텐츠 팀이 처리할 수 있지만, source 보강은 PR/브랜드/뉴스룸이 필요하고, 기술 점검은 개발/SEO 담당자가 함께 봐야 합니다.

## 질문에서 갭을 찾는 순서

1. 02장 기준선에서 약한 질문 5개를 고릅니다.
2. 03-01 방식으로 질문을 하위 노드로 펼칩니다.
3. 각 질문이 요구하는 답변 재료를 적습니다.
4. 현재 자산이 그 재료를 제공하는지 확인합니다.
5. 빈칸을 신규 작성/리라이트/source 보강/기술 점검/제품 메시지 보강으로 나눕니다.
6. 이번 주에 닫을 갭과 다음 달에 볼 갭을 분리합니다.
7. 수정 후 같은 질문으로 재측정합니다.

## 질문에서 실행 과제로 이어지는 샘플

| 질문 | AI가 답하려면 필요한 정보 | 현재 자산 | 갭 유형 | 액션 |
|---|---|---|---|---|
| GEO 도구를 고를 때 무엇을 봐야 하나? | 측정 지표, 질문셋 관리, 리포트 신뢰도 | 블로그 글 1개 | 비교 기준 부족 | 도구 평가 기준 표 추가 |
| AI 검색에서 우리 브랜드가 왜 안 보이나? | 기준선, 경쟁사 언급, 답변 근거 후보 | 없음 | 신규 콘텐츠 필요 | 기준선 진단 가이드 작성 |
| 답변 근거(source)와 화면 인용(citation)은 무엇이 다른가? | 개념 정의, 예시, 오해 방지 | 용어 설명만 있음 | 설명 깊이 부족 | 조합별 예시와 FAQ 보강 |
| Perplexity에는 인용되는데 ChatGPT 추천에는 약한 이유는? | 플랫폼별 답변 방식, 추천 조건, 경쟁 맥락 | 일부 있음 | 리라이트 + 제품 메시지 보강 | 추천 문맥 기준표 작성 |
| 병원 GEO에서 후기 표현은 어디까지 안전한가? | 광고/후기 리스크, 안전한 문장, 리뷰 대응 기준 | 없음 | 신규 작성 + 리스크 보강 | 업종별 리스크 체크리스트 작성 |

이 표를 채우면 콘텐츠 갭이 `글이 없다`가 아니라 서로 다른 실행 과제로 분리됩니다. 특히 05장과 06장으로 넘어갈 갭을 여기서 미리 분리해야 합니다.

## 현재 자산을 평가하는 기준

| 평가 항목 | 질문 | 충분한 상태 |
|---|---|---|
| 첫 답변 | 페이지 첫 부분이 질문에 바로 답하는가? | 2~4문단 안에 결론이 보임 |
| 비교 기준 | AI가 후보를 비교할 기준이 있는가? | 표/조건/장단점이 있음 |
| 신뢰 근거 | 주장 뒤에 근거가 붙어 있는가? | 공식 문서/사례/source 후보가 있음 |
| 구조 | AI가 문단/표/FAQ를 읽기 쉬운가? | H2, 표, FAQ, 내부 링크가 정리됨 |
| 최신성 | 오래된 정보가 답변에 남지 않는가? | 날짜/정책/가격/기능 변화가 관리됨 |
| 전환 행동 | 독자가 다음 행동을 알 수 있는가? | 체크리스트/문의/다운로드/실행 단계가 있음 |

Google의 [유용한 콘텐츠 만들기](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)는 콘텐츠가 독자의 실제 문제를 해결하는지 확인하는 기준입니다. GEO 콘텐츠 갭 분석에서도 같은 질문을 해야 합니다. 이 페이지가 AI 답변에 들어가더라도 독자가 판단하고 행동하는 데 충분한가를 봅니다.

## 좋은 갭 진단 문장과 나쁜 갭 진단 문장

| 구분 | 문장 | 문제/장점 |
|---|---|---|
| 나쁨 | GEO 글이 부족하다 | 무엇이 부족한지 알 수 없음 |
| 나쁨 | 블로그를 5개 더 쓴다 | 질문/지표/목표가 없음 |
| 좋음 | `GEO 도구 추천` 질문에서 비교 기준은 있으나 citation 추적 설명이 약하다 | 보강 위치와 이유가 보임 |
| 좋음 | `병원 GEO 추천` 질문에서 후기/효과 표현 리스크를 다루는 페이지가 없다 | 신규 작성 필요성이 분명함 |
| 좋음 | `AI 검색 리포트` 질문에서 첫 답은 있으나 30일 액션으로 이어지지 않는다 | 리라이트 방향이 보임 |

## 흔한 질문

**Q. 콘텐츠 갭과 source 갭은 어떻게 구분하나요?**

우리 페이지 안에 설명 자체가 없으면 콘텐츠 갭입니다. 설명은 있지만 믿을 근거나 외부 합의 신호가 부족하면 source 갭입니다. 둘이 함께 있는 경우도 많습니다.

**Q. 기술 점검은 왜 콘텐츠 갭에 포함하나요?**

내용이 있어도 렌더링, canonical, 내부 링크, schema, sitemap 문제로 AI와 검색엔진이 제대로 읽지 못하면 답변 재료가 되기 어렵습니다. 이 경우는 06장으로 넘깁니다.

**Q. 갭을 모두 닫아야 하나요?**

아닙니다. 먼저 상담/구매/신뢰에 가까운 질문과 기준선에서 실제로 약한 질문을 처리합니다. GEO는 한 번에 완성하는 프로젝트가 아니라 질문셋 단위로 반복 개선하는 운영입니다.
