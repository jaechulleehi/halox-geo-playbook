## 질문셋에서 콘텐츠 갭을 찾는 법

질문셋은 콘텐츠 아이디어 목록이 아니라 우선순위 판단표입니다. 질문은 있는데 답할 자산이 없으면 신규 콘텐츠, 자산은 있는데 구조가 약하면 리라이트, 출처가 약하면 오프사이트 과제로 넘깁니다.

콘텐츠 팀은 글 수를 늘리는 데 집중하기 쉽습니다. 하지만 GEO에서는 어떤 질문에서 답변 재료가 부족한지 먼저 봐야 합니다.

## 콘텐츠 갭을 볼 때 먼저 확인할 것

| 질문 상태 | 의미 | 다음 액션 |
|---|---|---|
| 질문은 많고 답변 자산이 없음 | 신규 콘텐츠가 필요합니다 | 핵심 질문부터 새 글로 작성합니다 |
| 자산은 있지만 첫 답이 약함 | 리라이트가 필요합니다 | Answer-first 구조로 고칩니다 |
| 설명은 있지만 근거가 약함 | 답변 근거 보강이 필요합니다 | 공식 자료, 사례, 외부 출처를 연결합니다 |
| 답변은 있지만 비교 기준이 없음 | 비교/추천 문맥에서 빠질 수 있습니다 | 기준표와 선택 조건을 추가합니다 |
| 측정 기준이 없음 | 개선 여부를 판단하기 어렵습니다 | 기준선과 재측정 날짜를 정합니다 |

## 실행 순서

1. 우선 질문 5개를 고릅니다.
2. AI가 답하려면 필요한 근거와 설명을 적습니다.
3. 현재 보유한 페이지가 그 근거를 제공하는지 확인합니다.
4. 빈칸을 신규 작성/리라이트/source 보강으로 나눕니다.
5. 가장 빨리 닫을 수 있는 갭부터 실행 목록에 넣습니다.

## 사례로 이해하기

AI 답변에서 경쟁사는 비교 기준을 갖고 등장하지만 우리 브랜드는 이름만 언급된다면 콘텐츠 갭은 `브랜드 소개`가 아니라 `선택 이유`입니다. 질문셋을 기준으로 현재 자산이 무엇을 답하지 못하는지 표시합니다.

## 왜 질문셋에서 콘텐츠 갭을 찾아야 하나

콘텐츠 갭은 단순히 “아직 안 쓴 글”이 아닙니다. AI가 답변을 만들 때 필요한 판단 재료가 우리 자산 안에 없거나, 있어도 찾기 어렵거나, 외부 출처로 뒷받침되지 않는 상태입니다. 그래서 질문셋을 기준으로 갭을 봐야 합니다.

예를 들어 “GEO 도구 추천”이라는 질문에서 AI가 측정 모델 수, citation 추적, 경쟁사 비교, 리포트 액션을 판단 기준으로 쓰는데 우리 사이트가 AVI 점수만 설명하고 있다면, 부족한 것은 글의 양이 아니라 비교 기준입니다. 이 갭을 모르면 블로그 글을 더 써도 AI 답변의 선택 이유에는 들어가지 못합니다.

콘텐츠 갭 분석은 04장의 리라이트와 05장의 source 전략으로 이어집니다. 질문에서 부족한 답변 재료를 찾고, 그 재료를 어떤 페이지/표/FAQ/외부 출처로 보강할지 정하는 중간 단계입니다. Google의 [유용한 콘텐츠 만들기 가이드](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)는 독자에게 실제 도움이 되는 콘텐츠를 판단할 때 함께 참고할 수 있습니다.

## 질문에서 실행 과제로 이어지는 샘플

| 질문 | AI가 답하려면 필요한 정보 | 현재 자산 | 갭 유형 | 액션 |
|---|---|---|---|---|
| GEO 도구를 고를 때 무엇을 봐야 하나? | 측정 지표, 질문셋 관리, 리포트 신뢰도 | 블로그 글 1개 | 비교 기준 부족 | 도구 평가 기준 표 추가 |
| AI 검색에서 우리 브랜드가 왜 안 보이나? | 기준선, 경쟁사 언급, 답변 근거 후보 | 없음 | 신규 콘텐츠 필요 | 기준선 진단 가이드 작성 |
| 답변 근거(source)와 화면 인용(citation)은 무엇이 다른가? | 개념 정의, 예시, 오해 방지 | 용어사전만 있음 | 설명 깊이 부족 | 예시 4가지와 FAQ 보강 |

이 표를 채우면 콘텐츠 갭이 단순히 “글이 없다”가 아니라 신규 작성, 리라이트, 답변 근거 보강, 비교 기준 추가처럼 서로 다른 실행 과제로 분리됩니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 질문 | 측정한 프롬프트 |
| AI 답변의 판단 기준 | AI가 비교에 사용한 기준 |
| 우리 자산 여부 | 있음/부족/없음 |
| 갭 유형 | 콘텐츠/출처/기술/제품 설명 |
| 보강안 | 한 문장 실행안 |

## 정리 양식

```text
질문 / AI 답변 요약 / 경쟁사 언급 이유 / 우리 브랜드 누락 이유 / 보강 페이지 / 담당자 / 완료 기준
```

## 작성 예시

이 예시는 개념을 실제 운영 언어로 바꿔 보는 용도입니다. 그대로 베끼기보다 자기 브랜드의 질문, 페이지, 출처 후보로 바꿔 적습니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 질문 | B2B SaaS GEO 도구 추천 |
| AI 답변의 판단 기준 | 측정 모델 수 / citation 추적 / 경쟁사 비교 / 리포트 액션 |
| 우리 자산 | AVI 점수 설명 글은 있으나 경쟁사 비교 기준 글은 부족 |
| 갭 유형 | 콘텐츠 갭 + 출처 갭 |
| 보강안 | 도구 검증 체크리스트 페이지와 샘플 리포트 이미지를 추가한다 |

## 완료 기준

- “질문셋에서 콘텐츠 갭을 찾는 법” 실습 결과가 질문맵으로 남습니다.
- 질문별 콘텐츠/출처 빈칸이 분리됩니다.
- 다음 실습에서 처리할 우선 질문이 정해집니다.

## 참고 링크 패키지

질문에서 발견한 빈칸을 실제 콘텐츠 구조로 바꿀 때는 HaloX의 [GEO 콘텐츠 구조화 가이드](https://haloxlabs.ai/ko/blog/geo-content-structure)를 함께 봅니다. 질문 갭을 찾는 일은 이 페이지에서 하고, 그 갭을 answer-first 본문/FAQ/표로 바꾸는 감각은 HaloX 글로 보강합니다.

## 체크리스트

- 질문마다 필요한 답변 근거가 적혀 있는가?
- 기존 콘텐츠가 답하지 못하는 빈칸이 보이는가?
- 신규 작성과 리라이트 과제를 구분했는가?
- source 보강이 필요한 질문을 따로 표시했는가?
- 이번 주에 닫을 갭이 정해졌는가?

## 흔한 질문

**Q. 이 내용을 이 책에서 바로 실습으로 써도 되나요?**

가능합니다. 다만 먼저 `질문셋에서 콘텐츠 갭을 찾는 법`에서 어떤 판단 기준을 얻는지 설명한 뒤 실습으로 넘어가는 편이 좋습니다. 표를 채우는 것보다 왜 이 기준을 쓰는지 이해시키는 것이 먼저입니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 모든 빈칸을 한 번에 채우기보다 AI 답변 품질에 가장 크게 영향을 주는 콘텐츠 갭부터 닫습니다.
