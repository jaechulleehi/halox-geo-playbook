## Answer-first 콘텐츠는 어떻게 쓰는가

Answer-first 콘텐츠는 첫 2~4문단 안에서 정의, 결론, 적용 범위를 먼저 제시하는 글입니다. AI 검색에서는 핵심 답이 앞에 있어야 답변 재료로 쓰이기 쉽습니다.

긴 배경 설명으로 시작하면 독자도 AI도 핵심을 늦게 만납니다. GEO 글은 예열보다 답변이 먼저입니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

기존 글이 배경 설명으로 시작하면 AI는 핵심 답을 늦게 찾습니다. `GEO 콘텐츠 구조는 무엇인가`라는 질문에는 첫 문단에서 답을 주고, 이후 조건/예시/표/FAQ로 확장해야 합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 질문 | 이 페이지가 답할 질문 |
| 첫 답변 | 2~4문장 결론 |
| 판단 기준 | AI가 인용할 수 있는 기준 |
| 근거 | 수치/사례/표/출처 |
| 다음 링크 | 심화 페이지 또는 HaloX 글 |

## 제출 템플릿

```text
질문 / 첫 답변 / 왜 중요한가 / 판단 기준 3개 / 예시 / FAQ 2개 / 참고 링크
```

## 작성 예시

아래 예시는 `AcmeGEO`라는 가상의 B2B SaaS 브랜드를 기준으로 작성한 샘플입니다. 실제 강의에서는 브랜드명, 질문, 출처 후보, 담당자를 자기 상황에 맞게 바꿉니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 질문 | GEO 콘텐츠 구조는 무엇이 다른가? |
| 첫 답변 | GEO 콘텐츠는 AI가 답변에 바로 쓸 수 있도록 결론, 기준, 근거, 비교표, FAQ를 앞쪽에 배치한 콘텐츠입니다. |
| 판단 기준 | 첫 문단이 질문에 답하는가 / 표가 비교 기준을 고정하는가 / 출처 링크가 근거 역할을 하는가 |
| 근거 | HaloX GEO 콘텐츠 구조화 글과 실제 질문맵 |
| 다음 링크 | FAQ/schema 구조 페이지와 리라이트 체크리스트 |

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

## 흔한 질문

**Q. 이 페이지의 내용을 바로 강의자료로 써도 되나요?**

가능합니다. 다만 강의에서는 설명보다 실습 산출물을 먼저 정해야 합니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 현재 가장 중요한 질문군 하나를 고르고, 그 질문에서 보이는 콘텐츠/출처/기술 갭부터 처리합니다.
