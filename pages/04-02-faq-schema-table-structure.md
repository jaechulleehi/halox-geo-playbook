## FAQ/표/schema는 언제 쓰는가

FAQ, 표, 리스트, schema는 글을 꾸미는 장식이 아니라 질문과 답을 구조화하는 도구입니다. 특히 비교 기준과 실행 기준은 표로 분리할수록 AI가 이해하기 쉽습니다.

본문이 긴 문단으로만 되어 있으면 AI가 항목 간 차이를 파악하기 어렵습니다. 사람에게도 같은 문제가 생깁니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 반복 질문과 비교 항목을 분리합니다.
2. 짧은 답변은 FAQ로, 여러 기준 비교는 표로 보냅니다.
3. 구조화 데이터가 필요한 페이지 유형을 표시합니다.
4. schema가 본문 내용과 충돌하지 않는지 확인합니다.
5. 적용 후 crawler와 검색 노출 변화를 다시 봅니다.

## 사례로 이해하기

긴 문단만 있는 글은 AI가 항목별 근거를 분리하기 어렵습니다. FAQ는 질문 단위를 만들고, 표는 비교 기준을 고정하며, schema는 검색 엔진이 구조를 이해하는 보조 신호가 됩니다.

## FAQ/표/schema를 구분해야 하는 이유

FAQ, 표, schema는 모두 구조화 장치지만 역할이 다릅니다. FAQ는 짧은 질문과 답을 모을 때, 표는 여러 기준을 비교할 때, schema는 검색 엔진과 crawler가 페이지의 의미를 더 명확히 이해하도록 도울 때 사용합니다.

무조건 많이 넣는다고 좋은 것은 아닙니다. 본문에 없는 내용을 schema로 넣거나, 비교가 필요 없는 내용을 표로 만들면 오히려 신뢰가 떨어집니다. 구조화는 장식이 아니라 답변 재료를 정리하는 방식이어야 합니다. 기본 개념은 [schema.org](https://schema.org/)와 Google의 [구조화 데이터 소개](https://developers.google.com/search/docs/appearance/structured-data/intro-structured-data)를 참고합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 본문 질문 | FAQ로 바꿀 질문 |
| 짧은 답변 | 2~3문장 답 |
| 표 기준 | 비교할 축 |
| schema 후보 | FAQPage/HowTo/Article 등 |
| 주의점 | 과장/중복/빈약한 답변 제거 |

## 정리 양식

```text
FAQ 질문 5개 / 각 답변 2문장 / 비교표 열 3개 / 적용할 schema / 검수 체크
```

## 작성 예시

이 예시는 개념을 실제 운영 언어로 바꿔 보는 용도입니다. 그대로 베끼기보다 자기 브랜드의 질문, 페이지, 출처 후보로 바꿔 적습니다.

| 입력 항목 | 작성 예시 |
|---|---|
| FAQ 질문 | GEO 콘텐츠에 FAQ는 반드시 필요한가? |
| 짧은 답변 | 반드시 필요한 것은 아니지만 질문 단위를 명확히 만들 때 유리합니다. |
| 표 기준 | SEO 글 / GEO 글 / AI citation 대응 글 |
| schema 후보 | FAQPage와 Article |
| 주의점 | 같은 질문을 반복하거나 답변을 너무 짧게 쓰지 않는다 |

## 완료 기준

- “FAQ/표/schema는 언제 쓰는가” 기준으로 수정할 문단과 구조가 보입니다.
- 답변 우선 구조, FAQ, 표, schema의 역할이 분리됩니다.
- 리라이트 후 다시 확인할 질문셋이 남습니다.

## 흔한 질문

**Q. 이 내용을 이 책에서 바로 실습으로 써도 되나요?**

가능합니다. 다만 먼저 `FAQ/표/schema는 언제 쓰는가`에서 어떤 판단 기준을 얻는지 설명한 뒤 실습으로 넘어가는 편이 좋습니다. 표를 채우는 것보다 왜 이 기준을 쓰는지 이해시키는 것이 먼저입니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. FAQ, 표, schema는 필요한 곳에만 써야 합니다. 질문이 짧은 답을 요구할 때 FAQ를 쓰고, 비교 기준이 많을 때 표를 씁니다.
