## 기존 글을 GEO 콘텐츠로 리라이트하는 체크리스트

![GEO 콘텐츠 리라이트 체크리스트](../assets/images/page-heroes/halox-geo-04-03-geo-content-rewrite-checklist-hero.png)

GEO 리라이트는 문장 몇 개를 자연스럽게 바꾸는 일이 아닙니다. AI와 독자가 같은 질문에 대해 답, 기준, 근거, 다음 행동을 찾을 수 있도록 페이지 구조를 다시 잡는 작업입니다.

기존 SEO 글을 그대로 두고 FAQ만 붙이면 GEO 콘텐츠가 되지 않습니다. 02장에서 측정한 약한 질문군과 03장에서 찾은 fan-out 갭을 기준으로 고쳐야 합니다.

## 리라이트 전에 볼 것

수정할 페이지를 고르기 전에 먼저 질문을 봅니다. 어떤 질문에서 브랜드가 빠졌는지, 어떤 source/citation이 약했는지, 어떤 fan-out 노드가 비어 있는지 확인해야 합니다.

| 확인 항목 | 읽는 기준 |
|---|---|
| 약한 질문군 | 비브랜드/비교/추천/검증 중 어디서 빠졌는가 |
| 빠진 근거 | 사례, 비교 기준, 가격/기능, 리포트 예시 중 무엇이 없는가 |
| 수정 URL | 새 글이 필요한가, 기존 URL을 고치면 되는가 |
| 재측정 조건 | 같은 질문으로 다시 비교할 수 있는가 |

## 7단계 리라이트 흐름

1. 대표 질문을 하나 고른다.
2. 첫 문단을 Answer-first로 바꾼다.
3. 비교 기준과 판단 조건을 본문 위쪽에 둔다.
4. 근거가 필요한 곳에 사례, 출처, 수치, 공식 설명을 붙인다.
5. FAQ/표/schema는 본문 답변을 보조하는 범위에서 쓴다.
6. 관련 페이지로 내부 링크를 연결한다.
7. 같은 질문군으로 다시 측정한다.

이 순서에서 가장 자주 빠지는 단계는 7번입니다. 리라이트는 발행으로 끝나지 않고, 같은 조건의 재측정으로 끝납니다.

## Before/After

Before는 보통 이렇게 시작합니다.

```text
GEO는 최근 많은 기업이 관심을 갖는 AI 검색 최적화 방법입니다.
```

After는 질문의 답을 먼저 줍니다.

```text
GEO는 AI 답변 안에서 브랜드가 언급되고, 답변 근거와 화면 인용으로 연결되도록 콘텐츠와 출처, 기술 구조를 정리하는 작업입니다.
```

뒤 문장은 독자가 가져갈 답과 이 책의 측정 기준을 함께 담습니다.

![GEO 콘텐츠 리라이트 체크 보드](../assets/images/body-figures/halox-geo-04-03-geo-rewrite-check-board-codex-only.png)

*리라이트는 문장 교정이 아니라 질문, 답변, 근거, 재측정을 연결하는 구조 수정이다.*

## 페이지 유형별 액션

| 페이지 유형 | 먼저 고칠 것 |
|---|---|
| 정의 글 | 첫 문단의 뜻/구분/오해 정리 |
| 비교 글 | 선택 기준과 차이의 근거 |
| 제품 글 | 기능보다 사용 결과와 리포트 예시 |
| 사례 글 | 상황, 판단, 실행, 재측정 흐름 |
| 기술 글 | 점검 항목과 실제 확인 방법 |

## 05장으로 넘길 것

페이지 안의 문장 구조만으로 해결되지 않는 문제가 있습니다. 외부 출처가 부족하거나, 브랜드 설명이 여러 채널에서 다르게 쓰이거나, 화면 인용이 경쟁사 URL에 반복해서 붙는 경우입니다.

이때는 05장으로 넘어가 source/citation, 엔티티, 오프사이트 출처를 정리해야 합니다.

## 정리 양식

```text
수정할 URL:
대표 질문:
현재 첫 문단의 문제:
새 첫 답변:
보강할 근거:
연결할 내부 링크:
외부 source 보강 필요 여부:
재측정 질문과 날짜:
```
