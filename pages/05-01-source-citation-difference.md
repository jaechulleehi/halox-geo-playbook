## Source와 Citation은 무엇이 다른가

Source는 AI가 답변을 만들 때 참고한 정보 재료이고, citation은 사용자 화면에 보이는 인용 링크입니다. 둘을 분리해야 GEO 리포트가 실행 과제로 바뀝니다.

인용됐다는 말 하나로는 무엇을 고칠지 알 수 없습니다. source가 약한지, citation이 약한지, 브랜드 entity 설명이 흔들리는지 나눠야 합니다. 특히 PR 캠페인, 뉴스룸, 제품 페이지, 외부 기사처럼 여러 URL이 함께 움직이는 경우에는 source와 citation을 한 줄로 합치면 원인을 놓칩니다.

## 왜 둘을 분리해야 하나

AI 답변에서는 세 가지 상황이 자주 생깁니다.

| 상황 | 의미 | 다음 액션 |
|---|---|---|
| source로 보이지만 citation은 없음 | AI가 참고했을 가능성은 있지만 사용자가 클릭할 근거로 보이지 않음 | 요약문/표/원문 근거/citation 친화 구조 보강 |
| citation은 있지만 브랜드 mention은 약함 | 링크는 보이지만 답변의 추천 이유가 약함 | 답변에 들어갈 차별 문장과 비교 기준 보강 |
| 경쟁사 citation만 반복 | 같은 질문에서 경쟁사의 출처가 더 강함 | 경쟁사 source map 분석 후 외부 출처/뉴스룸 보강 |
| 오래된 source가 반복 | AI가 최신 설명보다 과거 자료를 더 신뢰함 | 최신 정책/팩트시트/정정 콘텐츠 발행 |

이 구분이 있어야 HaloX의 source/citation 리포트가 단순 현황표가 아니라 실행 우선순위가 됩니다.

## 사례로 이해하기

캠페인 URL 인용 추적 사례를 보면 차이가 분명합니다. 캠페인 랜딩 페이지가 검색 결과에는 있지만 AI 답변 citation으로 보이지 않을 수 있습니다. 이 경우 문제는 “검색 노출”이 아니라 “AI 답변이 그 URL을 근거로 보여줄 만큼 구조화되어 있는가”입니다.

PR 에이전시형 사례에서는 고객에게 “언급됐습니다”라고 말하는 것만으로 부족합니다. 어떤 질문에서 어떤 source가 쓰였고, 어떤 URL이 citation으로 드러났으며, 어느 질문에서는 경쟁사 URL만 보였는지를 나눠야 제안서가 설득력을 갖습니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 어떤 질문에서 citation을 기대하는가 | 질문 포션을 정보/비교/추천/검증으로 나눈다 |
| URL | 어떤 페이지가 source 후보인가 | 제품/블로그/뉴스룸/캠페인 URL을 분류한다 |
| 화면 | 실제 citation이 보이는가 | 플랫폼별 화면과 링크를 기록한다 |
| 문맥 | 답변에서 브랜드가 어떤 이유로 설명되는가 | answer quality와 co-mention을 함께 본다 |
| 반복성 | 같은 질문에서 다시 측정해도 유지되는가 | 7일/30일 재측정 기준을 정한다 |

## HaloX 기능으로 설명하는 법

Source/Citation 차이는 HaloX의 기능 설명과 바로 연결됩니다.

| 기능 흐름 | 설명 방식 |
|---|---|
| Source/Citation map | 답변 재료와 화면 인용 링크를 분리해 보여준다 |
| Campaign URL tracking | 특정 캠페인 URL이 어떤 질문에서 citation 되는지 추적한다 |
| Search TOP10 vs AI answer overlap | 검색에는 있는 URL이 AI 답변에는 빠지는지 비교한다 |
| Answer quality | citation이 있어도 추천 문맥이 약한지 확인한다 |
| 30일 재측정 | 보강 후 같은 질문셋으로 변화 여부를 본다 |

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 질문 | 측정 프롬프트 |
| Source 후보 | AI가 참고할 수 있는 페이지 |
| Citation 여부 | 답변에 실제 링크/출처로 드러났는가 |
| 인용 문맥 | 브랜드 설명/비교/추천/주의 문맥 |
| 누락 이유 | 권위/구조/접근성/문맥 |
| 보강안 | 출처성 강화 액션 |

## 제출 템플릿

```text
질문 / source 후보 URL / citation 여부 / 인용 문맥 / 빠진 이유 / 보강할 근거 / 다음 측정일
```

## 작성 예시

예시는 `Source와 Citation은 무엇이 다른가`를 가상의 브랜드에 적용한 것입니다. 실제 운영에서는 자사 페이지, 외부 근거, 인용 가능성을 따로 표시합니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 질문 | AI 검색에서 우리 브랜드가 왜 출처로 안 잡히나? |
| Source 후보 | HaloX AVI 점수 가이드, GEO 콘텐츠 구조화 글 |
| Citation 여부 | 답변 본문에는 언급되지만 링크 출처로는 드러나지 않음 |
| 인용 문맥 | GEO 측정 방법 설명에 참고된 것으로 보이나 추천 근거는 약함 |
| 누락 이유 | 페이지 안에 비교 기준과 원문 근거가 부족함 |
| 보강안 | 리포트 예시와 측정 기준 표를 추가한다 |

## 완료 기준

- source와 citation을 같은 지표로 합치지 않았습니다.
- 질문별 URL, 인용 문맥, 반복 측정 기준이 보입니다.
- 다음 액션이 콘텐츠 구조, 외부 source, 기술 접근성 중 어디인지 나뉩니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

## 흔한 질문

**Q. Citation이 없으면 source로도 쓰이지 않은 건가요?**

반드시 그렇지는 않습니다. AI가 답변을 만들 때 참고했더라도 화면 링크로 보여주지 않을 수 있습니다. 그래서 source 후보와 citation 결과를 따로 기록해야 합니다.

**Q. Source/Citation을 수동으로 봐도 되나요?**

초기 baseline은 수동으로 봐도 됩니다. 다만 질문 수가 늘어나면 HaloX 같은 도구로 반복 측정과 URL별 추적을 자동화해야 리포트가 안정됩니다.
