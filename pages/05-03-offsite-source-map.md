## 오프사이트 source map: 외부 출처 후보 설계

![오프사이트 출처 후보 맵](../assets/images/page-heroes/halox-geo-05-03-offsite-source-map-hero.png)

오프사이트 출처 후보 맵은 “백링크를 어디서 받을까”를 정리하는 표가 아닙니다. AI가 답변을 만들 때 참고할 수 있는 외부 근거를 질문별로 정리하는 작업입니다.

03장의 fan-out 노드와 04장의 페이지 구조를 기준으로, 어떤 질문에서 어떤 외부 출처가 부족한지 찾습니다.

## 질문별로 출처를 찾는다

모든 채널에 같은 콘텐츠를 뿌리면 source 전략이 흐려집니다. 추천형 질문에는 비교/리뷰/리포트 예시가 강하고, 검증형 질문에는 언론/공식 프로필/고객 사례가 강할 수 있습니다.

| 질문 유형 | 필요한 출처 후보 |
|---|---|
| 정의형 | 용어사전, 공식 가이드, 업계 설명 글 |
| 비교형 | 비교 글, 리뷰, 대안 페이지 |
| 추천형 | 도구 목록, 사례, 리포트 예시 |
| 검증형 | 언론, 프로필, 인증, 고객 사례 |
| 실행형 | 체크리스트, 템플릿, 운영 가이드 |

## 출처 후보를 고르는 순서

1. 약한 질문군을 고른다.
2. 답변에서 반복되는 경쟁 URL을 확인한다.
3. 그 URL이 어떤 역할을 하는지 분류한다.
4. 우리 공식 URL로 해결할 문제와 외부 출처가 필요한 문제를 나눈다.
5. 30일 안에 만들거나 고칠 수 있는 출처부터 정한다.

오프사이트 전략은 “많이”보다 “어떤 질문의 어떤 근거를 채우는가”가 먼저입니다.

![오프사이트 source map 보드](../assets/images/body-figures/halox-geo-05-03-offsite-source-map-board-codex-only.png)

*출처 후보 맵은 질문, 경쟁 URL, 우리 액션을 한 장에 연결한다.*

## 우선순위 판단

좋은 출처 후보는 세 조건을 만족합니다. 질문과 직접 연결되고, 독자가 읽어도 도움이 되며, 브랜드 설명의 합의 신호를 강화합니다.

단순 링크 삽입이나 얕은 소개 글은 우선순위가 낮습니다. AI가 답변 근거로 삼을 만한 문장, 비교 기준, 사례, 최신 정보가 있어야 합니다.

## AcmeGEO 예시

AcmeGEO가 추천형 질문에서 경쟁사 비교 글에 밀리고 있다면, 첫 액션은 외부 블로그 10개 발행이 아닙니다. 먼저 경쟁 글이 어떤 기준으로 인용되는지 봅니다. 가격, 리포트 예시, citation 추적, 업종 적합성 중 무엇이 강한지 확인합니다.

그다음 공식 비교 페이지와 외부 해설 글을 한 묶음으로 보강합니다. 이렇게 해야 source와 citation 후보가 함께 생깁니다.

## 정리 양식

```text
약한 질문군:
반복 경쟁 URL:
경쟁 URL의 역할:
우리 공식 URL 보강 과제:
필요한 외부 출처:
우선순위:
재측정 질문:
```
