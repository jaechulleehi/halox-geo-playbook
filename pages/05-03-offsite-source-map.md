## 오프사이트 출처 후보 맵을 만드는 법

오프사이트 출처 후보 맵은 질문군별로 AI가 자주 참고하는 외부 채널과 우리 브랜드가 보강해야 할 신호를 정리하는 표입니다. 백링크 수보다 질문 문맥과 출처 적합성이 중요합니다.

외부 링크를 많이 만드는 방식은 GEO 전략이 아닙니다. 어떤 질문에서 어떤 출처가 답변 재료로 쓰이는지 봐야 합니다. PR 기사, 리뷰 사이트, 커뮤니티 글, 유튜브 설명, 파트너 페이지, 디렉터리, 위키성 자료는 모두 역할이 다릅니다.

## Source map의 기본 구조

오프사이트 source map은 출처 후보를 URL 목록으로 끝내지 않습니다. 질문, 출처 유형, 권위 신호, 획득 방식, 리스크를 함께 기록합니다.

| 항목 | 기록할 내용 | 예시 |
|---|---|---|
| 질문군 | 어떤 AI 질문에 영향을 주는가 | GEO 도구 추천, 병원 선택 기준 |
| 출처 유형 | 언론/리뷰/커뮤니티/위키/파트너/유튜브 | 업계 미디어, 지도 리뷰 |
| 권위 신호 | 왜 AI가 참고할 만한가 | 전문성, 사용자 리뷰, 도메인 신뢰 |
| 획득 방식 | 어떻게 만들거나 수정할 수 있는가 | PR, 기고, 프로필 정리, 제휴 |
| 리스크 | 오래된 정보나 부정확한 설명이 있는가 | 과거 가격, 단종 상품, 이슈 기사 |

## 사례로 이해하기

로컬 병원/지점형 사례에서는 지도 리뷰, 지역 미디어, 지점 페이지, 전문의 프로필, FAQ가 함께 source map에 들어갑니다. 단순 블로그 글보다 지역 질문에서 어떤 외부 신호가 추천 기준으로 쓰이는지 봐야 합니다.

B2B SaaS 사례에서는 업계 미디어, 파트너 페이지, 비교 리뷰, 고객 사례, 통합 문서가 중요합니다. “도구 추천” 질문에서 AI는 제품 페이지 하나만 보지 않고 외부 비교 문맥을 함께 참고할 수 있습니다.

커머스/플랫폼 사례에서는 상품 리뷰, merchant feed, 카테고리 페이지, 유튜브 리뷰, 커뮤니티 반응이 source map에 들어갑니다. 가격/재고/리뷰처럼 변동성이 큰 정보는 최신성과 구조화가 중요합니다.

## HaloX 기능으로 설명하는 법

Source map은 HaloX 기능을 실행 계획으로 바꾸는 연결 지점입니다.

| 기능 흐름 | 설명 방식 |
|---|---|
| 반복 source 추출 | 질문셋에서 자주 등장하는 출처 유형을 묶는다 |
| Source/Citation 분리 | 근거로 쓰이는 출처와 화면에 보이는 링크를 나눈다 |
| Co-mention 분석 | 어떤 경쟁사와 함께 언급되는지 본다 |
| Offsite priority | 보강할 외부 채널을 난이도/영향 기준으로 정한다 |
| 30일 리포트 | PR/리뷰/프로필/콘텐츠 보강 후 재측정한다 |

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 출처 후보 | 사이트/미디어/리뷰/커뮤니티 |
| 권위 신호 | 도메인/전문성/업계 관련성 |
| 획득 방식 | 기고/제휴/PR/프로필 정리 |
| 연결 질문 | 어떤 AI 질문에 기여하는가 |
| 리스크 | 오래된 정보/부정확한 설명/경쟁사 편향 |
| 우선순위 | 난이도 대비 영향 |


## 완료 기준

- 출처 후보가 단순 URL 목록이 아니라 질문군과 연결되어 있습니다.
- 외부 채널별 역할이 나뉘어 있습니다.
- 획득 방식과 리스크가 함께 적혀 있습니다.
- 다음 측정 시점과 담당자가 보입니다.

## 흔한 질문

**Q. 오프사이트 source map은 백링크 목록인가요?**

아닙니다. 백링크는 일부일 뿐입니다. GEO에서는 질문별로 AI가 신뢰할 만한 근거로 참고하는 채널을 찾고, 그 채널의 문맥을 보강하는 것이 핵심입니다.

**Q. 모든 외부 채널을 동시에 관리해야 하나요?**

아닙니다. 먼저 AI 답변에 반복 등장하는 출처와 구매/신뢰 질문에 가까운 출처를 우선합니다.
