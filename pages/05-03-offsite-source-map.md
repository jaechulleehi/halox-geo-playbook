## 오프사이트 출처 후보 맵을 만드는 법

![오프사이트 출처 후보 맵](../assets/images/page-heroes/halox-geo-05-03-offsite-source-map-hero.png)

오프사이트 출처 후보 맵은 질문군별로 AI가 자주 참고하는 외부 채널과 우리 브랜드가 보강해야 할 신호를 정리하는 표입니다. 백링크 수보다 질문 문맥과 출처 적합성을 봐야 합니다.

외부 링크를 많이 만드는 방식은 GEO 전략이 아닙니다. 어떤 질문에서 어떤 출처가 답변 재료로 쓰이는지 봐야 합니다. PR 기사, 리뷰 사이트, 커뮤니티 글, 유튜브 설명, 파트너 페이지, 디렉터리, 위키성 자료는 모두 역할이 다릅니다.

## 03→04→05 흐름으로 출처 후보를 찾기

오프사이트 출처 후보 맵은 아무 외부 URL이나 모으는 표가 아닙니다. 03장에서 AI fan-out 노드를 보고, 04장에서 자사 페이지의 답변 재료를 만든 뒤, 05-03에서 그 답변 재료를 보강할 외부 근거를 찾습니다.

```text
AI fan-out 노드
→ 자사 페이지의 답변 재료
→ 부족한 source/citation/entity 신호
→ 외부 출처 후보
→ 수정/발행/요청/관찰 액션
```

| fan-out 노드 | 04장의 자사 자산 | 05-03의 외부 source 후보 |
|---|---|---|
| 정의 | 용어/가이드 페이지 | 위키/디렉터리/업계 미디어 |
| 비교 | 비교표/대안 페이지 | 리뷰 사이트, 비교 기사, 커뮤니티 토론 |
| 검증 | 공식 팩트시트/뉴스룸 | 언론 기사, 리포트, 파트너 자료 |
| URL 확인 | 제품/사례/캠페인 대표 URL | 고객사 사례, 파트너 페이지, 외부 인용 페이지 |
| 리스크 | FAQ/정책/정정 문서 | 공식 입장문, 신뢰 가능한 기사, 커뮤니티 대응 |

## 답변 근거 맵의 기본 구조

![오프사이트 source map 보드](../assets/images/body-figures/halox-geo-05-03-offsite-source-map-board-codex-only.png)

_query별 출처 후보를 채널과 우선순위로 나누면 오프사이트 작업이 실행 단위로 바뀝니다._


오프사이트 답변 근거 맵은 출처 후보를 URL 목록으로 끝내지 않습니다. 질문, 출처 유형, 권위 신호, 획득 방식, 리스크를 함께 기록합니다.

| 항목 | 기록할 내용 | 예시 |
|---|---|---|
| 질문군 | 어떤 AI 질문에 영향을 주는가 | GEO 도구 추천, 병원 선택 기준 |
| 출처 유형 | 언론/리뷰/커뮤니티/위키/파트너/유튜브 | 업계 미디어, 지도 리뷰 |
| 권위 신호 | 왜 AI가 참고할 만한가 | 전문성, 사용자 리뷰, 도메인 신뢰 |
| 획득 방식 | 어떻게 만들거나 수정할 수 있는가 | PR, 기고, 프로필 정리, 제휴 |
| 리스크 | 오래된 정보나 부정확한 설명이 있는가 | 과거 가격, 단종 상품, 이슈 기사 |


## 출처 후보를 찾는 실제 절차

오프사이트 출처 후보 맵은 책상에서 URL을 떠올려 적는 문서가 아닙니다. 실제 AI 답변과 경쟁사 답변에서 반복되는 출처를 뽑아야 합니다.

1. 브랜드/카테고리/비교/신뢰/리스크 질문셋을 만듭니다.
2. 각 질문에서 브랜드 mention, source 후보, 화면 citation을 기록합니다.
3. 같은 질문에서 경쟁사가 어떤 출처와 함께 등장하는지 봅니다.
4. 반복 URL을 언론, 위키/디렉터리, 리뷰, 커뮤니티, 파트너, 뉴스룸으로 분류합니다.
5. 각 출처가 수정 가능한지, 새로 확보해야 하는지, 관찰만 해야 하는지 나눕니다.
6. 30일 안에 바꿀 수 있는 액션과 장기 과제를 분리합니다.

이 절차를 거치면 “어디에 글을 더 올릴까”가 아니라 “어떤 질문에서 어떤 외부 근거가 부족한가”가 보입니다.

## query별 source map 예시

오프사이트 source map은 채널 목록이 아니라 query별 근거 배치표입니다. 어떤 질문에서 어떤 외부 출처가 답변 근거로 필요하고, 어떤 owned URL과 연결되어야 하는지 함께 봐야 합니다.

| query/질문군 | 필요한 외부 source | 연결할 owned URL | 우선 액션 |
|---|---|---|---|
| GEO 도구 비교 | 업계 미디어, 파트너 비교 글, 리뷰 | 도구 비교 페이지 | 비교 기준 기고/파트너 글 확보 |
| ChatGPT 브랜드 노출 확인 | 실무 가이드, 케이스 스터디 | 기준선 진단 가이드 | 리포트 샘플과 사례 공개 |
| GEO 대행사 비용 | 제안서 검토 글, 시장 해설 | 대행사 체크리스트 | 비용/범위/성과 기준 정리 |
| 병원 로컬 GEO | 지도/리뷰/정책 자료 | 병원 GEO 체크리스트 | 리뷰/NAP/의료광고 source 분리 |
| AcmeGEO 브랜드 질문 | 디렉터리, 뉴스룸, 파트너 프로필 | About/제품 페이지 | 카테고리 문장과 URL 통일 |

이 표를 만들면 외부 채널 운영이 “어디에 글을 올릴까”에서 “어떤 질문의 어떤 fan-out 노드를 닫을까”로 바뀝니다.

## 오프사이트 우선순위 점수표

모든 출처 후보를 동시에 관리할 수는 없습니다. 우선순위는 영향도와 실행 가능성을 함께 봐야 합니다.

| 기준 | 1점 | 3점 | 5점 |
|---|---|---|---|
| 답변 영향도 | 거의 등장하지 않음 | 가끔 source/citation 후보로 등장 | 여러 질문에서 반복 등장 |
| 신뢰도 | 개인 의견/출처 불명 | 업계 블로그/파트너 글 | 언론/공식 문서/권위 있는 리뷰 |
| 수정 가능성 | 수정 불가 | 요청 가능 | 직접 수정/발행 가능 |
| 리스크 | 문제 없음 | 일부 오래된 정보 | 오해/규제/평판 리스크 큼 |
| 구매 질문 거리 | 일반 정보 질문 | 비교/검토 질문 | 구매/신뢰/추천 질문에 직접 영향 |

```text
우선순위 점수 = 답변 영향도 + 신뢰도 + 수정 가능성 + 리스크 + 구매 질문 거리
```

점수가 높다고 무조건 좋은 출처는 아닙니다. 리스크 점수가 높은 출처는 “강화”가 아니라 “정정/대응” 대상일 수 있습니다.

## 사례로 이해하기

로컬 병원/지점형 사례에서는 지도 리뷰, 지역 미디어, 지점 페이지, 전문의 프로필, FAQ가 함께 답변 근거 맵에 들어갑니다. 단순 블로그 글보다 지역 질문에서 어떤 외부 신호가 추천 기준으로 쓰이는지 봐야 합니다.

B2B SaaS 사례에서는 업계 미디어, 파트너 페이지, 비교 리뷰, 고객 사례, 통합 문서를 봐야 합니다. “도구 추천” 질문에서 AI는 제품 페이지 하나만 보지 않고 외부 비교 문맥을 함께 참고할 수 있습니다.

커머스/플랫폼 사례에서는 상품 리뷰, merchant feed, 카테고리 페이지, 유튜브 리뷰, 커뮤니티 반응이 답변 근거 맵에 들어갑니다. 가격/재고/리뷰처럼 변동성이 큰 정보는 최신성과 구조화를 봐야 합니다.

## 흔한 질문

**Q. 오프사이트 답변 근거 맵은 백링크 목록인가요?**

아닙니다. 백링크는 일부일 뿐입니다. GEO에서는 질문별로 AI가 신뢰할 만한 근거로 참고하는 채널을 찾고, 그 채널의 문맥을 보강하는 것이 핵심 기준으로 봅니다.

**Q. 모든 외부 채널을 동시에 관리해야 하나요?**

아닙니다. 먼저 AI 답변에 반복 등장하는 출처와 구매/신뢰 질문에 가까운 출처를 우선합니다.
