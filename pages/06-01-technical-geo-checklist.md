## 테크니컬 GEO 기본 점검표

테크니컬 GEO는 AI가 콘텐츠를 발견하고 읽고 해석할 수 있게 만드는 기본 체력입니다. robots.txt, sitemap, canonical, schema, 응답 코드, 렌더링 상태를 함께 봅니다.

좋은 콘텐츠가 있어도 crawler가 접근하지 못하거나 본문을 읽지 못하면 GEO 성과를 설명하기 어렵습니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

좋은 콘텐츠가 있어도 AI crawler가 접근하지 못하면 GEO 성과는 제한됩니다. robots, sitemap, canonical, 내부 링크, 페이지 속도, 구조화 데이터부터 확인합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 점검 항목 | robots/sitemap/canonical/schema/internal link |
| 현재 상태 | 정상/주의/오류 |
| 증거 | 확인 URL 또는 캡처 메모 |
| 영향 | AI 접근/인덱싱/출처성 |
| 수정 담당 | 개발/마케팅/콘텐츠 |

## 제출 템플릿

```text
도메인 / 점검일 / 오류 항목 / 영향받는 페이지 / 수정 담당 / 완료 기준 / 재점검일
```

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

## 참고 링크 패키지

이 실습은 HaloX의 [온페이지 SEO 체크리스트](https://haloxlabs.ai/ko/blog/on-page-seo-checklist-2026)와 함께 보면 좋습니다. 함께 볼 링크를 하나만 고르지 말고, 내부 WikiDocs 흐름과 HaloX 심화 글을 같이 연결해 강의자료/워크시트의 참고 자료로 남깁니다.

## 체크리스트

- 질문이 실제 사용자의 말투로 적혀 있는가?
- 콘텐츠 과제와 출처 과제가 섞이지 않았는가?
- 점수보다 다음 액션이 먼저 보이는가?
- 강의자료로 옮겼을 때 산출물이 명확한가?
- 이북으로 바꿔도 독자가 혼자 따라 할 수 있는가?

## 흔한 질문

**Q. 이 페이지의 내용을 바로 강의자료로 써도 되나요?**

가능합니다. 다만 강의에서는 설명보다 실습 산출물을 먼저 정해야 합니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 현재 가장 중요한 질문군 하나를 고르고, 그 질문에서 보이는 콘텐츠/출처/기술 갭부터 처리합니다.
