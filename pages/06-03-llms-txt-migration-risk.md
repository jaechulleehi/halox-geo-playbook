## llms.txt와 사이트 이전 리스크 관리

llms.txt는 AI 시스템에 중요한 문서 경로를 안내하는 보조 파일입니다. 하지만 사이트 이전, redirect, canonical, sitemap 문제가 정리되지 않으면 llms.txt만으로는 GEO가 해결되지 않습니다.

사이트 개편과 URL 변경은 기존 citation/source를 흔들 수 있습니다. 기술 변경 전후로 같은 질문셋을 다시 측정해야 합니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

사이트 이전이나 URL 변경 뒤 AI 답변에서 오래된 URL이 남을 수 있습니다. llms.txt, redirect, canonical, sitemap, 핵심 문서 경로를 함께 관리해야 이전 리스크를 줄입니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 변경 항목 | 도메인/URL/문서 구조/언어 |
| AI 접근 파일 | robots/sitemap/llms.txt |
| 리다이렉트 | 301/302/누락 |
| 리스크 | 오래된 출처/중복/누락 |
| 대응 | 수정/모니터링/재측정 |

## 제출 템플릿

```text
이전 전 URL / 이전 후 URL / redirect 상태 / llms.txt 반영 여부 / 핵심 페이지 10개 / 재측정 질문
```

## 작성 예시

아래 예시는 `AcmeGEO`라는 가상의 B2B SaaS 브랜드를 기준으로 작성한 샘플입니다. 실제 강의에서는 브랜드명, 질문, 출처 후보, 담당자를 자기 상황에 맞게 바꿉니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 변경 항목 | 블로그 URL 구조 변경 |
| AI 접근 파일 | sitemap은 갱신, llms.txt는 미반영 |
| 리다이렉트 | 구 URL에서 신 URL로 301 처리 |
| 리스크 | AI 답변이 오래된 URL을 citation으로 남길 수 있음 |
| 대응 | 핵심 GEO 글 10개를 llms.txt와 내부 링크 허브에 다시 반영한다 |

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

- 질문이 실제 사용자의 말투로 적혀 있는가?
- 콘텐츠 과제와 출처 과제가 섞이지 않았는가?
- 점수보다 다음 액션이 먼저 보이는가?
- 강의자료로 옮겼을 때 산출물이 명확한가?
- 이북으로 바꿔도 독자가 혼자 따라 할 수 있는가?

## 흔한 질문

**Q. 이 페이지의 내용을 바로 강의자료로 써도 되나요?**

가능합니다. 다만 강의에서는 설명보다 실습 산출물을 먼저 정해야 합니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 현재 가장 중요한 질문군 하나를 고르고, 그 질문에서 보이는 콘텐츠/출처/기술 갭부터 처리합니다.
