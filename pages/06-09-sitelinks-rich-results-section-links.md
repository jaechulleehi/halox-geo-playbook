## SEO 사이트링크와 리치 리절트: 검색결과 섹션 링크

![사이트링크와 리치 리절트](../assets/images/page-heroes/halox-geo-06-09-sitelinks-rich-results-section-links-hero.png)

사이트링크와 리치 리절트는 검색결과를 예쁘게 꾸미는 기능이 아닙니다. 사용자가 검색결과에서 바로 핵심 섹션으로 이동하고, AI와 검색엔진이 페이지 안의 구조를 이해하게 만드는 신호입니다.

GEO 관점에서는 섹션 링크가 질문별 답변 위치를 분명히 해줍니다. 리포트 예시, FAQ, 비교표, 가격/정책, 상품 스펙 같은 섹션이 잘 나뉘어 있으면 AI 답변의 근거 후보가 더 명확해집니다.

## 섹션 구조는 답변 위치를 알려준다

검색결과의 섹션 링크는 마음대로 지정할 수 있는 광고 영역이 아닙니다. 하지만 제목 구조, 내부 링크, 앵커, 본문 품질이 정리되어 있으면 검색엔진이 페이지 안의 중요한 구간을 더 잘 이해할 수 있습니다.

| 구조 요소 | 확인할 것 | GEO에서의 의미 |
|---|---|---|
| H2/H3 heading | 질문과 답변이 보이는가 | AI가 답변 구간을 찾기 쉬움 |
| 목차/앵커 | 섹션 이동이 가능한가 | 긴 가이드의 구조 이해 |
| FAQ | 실제 질문에 답하는가 | AEO/GEO 답변 후보 |
| 표/비교 | 선택 기준이 명확한가 | 비교 질문 대응 |
| 내부 링크 | 다음 행동으로 이어지는가 | 리포트/브리프/문의 연결 |

## 리포트에서 먼저 확인할 기준

프롬프트 분석에서 반복되는 질문을 페이지 섹션 제목으로 바꿔 봅니다. 예를 들어 “GEO 리포트에서 무엇을 봐야 하나”라는 질문이 반복된다면 페이지 안에 `리포트에서 봐야 할 섹션` 같은 명확한 구조가 있어야 합니다.

사이트 진단에서는 heading, 메타, 내부 링크, schema 이슈를 확인합니다. 구조가 약한 페이지는 AI 답변에서 짧게 언급되더라도 citation 후보로 안정적으로 남기 어렵습니다.

![사이트링크와 리치 리절트 구조](../assets/images/body-figures/halox-geo-06-09-sitelinks-rich-results-serp-anatomy-codex-only.png)

*섹션 링크와 리치 리절트는 질문별 답변 위치를 검색결과와 AI 답변에 더 분명하게 보여주는 구조 신호다.*

## 가상 기업 AcmeGEO 예시

AcmeGEO의 긴 가이드 페이지에는 좋은 내용이 많지만 H2가 “개요”, “상세”, “마무리”처럼 되어 있습니다. AI 답변은 페이지를 인용하지 않고 외부 글의 FAQ를 citation으로 보여줍니다.

수정은 heading을 실제 질문으로 바꾸는 것입니다. `AVI는 무엇을 뜻하나`, `인용률은 어떻게 해석하나`, `다음 액션은 어떻게 정하나`처럼 섹션을 나누고, 내부 링크로 리포트 예시와 콘텐츠 브리프를 연결합니다.

## H2 수정 전/후 예시

| 수정 전 H2 | 수정 후 H2 |
|---|---|
| 개요 | GEO 리포트에는 어떤 지표가 들어가야 하나요? |
| 상세 | mention/source/citation은 어떻게 다른가요? |
| 마무리 | 리포트를 읽은 뒤 어떤 콘텐츠를 고쳐야 하나요? |

섹션 제목은 문서 작성자용 목차가 아니라 독자가 실제로 묻는 질문에 가까워야 합니다.

## 정리 양식

```text
점검 URL:
반복 질문:
현재 H2/H3 구조:
바꿀 섹션 제목:
FAQ/표 보강 항목:
내부 링크 대상:
검증할 rich result/schema:
재측정 질문:
```
