## 커머스와 플랫폼은 GEO보다 AIO를 먼저 봐야 할까

커머스와 플랫폼 서비스는 웹 문서 GEO만 보면 부족합니다. 상품 데이터, 리뷰, 상세페이지, Google AI Overviews, 쇼핑 검색, 플랫폼 추천 레이어까지 함께 봐야 합니다.

저관여 상품은 사용자가 AI에게 직접 묻지 않을 수 있습니다. 대신 사용 상황, 레시피, 추천, 비교 질문에서 노출 기회가 생깁니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

커머스와 플랫폼은 브랜드 추천보다 상품/카테고리/리뷰/재고/가격 문맥이 더 강할 수 있습니다. GEO와 AIO를 나눠 보고 어떤 질문이 매출과 연결되는지 확인합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 질문 유형 | 상품 추천/카테고리 비교/리뷰/가격/지역 |
| 답변 단위 | 브랜드/상품/카테고리/판매처 |
| 필요 데이터 | 가격/재고/리뷰/스펙 |
| AI 노출 리스크 | 잘못된 정보/오래된 정보/경쟁사 추천 |
| 개선안 | 구조화/리뷰/FAQ/피드 |

## 제출 템플릿

```text
카테고리 / 상품군 / AI 질문 20개 / 노출되어야 할 데이터 / 위험 정보 / 수정 요청
```

## 작성 예시

아래 예시는 `AcmeGEO`라는 가상의 B2B SaaS 브랜드를 기준으로 작성한 샘플입니다. 실제 강의에서는 브랜드명, 질문, 출처 후보, 담당자를 자기 상황에 맞게 바꿉니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 질문 유형 | 상품 추천 |
| 답변 단위 | 상품군과 판매처 |
| 필요 데이터 | 가격, 재고, 리뷰 수, 배송 조건 |
| AI 노출 리스크 | 단종 상품이나 품절 상품이 추천될 수 있음 |
| 개선안 | 상품 schema, 재고 피드, 카테고리 FAQ를 정리한다 |

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

## 참고 링크 패키지

커머스와 플랫폼에서는 상품 정보가 AI가 읽을 수 있는 구조인지가 중요합니다. 정보 구조 예시는 HaloX의 [AI 검색이 선택하는 콘텐츠 구조](https://haloxlabs.ai/ko/blog/ai-preferred-content-structure)를 함께 보면서, 이 페이지의 AIO/GEO 체크 항목과 연결해 확인합니다.

## 체크리스트

- 질문이 실제 사용자의 말투로 적혀 있는가?
- 콘텐츠 과제와 출처 과제가 섞이지 않았는가?
- 점수보다 다음 액션이 먼저 보이는가?
- 강의자료로 옮겼을 때 산출물이 명확한가?
- 이북으로 바꿔도 독자가 혼자 따라 할 수 있는가?

## 흔한 질문

**Q. 이 페이지의 내용을 바로 강의자료로 써도 되나요?**

가능합니다. 다만 강의에서는 설명보다 실습 산출물을 먼저 정해야 합니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 현재 가장 중요한 질문군 하나를 고르고, 그 질문에서 보이는 콘텐츠/출처/기술 갭부터 처리합니다.
