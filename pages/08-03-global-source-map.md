## 글로벌 source map을 만드는 법

글로벌 source map은 영어권에서 AI가 참고할 가능성이 높은 디렉터리, 리뷰, 커뮤니티, 위키, 문서 채널을 질문군별로 정리하는 작업입니다.

한국어 자산만 보면 글로벌 추천 문맥을 놓칩니다. Reddit, 리뷰 사이트, 앱스토어, 개발자 문서 같은 공개 출처를 함께 봐야 합니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 타깃 시장에서 AI가 신뢰할 만한 출처 유형을 나눕니다.
2. 미디어, 리뷰, 파트너, 커뮤니티, 공식 문서를 구분합니다.
3. 현재 브랜드가 등장하는 출처와 빠진 출처를 표시합니다.
4. 확보 가능한 source와 장기 과제를 나눕니다.
5. 글로벌 질문셋과 source map을 함께 업데이트합니다.

## 사례로 이해하기

국내 출처만 있으면 글로벌 AI 답변에서 브랜드 신뢰도가 약합니다. 영어권 미디어, 디렉토리, 리뷰, 커뮤니티, 파트너 페이지를 별도로 source map으로 관리합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 글로벌 출처 후보 | 미디어/디렉토리/리뷰/커뮤니티/파트너 |
| 지역/언어 | US/Global/English 등 |
| 획득 난이도 | 낮음/중간/높음 |
| 연결 질문 | 영어 AI 질문 |
| 액션 | 등록/기고/파트너/PR |

## 제출 템플릿

```text
영문 질문 / 필요한 출처 유형 / 후보 URL / 획득 방식 / 담당자 / 상태 / 다음 연락일
```

## 참고 링크 패키지

글로벌 source map은 단순 링크 목록이 아니라 여러 출처가 브랜드를 같은 방식으로 설명하는지 보는 작업입니다. 이 기준은 HaloX의 [GEO 평판 관리와 브랜드 합의 신호](https://haloxlabs.ai/ko/blog/geo-reputation-brand-consensus)를 함께 읽으면 더 선명해집니다.
