## GEO 대행사는 무엇을 증명해야 하나

좋은 GEO 대행사는 질문 설계, 측정 프레임, source/citation 해석, 콘텐츠 액션, 기술 점검, 재측정 계획을 설명해야 합니다. 단순 언급 횟수 리포트만으로는 부족합니다.

초기 GEO 시장에는 기존 SEO 패키지를 이름만 바꾼 제안도 많습니다. 실행 기준을 물어봐야 차이가 드러납니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

GEO 대행사를 고를 때 `AI에 잘 나오게 해드립니다`라는 말만으로는 부족합니다. 질문셋, 측정 방식, source/citation 구분, 실험 설계, 리포트 기준을 요구해야 합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 검증 항목 | 질문셋/측정/콘텐츠/출처/기술/리포트 |
| 대행사 답변 | 제안서에서 확인한 내용 |
| 증거 | 샘플 리포트/방법론/사례 |
| 위험 신호 | 측정 불명확/링크만 강조/과장 |
| 판정 | 통과/보류/탈락 |

## 제출 템플릿

```text
대행사명 / 제안 범위 / 질문셋 제공 여부 / 지표 정의 / 샘플 리포트 / 위험 신호 / 최종 판단
```

## 작성 예시

아래 예시는 `AcmeGEO`라는 가상의 B2B SaaS 브랜드를 기준으로 작성한 샘플입니다. 실제 강의에서는 브랜드명, 질문, 출처 후보, 담당자를 자기 상황에 맞게 바꿉니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 검증 항목 | 질문셋과 측정 방식 |
| 대행사 답변 | 월 1회 AI visibility 리포트 제공 |
| 증거 | 샘플 리포트에 source/citation 구분이 없음 |
| 위험 신호 | 백링크 수만 강조하고 질문 시장 설명이 부족 |
| 판정 | 보류, 질문셋과 지표 정의를 다시 요청한다 |

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.
