## GEO 도구와 리포트는 어떻게 검증할까

GEO 도구는 점수보다 질문별 실행 액션을 보여줘야 합니다. 어떤 모델/질문/조건에서 측정했는지, source/citation/co-mention을 어떻게 분리하는지 확인해야 합니다.

점수 화면만 있으면 무엇을 고칠지 알 수 없습니다. 좋은 리포트는 다음 콘텐츠/출처/기술 과제로 이어집니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

GEO 도구는 예쁜 점수보다 의사결정이 가능해야 합니다. 어떤 질문에서 빠졌는지, 어떤 출처가 쓰였는지, 다음에 무엇을 고칠지 보여주는 리포트가 필요합니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| 리포트 항목 | 질문/mention/source/citation/경쟁사/액션 |
| 제공 여부 | 있음/없음/수동 필요 |
| 해석 가능성 | 팀이 다음 액션을 알 수 있는가 |
| 한계 | 모델/지역/언어/샘플 수 |
| 판정 | 도입/보류/보조 |

## 제출 템플릿

```text
도구명 / 측정 모델 / 질문 수 / 지표 정의 / 리포트 샘플 평가 / 액션 도출 가능성 / 도입 판단
```

## 작성 예시

아래 예시는 `AcmeGEO`라는 가상의 B2B SaaS 브랜드를 기준으로 작성한 샘플입니다. 실제 강의에서는 브랜드명, 질문, 출처 후보, 담당자를 자기 상황에 맞게 바꿉니다.

| 입력 항목 | 작성 예시 |
|---|---|
| 리포트 항목 | mention/source/citation/경쟁사/액션 |
| 제공 여부 | mention과 경쟁사는 제공, citation 추적은 제한적 |
| 해석 가능성 | 어떤 콘텐츠를 고쳐야 하는지는 추가 분석 필요 |
| 한계 | 모델과 지역 샘플 수가 적음 |
| 판정 | 보조 도구로 사용하고 자체 질문셋으로 보완한다 |

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

## 참고 링크 패키지

이 실습은 HaloX의 [AVI 점수 가이드](https://haloxlabs.ai/ko/blog/avi-score-explained)와 함께 보면 좋습니다. 함께 볼 링크를 하나만 고르지 말고, 내부 WikiDocs 흐름과 HaloX 심화 글을 같이 연결해 강의자료/워크시트의 참고 자료로 남깁니다.

## 체크리스트

- 질문이 실제 사용자의 말투로 적혀 있는가?
- 콘텐츠 과제와 출처 과제가 섞이지 않았는가?
- 점수보다 다음 액션이 먼저 보이는가?
- 강의자료로 옮겼을 때 산출물이 명확한가?
- 이북으로 바꿔도 독자가 혼자 따라 할 수 있는가?

## 흔한 질문

**Q. 이 페이지의 내용을 바로 강의자료로 써도 되나요?**

가능합니다. 다만 강의에서는 설명보다 실습 산출물을 먼저 정해야 합니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 현재 가장 중요한 질문군 하나를 고르고, 그 질문에서 보이는 콘텐츠/출처/기술 갭부터 처리합니다.
