## GEO 도구 비교: AI 검색 리포트는 어떻게 검증할까

GEO 도구 비교에서 가장 중요한 기준은 대시보드가 예쁜지가 아니라 AI 검색 리포트가 실행으로 이어지는지입니다. 어떤 모델/질문/조건에서 측정했는지, 브랜드 언급률, 답변 근거(source), 화면 인용(citation)을 어떻게 분리하는지, 다음에 어떤 콘텐츠와 출처를 고쳐야 하는지 확인해야 합니다.

점수 화면만 있으면 팀은 움직이지 못합니다. 좋은 리포트는 “우리가 왜 빠졌는가”와 “무엇부터 고칠 것인가”를 같은 화면에서 설명합니다.

## 좋은 GEO 리포트의 기본 구조

| 리포트 요소 | 확인할 질문 | 좋은 산출물 |
|---|---|---|
| 질문셋 | 어떤 질문을 측정했나 | 브랜드/산업/구매 단계별 질문표 |
| 모델/조건 | 어떤 AI 환경에서 측정했나 | 모델/언어/지역/날짜/반복 횟수 |
| mention | 브랜드가 언급되었나 | 언급 여부와 문맥 |
| 답변 근거 | AI가 참고한 출처는 무엇인가 | 도메인/페이지/채널 분류 |
| citation | 실제 링크로 인용되었나 | URL 단위 인용 목록 |
| competitor | 경쟁사는 왜 나오는가 | 경쟁 브랜드/출처/콘텐츠 비교 |
| action | 무엇을 고칠 것인가 | 콘텐츠/오프사이트/기술 액션 |

## 점수보다 먼저 봐야 할 것

AVI나 visibility 점수는 유용한 요약 지표가 될 수 있습니다. 하지만 점수만으로는 액션이 나오지 않습니다. 점수가 낮다면 어느 질문군에서 낮은지, 경쟁사는 어떤 source를 통해 등장하는지, 우리 사이트가 인용되지 않는 이유가 콘텐츠인지 기술인지 오프사이트인지 분해해야 합니다.

그래서 리포트는 항상 세 층으로 읽습니다.

1. **상태**: 지금 AI 답변에 보이는가?
2. **원인**: 왜 보이거나 빠지는가?
3. **액션**: 다음 측정 전 무엇을 바꿀 것인가?

## 사례로 이해하기

엔터프라이즈 뉴스룸은 보도자료와 블로그를 많이 운영하지만 AI 답변에서는 경쟁사나 제3자 매체가 더 자주 인용될 수 있습니다. 이때 리포트가 단순히 “노출 낮음”이라고만 말하면 팀은 무엇을 고쳐야 할지 모릅니다.

좋은 리포트는 질문별로 답변 근거(source)와 화면 인용(citation)을 분리합니다. 예를 들어 특정 질문에서는 자사 뉴스룸이 답변 근거 후보로 잡히지만 화면 인용은 외부 기사로 빠질 수 있습니다. 이 경우 자사 페이지의 answer-first 구조, schema, 내부 링크, 최신성, 외부 출처와의 일관성을 함께 확인해야 합니다.

## GEO 도구 비교 체크리스트

| 검증 항목 | 통과 기준 | 보류 기준 |
|---|---|---|
| 질문셋 관리 | 질문 추가/분류/버전 관리 가능 | 고정 질문만 제공 |
| 반복 측정 | 날짜별 변화 비교 가능 | 단발성 캡처만 제공 |
| 답변 근거(source)/화면 인용(citation) | 출처와 인용 URL 분리 | 둘을 하나로 표시 |
| 경쟁사 비교 | 경쟁사가 나온 이유를 추적 | 단순 순위표만 제공 |
| 액션 연결 | 콘텐츠/기술/오프사이트 과제로 연결 | 점수 해석만 제공 |
| export | 팀 공유용 리포트 추출 가능 | 화면 캡처 의존 |

## 도구 비교 점수표 예시

여러 GEO 도구를 볼 때는 기능 유무만 보지 말고, 우리 팀이 다음 액션을 뽑을 수 있는지까지 점수화합니다.

| 평가 항목 | 가중치 | 도구 A | 도구 B | 확인 메모 |
|---|---:|---:|---:|---|
| 질문셋을 직접 관리할 수 있는가 | 25 | 4 | 2 | 산업/구매 단계별 질문을 바꿀 수 있는지 확인 |
| mention/source/citation을 분리하는가 | 25 | 3 | 4 | source와 citation을 같은 지표로 뭉개면 감점 |
| 경쟁사와 출처 차이를 보여주는가 | 20 | 3 | 2 | 왜 경쟁사가 나오는지 설명해야 함 |
| 콘텐츠/기술/오프사이트 액션으로 이어지는가 | 20 | 2 | 3 | 리포트가 업무 티켓으로 바뀌는지 확인 |
| 반복 측정과 export가 가능한가 | 10 | 4 | 3 | 월간 보고와 재측정에 필요 |

이 점수표는 구매 결정을 대신하지 않습니다. 다만 `예쁜 대시보드`와 `실행 가능한 AI 검색 리포트`를 구분하게 해줍니다.

## 실습 정리표

| 입력 항목 | 작성 기준 |
|---|---|
| 리포트 항목 | 질문/mention, 답변 근거(source), 화면 인용(citation), 경쟁사/액션 |
| 제공 여부 | 있음/없음/수동 필요 |
| 해석 가능성 | 팀이 다음 액션을 알 수 있는가 |
| 한계 | 모델/지역/언어/샘플 수/재현성 |
| 판정 | 도입/보류/보조 도구 |

## 정리 양식

```text
도구명:
측정 모델/조건:
질문 수:
지표 정의:
답변 근거(source)와 화면 인용(citation) 분리 여부:
경쟁사 비교 방식:
액션 도출 가능성:
도입 판단:
```

## 작성 예시

| 입력 항목 | 작성 예시 |
|---|---|
| 리포트 항목 | mention, 답변 근거(source), 화면 인용(citation), 경쟁사/액션 |
| 제공 여부 | mention과 경쟁사는 제공, citation 추적은 제한적 |
| 해석 가능성 | 어떤 콘텐츠를 고쳐야 하는지는 추가 분석 필요 |
| 한계 | 모델과 지역 샘플 수가 적음 |
| 판정 | 보조 도구로 사용하고 자체 질문셋으로 보완한다 |

## 완료 기준

- 리포트 점수와 다음 액션이 연결되어 있습니다.
- 질문셋, 답변 근거(source)/화면 인용(citation), 경쟁사 비교, 재측정 기준이 한 흐름으로 보입니다.
- 도구의 강점과 보완해야 할 수동 분석 영역이 분리되어 있습니다.

## 흔한 질문

**Q. 점수가 높으면 GEO가 잘 되고 있다는 뜻인가요?**

부분적으로만 맞습니다. 점수가 높아도 중요한 구매 질문에서 빠지거나, citation이 경쟁사나 외부 매체로만 잡히면 실행 과제가 남아 있습니다.

**Q. 여러 AI 모델을 모두 측정해야 하나요?**

처음부터 모두 볼 필요는 없습니다. 핵심 구매 질문과 주요 모델을 정하고, 같은 조건으로 반복 측정하는 것이 먼저입니다.
