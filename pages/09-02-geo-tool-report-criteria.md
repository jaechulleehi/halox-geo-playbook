## mention/source/citation 지표 해석

![mention source citation 리포트 해석](../assets/images/page-heroes/halox-geo-09-02-geo-tool-report-criteria-hero.png)

mention/source/citation은 한 점수로 합치기보다 조합으로 읽어야 합니다. mention은 브랜드가 등장했는지, source는 답변 근거가 되는지, citation은 화면 링크로 보이는지를 뜻합니다.

세 지표의 조합에 따라 액션이 달라집니다. mention은 높은데 citation이 낮으면 URL 구조나 답변 단위가 약할 수 있고, citation은 있는데 추천 문맥이 약하면 차별 기준이 부족할 수 있습니다.

`AcmeGEO`라는 이름은 설명을 위한 가상 기업명이며, 실제 고객 사례가 아닙니다.

## 먼저 볼 기준

| 기준 | 읽는 법 |
|---|---|
| mention | 비브랜드 질문에서 자연스럽게 등장하는지 본다 |
| source | 답변 문장의 근거로 쓰일 만한지 본다 |
| citation | 사용자가 확인할 URL로 보이는지 본다 |

## 실행 흐름

1. 도구가 어떤 모델, 기간, 질문셋으로 측정했는지 확인한다.
2. mention, source, citation이 분리되어 나오는지 본다.
3. 경쟁사가 왜 등장했는지 URL과 문맥으로 설명되는지 확인한다.
4. 점수 화면이 콘텐츠/기술/출처 실행 과제로 이어지는지 본다.
5. 보고서 export가 고객 회의 문장으로 바로 쓰이는지 검토한다.

## 지표 해석 예시

AcmeGEO가 답변에는 자주 나오지만 공식 URL citation이 거의 없다면 브랜드 인지도보다 citation 후보 페이지가 약한 상태입니다. 리포트 예시, 비교 기준, 내부 링크를 먼저 고칩니다.

## 작성 예시와 완료 기준

| 리포트 문장 | 판단 |
|---|---|
| “브랜드 언급률 60%” | 부족함. 어떤 질문에서 언급됐는지 알 수 없음 |
| “추천형 질문 30개 중 18개에서 언급, 공식 URL citation은 4개” | 좋음. 언급과 citation을 분리해 다음 액션을 정할 수 있음 |
| “경쟁사 블로그가 source로 12회 반복” | 좋음. 외부 출처 보강이나 공식 비교 페이지 제작으로 이어질 수 있음 |

완료 기준은 지표를 예쁘게 보여주는 것이 아니라, 리포트만 보고도 콘텐츠/기술/source 액션을 분리할 수 있는 상태입니다.
