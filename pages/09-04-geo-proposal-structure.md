## GEO 제안서 비교: 과장된 약속은 어떻게 걸러낼까

![GEO 제안서 과장 필터와 검증 구조](../assets/images/page-heroes/halox-geo-09-04-geo-proposal-structure-hero.png)

GEO 제안서 비교에서 핵심은 과장된 약속을 걸러내고 현재 문제, 실행 범위, 재측정 기준을 확인하는 것입니다. 독자는 제안서를 읽을 때 내 브랜드의 AI 답변 상태, 놓친 질문 시장, 답변 근거(source)/화면 인용(citation) 부족, 콘텐츠/기술/오프사이트 액션이 한 흐름으로 설명되는지 봐야 합니다.

제안서는 첫 리포트처럼 읽어야 합니다. 제안서에서 질문과 지표가 모호하면 실제 운영 리포트도 모호해질 가능성이 높습니다. 이 페이지가 9장에 들어가는 이유도 외주를 권하기 위해서가 아니라, 독자가 남의 제안서를 GEO 실행 검증 기준으로 읽을 수 있게 하기 위해서입니다.

## 제안서를 읽을 때 확인할 기본 목차

| 제안서 섹션 | 들어갈 내용 | 주의할 점 |
|---|---|---|
| 1. 현재 문제 | AI 답변에서 보이지 않거나 잘못 설명되는 질문 | “AI 시대라 필요” 같은 일반론으로 시작하지 않기 |
| 2. 질문 시장 | 구매/비교/문제해결/브랜드 질문셋 | 키워드 목록만 붙이지 않기 |
| 3. 기준선 | mention, 답변 근거(source), 화면 인용(citation), 경쟁사 상태 | 단일 점수로 끝내지 않기 |
| 4. 실행 전략 | 콘텐츠/오프사이트/테크니컬 액션 | 실행 담당과 순서 명확히 하기 |
| 5. 리포트 방식 | 월간/분기 재측정과 의사결정 기준 | 예쁜 대시보드만 보여주지 않기 |
| 6. 일정/범위 | 4주/8주/월간 운영 계획 | 우리 팀이 해야 할 일도 분리하기 |

## 제안서 한 장 요약 예시

```text
문제 상황: AI 답변에서 브랜드는 언급되지만 공식 페이지가 citation으로 잡히지 않는다.
핵심 질문군: 구매 비교 / 대안 탐색 / 가격/기능 / 리스크 검토
현재 상태: mention 있음, citation 약함, 경쟁사 source 우세
우선 액션: answer-first 페이지 3개 리라이트, FAQ/schema 보강, 외부 답변 근거 후보 5개 정리
측정 기준: 같은 질문셋 40개로 4주 후 mention, 답변 근거(source), 화면 인용(citation) 변화 비교
운영 산출물: 기준선 리포트, 콘텐츠 수정안, 답변 근거 맵, 월간 리포트
```

## 사례로 이해하기

커머스/플랫폼형 고객은 상품 페이지가 많아도 AI 답변에서는 카테고리 설명, 비교표, 리뷰 출처, 가격 정책, 배송/반품 정보가 제대로 연결되지 않을 수 있습니다. 이때 제안서는 “상품 페이지를 더 많이 만들자”가 아니라 질문군별로 어떤 정보 단위가 부족한지 보여줘야 합니다.

PR/뉴스룸형 조직은 보도자료가 많아도 AI가 공식 뉴스룸보다 제3자 기사나 오래된 소개 페이지를 더 신뢰할 수 있습니다. 이 경우 제안서는 뉴스룸 구조, 엔티티 일관성, 답변 근거 후보 채널, 평판 리스크 질문군을 분리해서 보여줘야 합니다.

|---|---|

## 좋은 제안서가 포함해야 할 검증 조건

GEO 제안서는 성과를 약속하기보다 검증 가능한 조건을 명시해야 합니다. 특히 AI 답변은 플랫폼별 변동성이 있으므로 측정 조건과 비교 기준이 없으면 제안의 신뢰도가 낮습니다.

| 검증 조건 | 확인할 질문 |
|---|---|
| 질문셋 버전 | 어떤 질문을 기준으로 전후 비교하는가? |
| 플랫폼 범위 | ChatGPT/Perplexity/AI Overviews/Gemini 중 무엇을 보는가? |
| 반복 횟수 | 단발 결과와 반복 패턴을 구분하는가? |
| 경쟁사 기준 | 어떤 경쟁사와 co-mention을 비교하는가? |
| 실행 범위 | 콘텐츠/source/기술/PR 중 어디까지 포함하는가? |
| 재측정일 | 몇 주 뒤 같은 조건으로 다시 보는가? |

AcmeGEO가 제안서를 받는다면 “AI 답변 1위 보장”보다 “추천형 질문 40개를 같은 조건으로 4주 후 재측정하고, citation 후보 URL과 source map 변화를 비교한다”는 제안이 더 신뢰할 만합니다.

## 과장된 약속 체크리스트

제안서 문장이 좋아 보여도 아래 항목에 걸리면 다시 질문해야 합니다. 좋은 제안서는 `성과 보장`보다 `측정 조건`, `실행 범위`, `재측정 기준`을 분명히 합니다.

| 제안서 문구 | 위험한 이유 | 다시 물어볼 질문 |
|---|---|---|
| AI 검색 상위 노출 보장 | AI 답변은 모델/질문/시점에 따라 달라짐 | 어떤 질문셋과 플랫폼 기준인가? |
| 모든 AI에서 브랜드 노출 가능 | 플랫폼별 citation 구조가 다름 | ChatGPT/Perplexity/AI Overviews를 어떻게 나누나? |
| 월 20개 글 발행 | 글 수와 AI 인용은 직접 비례하지 않음 | 어떤 질문군과 출처 갭을 해결하나? |
| 백링크 대량 확보 | 품질 낮은 링크는 리스크가 될 수 있음 | 어떤 source 후보를 왜 확보하나? |
| 대시보드 제공 | 화면만 있고 해석이 없을 수 있음 | 다음 액션과 담당까지 나오나? |
| 경쟁사보다 빠르게 개선 | 기준선과 비교 조건이 없으면 검증 불가 | 전후 비교 기준과 재측정 날짜는? |

## 흔한 질문

**Q. 제안서에서 가격을 먼저 봐도 되나요?**

가격보다 범위가 먼저입니다. 질문 수, 리포트 깊이, 실행 범위, 재측정 주기가 정해져야 가격 근거가 생깁니다.

**Q. 제품 기능 화면을 많이 넣는 게 좋나요?**

화면은 필요하지만 중심은 아닙니다. 독자가 어떤 결정을 하게 되는지 설명하는 화면만 넣는 편이 좋습니다.

## 과장된 약속을 거르는 질문

GEO 제안서를 볼 때 아래 질문에 답하지 못하면 실행 리스크가 큽니다.

| 질문 | 왜 필요한가 |
|---|---|
| 어떤 질문셋을 기준으로 측정하나? | 측정 범위가 없으면 성과도 모호합니다 |
| 현재 기준선은 어떻게 남기나? | 개선 전 상태가 있어야 변화가 보입니다 |
| mention/source/citation을 분리하나? | 단순 언급과 실제 인용은 다릅니다 |
| 콘텐츠와 기술 중 무엇을 먼저 고치나? | 실행 우선순위가 있어야 합니다 |
| 외부 출처는 어떻게 확보하나? | 자기 사이트만으로 해결되지 않는 질문이 있습니다 |
| 한 달 뒤 무엇을 다시 보나? | 재측정 없는 제안은 운영으로 이어지기 어렵습니다 |

좋은 제안서는 장밋빛 결과보다 측정 범위, 한계, 실행 순서를 더 분명하게 말합니다.
