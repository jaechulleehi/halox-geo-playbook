## 2주차: Fan-out 질문맵과 콘텐츠 갭

2주차는 대표 질문을 하위 질문으로 확장하고, 질문별 콘텐츠 갭을 찾는 실습입니다. 신규 글, 리라이트, 오프사이트 과제를 분리합니다.

질문이 많아지는 것이 목표가 아닙니다. 사업적으로 중요한 질문에서 답변 재료가 있는지 확인하는 것이 목표입니다.

## 실무에서 먼저 볼 것

| 점검 항목 | 확인 질문 | 다음 액션 |
|---|---|---|
| 질문 | 이 페이지가 답하는 핵심 질문은 무엇인가? | 질문을 한 문장으로 다시 쓴다 |
| 자산 | 이미 답할 콘텐츠나 페이지가 있는가? | 신규/리라이트/보강으로 나눈다 |
| 출처 | AI가 참고할 근거가 충분한가? | 내부 근거와 외부 근거를 분리한다 |
| 측정 | 어떤 지표로 변화를 볼 것인가? | mention/source/citation/co-mention 중 핵심 지표를 고른다 |

## 실행 순서

1. 대표 질문을 하나 고릅니다.
2. 그 질문에 답하기 위해 필요한 정보 단위를 적습니다.
3. 현재 보유한 콘텐츠, 제품 페이지, 외부 출처를 연결합니다.
4. 부족한 부분을 신규 작성, 리라이트, 오프사이트 보강, 기술 점검으로 분류합니다.
5. 같은 질문셋으로 다시 측정할 시점을 정합니다.

## 사례로 이해하기

2주차는 baseline에서 빠진 질문을 Fan-out으로 확장하고 콘텐츠 갭으로 바꾸는 시간입니다. 좋은 질문맵은 다음 주차 리라이트 과제를 자동으로 만들어줍니다.

## 실습 워크시트

| 입력 항목 | 작성 기준 |
|---|---|
| Seed 질문 | 1주차에서 고른 핵심 질문 |
| 확장 질문 | Fan-out 결과 |
| 현재 콘텐츠 | 답할 수 있는 자산 |
| 갭 | 부족한 근거/구조/출처 |
| 리라이트 후보 | 다음 주차 작업 페이지 |

## 제출 템플릿

```text
Seed 질문 5개 / 확장 질문 50개 / 콘텐츠 갭 10개 / 우선 리라이트 3개 / 필요한 출처 후보
```

## 작성 예시

아래 예시는 `AcmeGEO`라는 가상의 B2B SaaS 브랜드를 기준으로 작성한 샘플입니다. 실제 강의에서는 브랜드명, 질문, 출처 후보, 담당자를 자기 상황에 맞게 바꿉니다.

| 입력 항목 | 작성 예시 |
|---|---|
| Seed 질문 | GEO 도구 추천 |
| 확장 질문 | 가격 비교 / citation 측정 / AI Overview 대응 / B2B SaaS 도입 기준 |
| 현재 콘텐츠 | GEO 정의 글과 AVI 점수 글 |
| 갭 | 도구 비교표와 샘플 리포트 부족 |
| 리라이트 후보 | GEO 도구 검증 기준 글 |

## 완료 기준

- 작성한 표만 봐도 다음 액션이 보입니다.
- 질문, 콘텐츠, 출처, 측정 기준이 서로 연결되어 있습니다.
- 강의 후 제출물로 쓰거나 이북 워크시트로 옮겨도 설명이 부족하지 않습니다.

## 참고 링크 패키지

이 실습은 HaloX의 [SEO/GEO 키워드 전략 프레임워크](https://haloxlabs.ai/ko/blog/seo-geo-keyword-strategy-framework)와 함께 보면 좋습니다. 함께 볼 링크를 하나만 고르지 말고, 내부 WikiDocs 흐름과 HaloX 심화 글을 같이 연결해 강의자료/워크시트의 참고 자료로 남깁니다.

## 체크리스트

- 질문이 실제 사용자의 말투로 적혀 있는가?
- 콘텐츠 과제와 출처 과제가 섞이지 않았는가?
- 점수보다 다음 액션이 먼저 보이는가?
- 강의자료로 옮겼을 때 산출물이 명확한가?
- 이북으로 바꿔도 독자가 혼자 따라 할 수 있는가?

## 흔한 질문

**Q. 이 페이지의 내용을 바로 강의자료로 써도 되나요?**

가능합니다. 다만 강의에서는 설명보다 실습 산출물을 먼저 정해야 합니다.

**Q. 모든 항목을 한 번에 실행해야 하나요?**

아닙니다. 현재 가장 중요한 질문군 하나를 고르고, 그 질문에서 보이는 콘텐츠/출처/기술 갭부터 처리합니다.
