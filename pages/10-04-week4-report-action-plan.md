## 4주차: GEO 실행 리포트와 30일 액션 플랜

4주차는 기술 점검, 리포트 해석, 30일 실행 계획을 하나로 묶는 마무리입니다. 산출물은 점수 보고서가 아니라 팀이 바로 실행할 수 있는 GEO 실행 리포트여야 합니다.

읽고 난 뒤 실행이 멈추면 GEO는 지식으로만 남습니다. 담당, 우선순위, 완료 기준, 재측정 일정을 넣어야 합니다.

## 4주차 목표

| 목표 | 설명 | 산출물 |
|---|---|---|
| 4주 결과 요약 | 기준선, 질문맵, 리라이트 결과를 묶는다 | 5줄 요약 |
| 기술/운영 점검 | robots/sitemap/schema/rendering 등 병목을 확인한다 | 기술 점검표 |
| 우선순위 결정 | 콘텐츠/source/기술 액션을 정렬한다 | 액션 10개 |
| 담당/기한 합의 | 실행 주체와 마감일을 정한다 | 30일 계획 |
| 재측정 설계 | 같은 질문셋으로 다시 볼 날짜를 정한다 | 다음 리포트 기준 |

## 따라 해보는 순서

| 단계 | 할 일 | 확인할 것 |
|---|---|---|
| 1 | 3주차 리라이트 결과 공유 | 이번 단계에서는 실행 리포트로 정리합니다. |
| 2 | 좋은 리포트와 나쁜 리포트 비교 | 점수보다 액션과 담당이 중요함 |
| 3 | 기술/운영 체크 | AI가 발견/읽기/해석/인용할 수 있는지 확인 |
| 4 | 30일 액션 플랜 작성 | 콘텐츠/source/기술을 담당자별로 분리 |
| 5 | 피드백과 우선순위 조정 | 너무 많은 액션을 3~5개로 줄이기 |
| 6 | 재측정 일정 확정 | 다음 리포트 날짜와 질문셋 고정 |

## 실행 리포트 구조

| 섹션 | 들어갈 내용 | 작성 기준 |
|---|---|---|
| 요약 5줄 | 4주 동안 발견한 핵심 변화 | 경영진이 읽어도 이해 가능해야 함 |
| 기준선 | 1주차 측정 결과 | 질문/모델/날짜/지표 포함 |
| 질문맵 | 2주차 Fan-out과 갭 | 우선 질문군 중심으로 압축 |
| 실행 로그 | 3주차 리라이트/source 설계 | 무엇을 고쳤는지 기록 |
| 기술 점검 | 06장 기준 병목 | 개발팀 액션으로 변환 |
| 30일 계획 | 담당/기한/완료 기준 | 실행 가능한 항목만 남김 |
| 재측정 | 같은 질문셋과 날짜 | 다음 리포트의 기준 |

## 실습 정리표

| 입력 항목 | 작성 기준 |
|---|---|
| 발견 | 4주 동안 확인한 문제 |
| 근거 | 질문/답변/출처/기술 증거 |
| 액션 | 콘텐츠/출처/기술/제품 |
| 담당/기한 | 누가 언제까지 |
| 완료 기준 | 무엇이 끝나면 완료인가 |
| 재측정 | 질문셋과 날짜 |

## 정리 양식

```text
요약 5줄:
핵심 지표:
우선 액션 10개:
이번 달 반드시 할 3개:
담당자/기한:
완료 기준:
재측정 질문셋:
다음 리포트 날짜:
```

## 작성 예시

| 입력 항목 | 작성 예시 |
|---|---|
| 발견 | AI 답변에서 브랜드 mention은 낮고 경쟁사 비교 기준도 부족 |
| 근거 | 30개 질문 중 브랜드 언급 4개, citation 0개 |
| 액션 | 비교표 2개 작성, 답변 근거 맵 10개 확보, glossary 5개 보강 |
| 담당/기한 | 콘텐츠 담당 2주, 개발 담당 1주, 외부 출처 담당 4주 |
| 재측정 | 30일 뒤 같은 질문셋으로 ChatGPT/Perplexity/Gemini 재측정 |

## 피드백 기준

| 항목 | 좋은 산출물 | 보완이 필요한 산출물 |
|---|---|---|
| 요약 | 5줄 안에 문제/근거/액션이 보인다 | 점수만 있고 의미가 없다 |
| 액션 | 담당/기한/완료 기준이 있다 | “콘텐츠 보강”처럼 추상적이다 |
| 재측정 | 같은 질문셋과 날짜가 정해져 있다 | 다음 확인 기준이 없다 |

## 월간 리포트로 확장하기

4주차 산출물은 그대로 월간 GEO 리포트의 초안이 됩니다. 다음 달부터는 질문셋 변화, mention, 답변 근거(source), 화면 인용(citation) 변화, 실행 완료 여부, 새 경쟁사/출처 변화를 반복해서 봅니다. 이 흐름은 [09-05. 월간 GEO 리포트로 브랜드 가시성을 관리하는 법](https://wikidocs.net/346398)와 연결됩니다.

## 완료 기준

- 실행 리포트에 요약, 근거, 액션, 담당, 기한, 재측정 기준이 모두 들어 있습니다.
- 30일 안에 실행할 항목이 3~5개로 압축되어 있습니다.
- 다음 리포트 날짜와 같은 질문셋이 정해져 있습니다.

## 흔한 질문

**Q. 30일 액션은 몇 개가 적당한가요?**

10개를 모두 실행하려고 하면 흐려집니다. 리포트에는 전체 후보를 남기되, 이번 달 반드시 할 항목은 3~5개로 줄입니다.

**Q. 리포트는 누가 봐야 하나요?**

콘텐츠팀, PR팀, 개발팀, 의사결정자가 함께 봐야 합니다. GEO 실행은 글만의 문제가 아니라 출처, 기술, 운영 리듬의 문제이기 때문입니다.
