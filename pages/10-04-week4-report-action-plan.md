## 4주차: GEO 실행 리포트와 30일 액션 플랜

![GEO 실행 리포트와 30일 액션 플랜](../assets/images/page-heroes/halox-geo-10-04-week4-report-action-plan-hero.png)

4주차는 기준선, 질문맵, 리라이트, 기술 점검을 하나의 실행 리포트로 묶는 단계입니다. 산출물은 점수 보고서가 아니라 팀이 바로 실행할 수 있는 GEO 운영 문서여야 합니다.

읽고 난 뒤 실행이 멈추면 GEO는 지식으로만 남습니다. 담당, 우선순위, 완료 기준, 재측정 일정을 넣어야 합니다. 특히 콘텐츠팀/PR팀/개발팀/의사결정자가 같은 표를 보고 움직일 수 있어야 합니다.

## 실행 리포트의 역할

GEO 실행 리포트는 “이번 달 성과 보고서”이면서 동시에 “다음 달 작업 지시서”입니다. 둘 중 하나만 있으면 약합니다. 성과만 있으면 실행이 없고, 작업 목록만 있으면 왜 그 작업을 해야 하는지 설명할 수 없습니다.

| 리포트 역할 | 필요한 내용 | 없을 때 생기는 문제 |
|---|---|---|
| 현재 상태 설명 | 질문셋/mention/source/citation/경쟁사 | 감상형 보고가 됨 |
| 원인 진단 | 콘텐츠 갭/source 갭/기술 갭/메시지 갭 | 액션이 막연해짐 |
| 실행 기록 | 수정 URL/출처 확보/기술 수정 요청 | 무엇을 했는지 재측정할 수 없음 |
| 의사결정 | 우선순위/담당/기한/완료 기준 | 팀이 움직이지 않음 |
| 반복 측정 | 다음 질문셋/날짜/비교 기준 | 한 번의 캠페인으로 끝남 |

## 실행 리포트 구조

| 섹션 | 들어갈 내용 | 작성 기준 |
|---|---|---|
| 요약 5줄 | 4주 동안 발견한 핵심 변화 | 경영진이 읽어도 이해 가능해야 함 |
| 기준선 | 1주차 측정 결과 | 질문/모델/날짜/지표 포함 |
| 질문맵 | 2주차 Fan-out과 갭 | 우선 질문군 중심으로 압축 |
| 실행 로그 | 3주차 리라이트/source 설계 | 무엇을 고쳤는지 기록 |
| 기술 점검 | 06장 기준 병목 | 개발팀 액션으로 변환 |
| 30일 계획 | 담당/기한/완료 기준 | 실행 가능한 항목만 남김 |
| 재측정 | 같은 질문셋과 날짜 | 다음 리포트의 기준 |

## 5줄 요약 작성법

리포트 첫 페이지에는 아래 다섯 줄이 있어야 합니다.

```text
1. 이번 달 핵심 결론: __ 질문군에서 우리 브랜드는 __ 상태입니다.
2. 가장 큰 원인: __ 때문에 mention/source/citation이 약합니다.
3. 이번에 실행한 것: __ URL/출처/기술 항목을 수정했습니다.
4. 다음 30일 우선 액션: __ 3가지를 먼저 실행합니다.
5. 재측정 기준: __ 질문셋을 __ 날짜에 다시 확인합니다.
```

좋은 요약은 숫자와 해석을 함께 넣습니다. 예를 들어 “언급률이 낮다”보다 “추천형 질문 10개 중 2개에서만 언급되고, citation은 모두 경쟁사 블로그로 이동한다”가 더 실행에 가깝습니다.

## 액션 우선순위 결정표

4주차에는 후보 액션이 많이 나옵니다. 모두 실행하려고 하면 아무것도 끝나지 않습니다. 아래 표로 우선순위를 정합니다.

| 액션 후보 | 영향도 | 난이도 | 담당 | 완료 기준 | 우선순위 |
|---|---|---|---|---|---|
| 비교형 질문 대응 표 추가 | 높음 | 낮음 | 콘텐츠 | 핵심 페이지 2개에 비교표 반영 | 1 |
| Organization schema 정리 | 중간 | 중간 | 개발 | 공식 사이트/소개 본문과 schema 일치 | 2 |
| 외부 디렉터리 프로필 정렬 | 중간 | 낮음 | PR/운영 | 브랜드명/설명/URL 일관화 | 3 |
| 신규 리서치 리포트 작성 | 높음 | 높음 | 콘텐츠/전략 | 4주 뒤 초안 발행 | 보류 |
| 커뮤니티 후기 확보 | 중간 | 높음 | 운영 | 조작 없이 실제 사용 사례 수집 | 보류 |

우선순위는 영향도만으로 정하지 않습니다. 30일 안에 완료할 수 있고 재측정 가능한 항목을 먼저 고릅니다.

![4주차 리포트와 30일 액션 플랜 보드](../assets/images/body-figures/halox-geo-10-04-week4-report-action-plan-priority-board-codex-only.png)

*4주차 리포트는 결과 요약보다 다음 30일 동안 무엇을 고칠지 우선순위와 담당을 정하는 데 목적이 있다.*


## 기술 점검을 리포트 언어로 바꾸기

기술 점검은 개발팀만 보는 체크리스트로 끝나면 안 됩니다. 리포트에서는 문제/영향/요청/완료 기준으로 바꿔야 합니다.

| 기술 항목 | 리포트 문장 | 담당 요청 | 완료 기준 |
|---|---|---|---|
| robots/sitemap | 핵심 URL은 색인 후보에 들어가야 하지만 sitemap 반영이 불확실합니다 | sitemap 포함 여부와 robots 차단 여부 확인 | Search Console에서 확인 가능 |
| rendering | 주요 답변 문장이 렌더링 전 HTML에서 확인되지 않습니다 | SSR/프리렌더/본문 노출 방식 점검 | 크롤러가 본문 텍스트 확인 |
| schema | FAQ schema가 본문 FAQ와 일부 불일치합니다 | 본문과 schema 항목 일치 | Rich Results Test 오류 없음 |
| canonical | 캠페인 URL과 대표 URL 신호가 충돌합니다 | canonical/redirect 정책 정리 | 대표 URL 하나로 신호 집중 |
| internal link | 핵심 페이지가 허브에서 연결되지 않습니다 | 관련 글/허브에서 crawlable href 추가 | 내부 링크 확인 가능 |

06장의 [테크니컬 GEO 기본 점검표](https://wikidocs.net/346335)와 [Google 공식 도구로 SEO/GEO 기술 상태 점검하기](https://wikidocs.net/346389)를 함께 보면 이 표를 더 정확히 채울 수 있습니다.

## 30일 액션 플랜 구조

| 구분 | 이번 달 할 일 | 담당 | 완료 기준 | 재측정 질문 |
|---|---|---|---|---|
| 콘텐츠 | 비교형 질문 대응 페이지 2개 리라이트 | 콘텐츠 | 첫 문단/표/FAQ 반영 | GEO 도구 추천/비교 질문 |
| source | 외부 프로필/디렉터리 3개 정보 정렬 | PR/운영 | 브랜드명/설명/URL 일치 | 브랜드형/카테고리형 질문 |
| 기술 | Organization/FAQ schema 점검 | 개발 | 본문과 schema 일치/오류 없음 | 브랜드 설명/FAQ 질문 |
| 리포트 | 같은 질문셋으로 재측정 | 전략 | 30개 질문 결과표 업데이트 | 전체 질문셋 |

30일 액션은 3~5개가 적당합니다. 후보 10개를 모두 넣을 수는 있지만, “이번 달 반드시 할 항목”은 줄여야 합니다.

## 회의에서 리포트를 읽는 순서

| 순서 | 질문 | 결정할 것 |
|---|---|---|
| 1 | 이번 달 가장 중요한 변화는 무엇인가? | 문제의 우선순위 |
| 2 | 어떤 질문군에서 빠졌는가? | 타깃 질문군 |
| 3 | 원인은 콘텐츠/source/기술 중 어디인가? | 담당 부서 |
| 4 | 30일 안에 고칠 수 있는 것은 무엇인가? | 실행 항목 3~5개 |
| 5 | 다음 리포트에서 무엇을 비교할 것인가? | 재측정 질문셋/날짜 |

회의 시간이 짧다면 점수표를 길게 설명하지 말고 이 다섯 질문만 봅니다.

## 4주 결과를 월간 운영 백로그로 넘기기

4주 프로젝트가 끝나도 GEO 운영은 끝나지 않습니다. 최종 리포트의 액션은 다음 달 백로그로 넘어가야 합니다.

| 백로그 유형 | 예시 | 담당 |
|---|---|---|
| 콘텐츠 | 비교형 페이지 2개 리라이트 | 콘텐츠/SEO |
| source | 파트너 글 3개, 디렉터리 2개 정리 | PR/브랜드 |
| 기술 | schema 오류, canonical, sitemap 티켓 | 개발/SEO |
| 산업별 | B2B SaaS/커머스/로컬 질문셋 분리 | 마케팅 리드 |
| 글로벌 | 영문 카테고리 페이지와 hreflang 점검 | 글로벌/개발 |
| 리포트 | 다음 달 같은 질문셋으로 재측정 | 운영 담당 |

## 최종 리포트 1페이지 예시

아래 구조를 그대로 복사해 자기 브랜드의 4주 결과를 한 페이지로 정리할 수 있습니다.

```text
이번 달 GEO 결론
- 핵심 질문 30개 중 브랜드 mention: __개
- 답변 근거(source)로 확인된 URL: __개
- 화면 인용(citation)으로 반복 노출된 URL: __개
- 가장 자주 함께 언급된 경쟁사/대안: __
- 가장 큰 갭: 콘텐츠 / source / 기술 / 메시지 중 __

핵심 원인
- 콘텐츠: __ 질문군에서 첫 답변/비교표/FAQ 부족
- 출처: __ 질문군에서 외부 근거 부족
- 기술: __ URL에서 HTML/schema/internal link 점검 필요
- 메시지: 공식 소개와 외부 프로필 문장이 __ 부분에서 다름

이번 달 실행 로그
1. __ URL 첫 문단/표/FAQ 수정
2. __ source 후보 추가 또는 정렬
3. __ 기술 점검 요청 전달

다음 30일 액션
1. __ 콘텐츠 리라이트 / 담당 __ / 완료 기준 __
2. __ 출처 확보 / 담당 __ / 완료 기준 __
3. __ 기술 수정 / 담당 __ / 완료 기준 __
4. __ 재측정 / 날짜 __ / 질문셋 __
```

좋은 리포트는 많은 내용을 담는 문서가 아니라, 다음 30일 동안 무엇을 고칠지 분명하게 만드는 문서입니다.

## GSC/GA4와 함께 보는 최종 확인

4주 리포트는 AI 답변 변화만 보지 않습니다. GSC에서 query/page 노출과 CTR이 움직였는지, GA4에서 수정 URL의 engagement와 conversion event가 개선됐는지 함께 확인해야 합니다. AI 답변에서 citation이 늘어도 방문 이후 행동이 약하면 첫 화면 답변, CTA, 내부 링크를 다시 봐야 합니다.

| 데이터 | 확인 질문 | 다음 액션 |
|---|---|---|
| GSC | 노출/CTR/landing page가 변했는가? | title/meta/본문 의도 재점검 |
| GA4 | engagement/conversion event가 개선됐는가? | CTA/내부 링크/전환 경로 보강 |
| AI 리포트 | mention/source/citation이 바뀌었는가? | source/기술/콘텐츠 병목 재분류 |

## 월간 리포트로 확장하기

4주차 산출물은 그대로 월간 GEO 리포트의 초안이 됩니다. 다음 달부터는 질문셋 변화, mention, 답변 근거(source), 화면 인용(citation) 변화, 실행 완료 여부, 새 경쟁사/출처 변화를 반복해서 봅니다. 이 흐름은 [09-05. GEO 리포트 운영: 브랜드 가시성을 매달 관리하는 법](https://wikidocs.net/346398)와 연결됩니다.

반복 리포트에서는 모든 질문을 매번 바꾸지 않습니다. 핵심 질문셋은 유지하고, 새 캠페인/제품/경쟁 상황이 생겼을 때만 일부 질문을 추가합니다. 그래야 지난달과 이번 달을 비교할 수 있습니다.

## 완료 기준

- 실행 리포트에 요약, 근거, 액션, 담당, 기한, 재측정 기준이 모두 들어 있습니다.
- 30일 안에 실행할 항목이 3~5개로 압축되어 있습니다.
- 기술 점검 항목이 문제/영향/요청/완료 기준으로 바뀌어 있습니다.
- 다음 리포트 날짜와 같은 질문셋이 정해져 있습니다.
- 리포트가 콘텐츠팀/PR팀/개발팀/의사결정자에게 각각 필요한 행동을 보여줍니다.

## 흔한 질문

**Q. 30일 액션은 몇 개가 적당한가요?**

10개를 모두 실행하려고 하면 흐려집니다. 리포트에는 전체 후보를 남기되, 이번 달 반드시 할 항목은 3~5개로 줄입니다.

**Q. 리포트는 누가 봐야 하나요?**

콘텐츠팀, PR팀, 개발팀, 의사결정자가 함께 봐야 합니다. GEO 실행은 글만의 문제가 아니라 출처, 기술, 운영 리듬의 문제이기 때문입니다.

**Q. 수치가 나빠도 공유해야 하나요?**
