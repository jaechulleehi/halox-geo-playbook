## 커머스 GEO 리스크와 2주 점검 체크리스트

![커머스 GEO 리스크와 2주 점검 체크리스트](../assets/images/page-heroes/halox-geo-11-04-commerce-geo-risk-checklist-hero.png)

커머스 GEO의 목적은 AI에게 무조건 더 많이 노출되는 것이 아닙니다. AI가 상품을 잘못 이해하지 않게 만드는 것입니다. 가격, 재고, 배송, 반품, 결제 조건이 틀리게 전달되면 노출은 오히려 리스크가 됩니다.

이 페이지에서는 커머스 GEO에서 자주 생기는 리스크와 2주 동안 실행할 수 있는 점검 순서를 정리합니다.

## 커머스 GEO에서 특히 위험한 오류

| 리스크 | 어떤 문제가 생기는가 | 먼저 볼 지점 |
|---|---|---|
| 오래된 가격 | AI가 실제보다 낮거나 높은 가격을 말합니다 | 페이지, feed, schema, 광고 소재 가격 |
| 품절 상품 추천 | 사용자가 살 수 없는 상품이 후보로 나옵니다 | 옵션별 재고, availability, 단종/예약판매 표시 |
| 배송 조건 오해 | 도착일이나 배송비를 잘못 설명합니다 | 배송 정책, 지역 제한, 마감 시간 |
| 반품 조건 누락 | 구매 후 분쟁 가능성이 커집니다 | 반품 기간, 제외 조건, 왕복 배송비 |
| 리뷰 신호 왜곡 | 오래된 리뷰나 다른 옵션 리뷰가 근거로 쓰입니다 | 리뷰 최신성, 옵션별 리뷰, 외부 리뷰 |
| 결제/취소 흐름 혼란 | AI가 실행 불가능한 구매 경로를 제안합니다 | 결제 수단, 취소 가능 시점, 구독/예약 상품 조건 |

이 리스크는 AI만의 문제가 아닙니다. 사람 고객에게도 불편을 만드는 지점입니다. AI 고객을 위해 정리하는 작업은 결국 커머스 기본기를 다시 맞추는 일입니다.

## 리스크를 판단하는 기준

| 질문 | 낮은 리스크 | 높은 리스크 |
|---|---|---|
| 오류가 나면 고객 피해가 큰가? | 단순 설명 오류 | 가격/배송/반품/건강/금융/계약 오류 |
| 정보가 자주 바뀌는가? | 브랜드 소개/사용법 | 재고/가격/프로모션/배송 마감 |
| AI가 조건을 붙여 물을 가능성이 높은가? | 일반 브랜드 질문 | 예산/시간/대상/정책/법인 결제 질문 |
| 외부에 다른 정보가 많은가? | 공식 정보 중심 | 리뷰/마켓/커뮤니티/비교 글이 많음 |
| 수정 주체가 분산되어 있는가? | 콘텐츠팀 단독 수정 가능 | MD/개발/CS/광고/물류 동시 필요 |

높은 리스크 항목은 “언젠가 고치기”가 아니라 2주 점검 목록에 넣어야 합니다.

## 2주 점검을 리포트로 남기는 법

커머스 GEO 점검은 수정 목록만 남기면 반복 운영이 어렵습니다. 2주가 끝나면 구매 조건별 리스크와 수정 결과를 리포트로 남겨야 합니다.

| 리포트 섹션 | 내용 |
|---|---|
| 우선 상품군 | 매출/문의/리스크 기준으로 고른 상품군 |
| AI 구매 질문셋 | 예산/배송/대상/정책 조건형 질문 |
| 데이터 충돌 | 페이지/feed/schema/정책 불일치 |
| 수정 내역 | 가격/재고/정책/schema/feed 수정 로그 |
| 재측정 결과 | 같은 질문에서 answer accuracy와 source 변화 |
| 다음 액션 | 상품군 확장/정책 보강/리뷰 정리 |

## 2주 점검안

### 1일차에서 3일차: 대표 상품 20개 선정

- 매출 상위 상품 10개
- 광고비가 큰 상품 5개
- 반품이나 문의가 많은 상품 5개
- 경쟁 비교가 자주 일어나는 상품 우선 선정

처음부터 모든 상품을 고치려고 하면 범위가 커집니다. AI 추천에 들어갈 가능성이 높거나, 오류가 났을 때 비용이 큰 상품군부터 봅니다.

### 4일차에서 7일차: 데이터 정합성 점검

- 상품명, SKU, 카테고리, 가격, 재고, 옵션을 확인합니다.
- 상세페이지, Product schema, merchant feed의 불일치를 기록합니다.
- 배송, 반품, 보증 정책 링크가 상품 단위로 연결되는지 봅니다.
- 이미지 안에만 있는 할인 조건과 예외 조건을 텍스트로 꺼냅니다.

핵심은 누락보다 충돌입니다. 한쪽은 최신이고 다른 쪽은 오래된 상태라면 AI는 무엇을 믿어야 할지 알기 어렵습니다.

### 8일차에서 10일차: AI 구매 질문 테스트

다음처럼 실제 구매 의도가 담긴 질문으로 테스트합니다.

- 10만 원 이하로 오늘 주문하면 금요일 전에 받을 수 있는 상품을 골라줘
- 반품이 쉬운 입문자용 상품을 추천해줘
- A 상품과 B 상품 중 재택근무용으로 더 나은 쪽을 골라줘
- 선물용으로 실패 확률이 낮은 옵션을 추천해줘
- 법인카드로 결제하고 세금계산서 처리가 쉬운 상품을 찾아줘

AI가 가격, 재고, 배송, 반품, 옵션을 잘못 이해했다면 해당 정보의 위치와 형식을 고쳐야 합니다.

### 11일차에서 14일차: 수정과 재측정

- Product schema 누락 항목을 보강합니다.
- feed와 페이지 데이터 동기화를 확인합니다.
- 정책 요약 박스를 추가합니다.
- 옵션별 재고와 가격 표시를 개선합니다.
- FAQ와 비교 기준 문장을 추가합니다.
- 같은 프롬프트로 재측정해 개선 여부를 확인합니다.

*2주 점검은 대표 상품 선정, 데이터 정합성 확인, AI 구매 질문 테스트, 수정과 재측정을 한 번의 리포트로 남기는 흐름이다.*

## 문제별 수정 예시

| 문제 | 약한 수정 | 좋은 수정 |
|---|---|---|
| 가격 불일치 | 상품 페이지 가격만 고침 | 페이지/schema/feed/광고 소재/쿠폰 조건을 함께 맞춤 |
| 품절 추천 | 품절 배너 추가 | 옵션별 availability와 단종/예약판매 상태를 feed와 본문에 반영 |
| 반품 오해 | 고객센터 링크만 추가 | 상품 상세에 반품 기간/예외/비용 요약을 넣고 정책 원문 연결 |
| 리뷰 왜곡 | 리뷰 수 강조 | 옵션별 리뷰, 최신 리뷰, 구매 인증, 장단점 요약을 분리 |
| 배송 오류 | 빠른 배송 문구 추가 | 지역/마감 시간/예상 도착일/추가 배송비 조건을 텍스트로 표시 |

## 팀별 담당 기준

| 담당 | 맡을 것 | 완료 기준 |
|---|---|---|
| MD/상품 운영 | 상품명, 옵션, 가격, 재고, 프로모션 | 실제 판매 조건과 feed 값 일치 |
| 콘텐츠/SEO | 상세페이지 첫 문단, 표, FAQ, 정책 요약 | AI 구매 질문에 답할 텍스트 확보 |
| 개발 | Product schema, canonical, 내부 링크, 렌더링 | Rich Results Test/Validator 오류 최소화 |
| CS/운영 | 배송/반품/환불/보증 예외 조건 | 고객센터 문서와 상품 상세 연결 |
| 광고/CRM | 광고 소재와 랜딩 가격/혜택 일치 | 광고 문구와 실제 구매 조건 충돌 없음 |
| 전략/리포트 | 질문셋, 재측정, 월간 리포트 | 수정 전후 결과 비교 가능 |

## 완료 기준

- 대표 상품 20개를 선정했습니다.
- 상품 상세페이지, Product schema, merchant feed, 정책 문서의 충돌을 기록했습니다.
- AI 구매 질문 10개 이상으로 가격/재고/배송/반품/리뷰 오류를 테스트했습니다.
- 2주 안에 수정할 항목과 담당/완료 기준이 정해졌습니다.
- 같은 질문셋으로 재측정할 날짜가 있습니다.

## 흔한 질문

**Q. 모든 상품에 Product schema를 넣어야 하나요?**

상품 페이지가 검색/추천/비교 후보가 되길 원한다면 우선순위 상품부터 점검하는 것이 좋습니다. 다만 schema만 넣는 것으로 끝내지 말고 본문, feed, 정책 문서와 값이 일치해야 합니다.

**Q. AI가 잘못 말한 내용을 바로 고칠 수 있나요?**

직접 수정할 수 있는 것은 자사 페이지, feed, schema, 정책 문서, 외부 프로필의 정확성입니다. AI 답변 자체를 즉시 통제하기보다, 다음 답변에서 참고할 수 있는 근거를 정리하고 같은 질문으로 재측정해야 합니다.

**Q. 커머스 GEO는 MD 업무인가요, SEO 업무인가요?**

둘 다입니다. 상품 조건은 MD/운영이 알고, 검색/콘텐츠 구조는 SEO/콘텐츠가 알고, schema/feed는 개발/광고 운영과 연결됩니다. 커머스 GEO는 한 팀의 체크리스트가 아니라 데이터 정합성 운영입니다.
