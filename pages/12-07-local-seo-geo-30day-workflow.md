## 로컬 SEO/GEO 30일 운영 워크플로우

![로컬 SEO/GEO 30일 운영 워크플로우 대표 이미지](../assets/images/page-heroes/halox-geo-12-07-local-seo-geo-30day-workflow-hero.png)

로컬 SEO/GEO는 한 번 수정하고 끝나는 프로젝트가 아닙니다. 병원, 학원, 오프라인 매장, 지역 전문 서비스는 주소, 영업시간, 리뷰, 사진, 예약 링크, 이벤트, 지점 정보가 계속 바뀝니다. 그래서 30일 단위로 `점검 → 수정 → 측정 → 리포트 → 다음 티켓` 루프를 만들어야 합니다.

이 페이지는 12-01부터 12-06까지의 내용을 실제 운영 보드로 바꾸는 방법을 정리합니다.

## 30일 운영 루프 한눈에 보기

| 주차 | 목표 | 핵심 작업 | 산출물 |
|---|---|---|---|
| 1주차 | 기준선 진단 | NAP/지도/리뷰/지점 페이지/질문셋 현황 점검 | 로컬 SEO/GEO 기준선 표 |
| 2주차 | 기본 신호 수정 | NAP 통일, 지도 프로필 수정, 카테고리/영업시간/사진 보강 | 채널별 수정 로그 |
| 3주차 | 콘텐츠/리뷰/전환 보강 | 지역 페이지, FAQ, 리뷰 답변, 예약/길찾기 CTA 개선 | 실행 티켓 완료표 |
| 4주차 | AI 답변 측정 | 지역 질문셋으로 mention/source/citation/오류 확인 | 월간 GEO 리포트 |

핵심은 순서입니다. 1주차에 기준선을 잡지 않으면 4주차 측정 결과가 좋아졌는지 나빠졌는지 알 수 없습니다. 2주차에 지도/기본 정보를 고치지 않으면 3주차 콘텐츠가 좋아도 방문 전환이 끊깁니다.

## 1주차: 기준선 진단

1주차에는 아무것도 고치기 전에 현재 상태를 기록합니다. 이때 검색결과, 지도, 리뷰, 홈페이지, AI 답변을 같은 표에 넣어야 합니다.

| 영역 | 확인 항목 | 기록 방식 |
|---|---|---|
| NAP | 상호명, 주소, 전화, 영업시간, URL | 공식 기준값과 채널별 값 비교 |
| 지도 | 네이버/Google/카카오/Apple 프로필 | 카테고리, 사진, 리뷰, 예약 링크 캡처 |
| 홈페이지 | 지점 페이지, 서비스 페이지, FAQ, CTA | 색인 여부, title/meta, 내부 링크 확인 |
| 리뷰 | 플랫폼별 수량, 최근성, 답변율, 주제 | 최근 30일/90일 기준 기록 |
| 외부 권위 | 협회, 전문 플랫폼, 지역 매체, 언론 | NAP 일치와 설명 문구 확인 |
| AI 답변 | 지역 질문셋 20개 이상 | mention/source/citation/오류 기록 |

## 2주차: 기본 신호 수정

2주차에는 수정 범위를 너무 넓히지 않습니다. 먼저 검색엔진과 지도 서비스가 업체를 정확히 식별하도록 만드는 작업에 집중합니다.

| 수정 작업 | 우선순위 | 완료 기준 |
|---|---|---|
| NAP 표준화 | 최상 | 공식 사이트/지도/외부 프로필의 이름/주소/전화가 일치 |
| 지도 카테고리 수정 | 최상 | 대표 서비스와 카테고리가 맞고 중복/오분류가 없음 |
| 영업시간/특별 영업시간 수정 | 높음 | 공휴일, 점심시간, 야간 운영, 예약 가능 시간이 최신 |
| 사진 보강 | 높음 | 외관, 입구, 내부, 주차, 대표 서비스 사진이 최신 |
| 지점 URL 연결 | 높음 | 지도 프로필에서 정확한 지점 페이지/예약 페이지로 이동 |
| 오래된 외부 정보 정리 | 중간 | 이전 주소/전화/폐점 정보 노출을 수정 요청 |

이 단계의 목표는 SEO 성과를 즉시 올리는 것이 아니라, 신호 분산을 줄이는 것입니다.

## 3주차: 콘텐츠/리뷰/전환 보강

3주차에는 사용자가 실제로 선택할 때 필요한 정보를 보강합니다. 지역명만 반복하는 페이지가 아니라, 방문 전 확인할 기준을 알려주는 페이지를 만들어야 합니다.

| 자산 | 보강 내용 | 예시 |
|---|---|---|
| 지역 랜딩페이지 | 위치, 접근성, 서비스, 대상, 예약, 주차, FAQ | `강남역 근처 야간 진료 치과 선택 기준` |
| 서비스 페이지 | 절차, 대상, 준비물, 주의사항, 비용 범위 | `도수치료 상담 전 확인할 질문` |
| FAQ | 방문 전 질문과 전환 질문 | `예약 없이 방문 가능한가요?`, `주차는 가능한가요?` |
| 리뷰 답변 | 감사/안내/개인정보 보호/위험 표현 회피 | 증상이나 치료 결과를 답변에 쓰지 않음 |
| CTA | 전화, 예약, 길찾기, 상담, 준비물 | 지점별 CTA 분리 |
| 내부 링크 | 서비스 → 지점 → 예약 → FAQ 연결 | 사용자가 행동까지 이동 가능 |

## 4주차: AI 답변 측정과 리포트

4주차에는 12-05에서 만든 질문셋을 다시 실행합니다. 같은 플랫폼, 같은 질문, 같은 위치 조건으로 측정해야 비교가 가능합니다.

| 항목 | 기록 내용 | 해석 |
|---|---|---|
| 브랜드 언급 | 우리 브랜드/지점이 답변에 나오는가 | 후보군 진입 여부 |
| 추천 이유 | 어떤 이유로 언급되는가 | 리뷰/위치/전문성/전환 정보 중 무엇이 작동하는가 |
| source | 답변 근거로 어떤 출처가 쓰이는가 | 공식 페이지/지도/외부 권위 중 무엇이 잡히는가 |
| citation | 화면에 어떤 링크가 보이는가 | 클릭 가능한 근거 확보 여부 |
| 경쟁사 | 어떤 경쟁사가 반복되는가 | 경쟁 신호와 콘텐츠 갭 |
| 오류 | 주소/전화/영업시간/가격/표현 오류 | 즉시 수정 티켓 |
| 리스크 | 의료광고/후기/개인정보/과장 표현 | 검수 필요 티켓 |

## 월간 리포트 구조

로컬 SEO/GEO 리포트는 예쁜 그래프보다 실행 가능한 판단을 봐야 합니다.

| 섹션 | 내용 | 질문 |
|---|---|---|
| 요약 | 이번 달 가장 큰 변화 3개 | 무엇이 좋아졌고 무엇이 위험한가? |
| 기본 신호 | NAP/지도/리뷰/지점 페이지 상태 | 검색엔진이 업체를 정확히 이해하는가? |
| 전환 신호 | 전화/예약/길찾기/웹사이트 클릭 | 노출이 행동으로 이어지는가? |
| AI 답변 | mention/source/citation/경쟁사/오류 | AI가 우리를 어떤 근거로 설명하는가? |
| 리스크 | 의료광고/후기/개인정보/과장 표현 | 공개 전 수정해야 할 문장은 무엇인가? |
| 다음 액션 | 다음 30일 티켓 | 누가 언제 무엇을 고칠 것인가? |

## 30일 실행 보드 템플릿

| 상태 | 티켓 | 담당 | 근거 | 완료 기준 | 마감 |
|---|---|---|---|---|---|
| 대기 | 강남점 Google Business Profile 영업시간 수정 | 운영 | 공휴일 시간 불일치 | Google/홈페이지/네이버 시간 일치 | 5/3 |

## 완료 기준

- NAP 기준값과 채널별 값이 일치합니다.
- 네이버 플레이스, Google Business Profile, 카카오맵, Apple Maps의 핵심 정보가 최신입니다.
- 지역 질문셋 20개 이상을 같은 조건으로 측정했습니다.
- mention/source/citation/경쟁사/오류/리스크를 분리해 기록했습니다.
- 다음 30일 실행 티켓이 담당자와 마감일로 정리되었습니다.
