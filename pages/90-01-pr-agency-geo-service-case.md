## PR/마케팅 에이전시: GEO를 설명 가능한 서비스로 만드는 법

![PR 마케팅 에이전시 GEO 서비스 패키지](../assets/images/page-heroes/halox-geo-90-01-pr-agency-geo-service-case-hero.png)

PR/마케팅 조직은 GEO를 가장 먼저 설명해야 하는 입장에 놓입니다. 그런데 `AI 검색에 잘 나오게 해드립니다`라고 말하면 기존 SEO, PR, 콘텐츠 마케팅과 무엇이 다른지 분명하지 않습니다.

이 사례에서 가장 중요한 포인트는 GEO를 유행어가 아니라 `질문 설계 → 기준선 측정 → 콘텐츠/출처/기술 진단 → 실행 리포트`로 판매 가능한 서비스 구조로 바꾸는 것입니다.

[TOC]

## 상황

AcmePR은 B2B SaaS, 소비재, 병원, 교육 브랜드의 PR과 콘텐츠를 맡고 있습니다. 고객사는 “ChatGPT에 우리 브랜드가 나오게 해달라”고 요청하지만, 내부 팀은 아직 SEO 리포트, 언론 클리핑, 블로그 발행량 중심으로 보고합니다.

핵심 문제는 GEO 성과가 단순 노출이 아니라는 점입니다. AI 답변에서는 브랜드가 언급되는지, 답변 근거가 무엇인지, 화면 인용이 보이는지, 경쟁사가 어떤 문맥으로 등장하는지, 답변이 안전한지까지 나눠 봐야 합니다.

## 기준선 진단

AcmePR은 먼저 고객사 1곳을 골라 아래처럼 기준선을 측정했습니다.

| 질문군 | 질문 예시 | 현재 결과 | 문제 |
|---|---|---|---|
| 브랜드 인식 | AcmeTool은 어떤 회사야? | 오래된 제품 설명이 나옴 | 엔티티 설명 불일치 |
| 비브랜드 추천 | 스타트업용 고객지원 툴 추천해줘 | 경쟁사 3곳만 반복 | 카테고리 페이지/source 부족 |
| 비교 | AcmeTool과 Zendesk 차이 알려줘 | 기능 비교가 부정확 | 비교 FAQ 없음 |
| 구매/전환 | 50명 팀이 바로 도입 가능한 툴은? | 가격/계약 정보 누락 | 요금제/도입 페이지 구조 약함 |
| 리스크 | 고객 데이터 보안은 괜찮아? | 보안 인증 언급 없음 | 신뢰 근거 페이지 없음 |

이 표 하나만으로도 서비스 범위가 분명해집니다. PR은 언론 노출만 담당하지 않고, AI가 브랜드를 설명할 때 필요한 근거 생태계를 설계해야 합니다.

## 0~12장 연결 지도

| 필요한 개념 | 연결 장 | AcmePR 적용 |
|---|---|---|
| SEO 키워드와 SERP 의도 | 01장 | 고객사의 비브랜드 카테고리 질문을 수집 |
| mention/source/citation | 02장 | AI 답변 기준선 리포트 상품화 |
| Query Fan-out | 03장 | 의사결정자/실무자/비교/리스크 질문으로 확장 |
| Answer-first 콘텐츠 | 04장 | 고객사 핵심 페이지 리라이트 |
| 오프사이트 source map | 05장 | 언론/기고/외부 블로그/커뮤니티를 역할별로 배치 |
| 기술 점검 | 06장 | 고객사 페이지가 크롤링/색인/스키마 측면에서 읽히는지 확인 |
| 리포트/제안서 | 09장 | 월간 GEO 리포트와 실행 범위 산정 |
| 4주 실행 | 10장 | PoC 상품 구성 |

## 초기에 자주 실패하는 패턴

AcmePR의 첫 제안서는 이렇게 시작했습니다.

| 나쁜 제안 | 왜 약한가 |
|---|---|
| AI 검색 최적화 콘텐츠 10건 발행 | 어떤 질문에 대응하는지 없음 |
| 보도자료 2건 배포 | source/citation에서 어떤 역할인지 불명확 |
| 브랜드 언급량 증가 목표 | mention/source/citation을 구분하지 않음 |
| ChatGPT 노출 보장 | 플랫폼별 답변 변동성과 측정 조건을 무시 |

이 제안은 고객이 이해하기 쉽지만 실행 후 성과를 설명하기 어렵습니다. GEO 서비스는 콘텐츠 수량보다 질문과 근거의 연결을 팔아야 합니다.


## 실전 시나리오: 첫 고객 제안 미팅

AcmePR의 첫 고객은 B2B SaaS 회사입니다. 고객은 “ChatGPT에서 우리 이름이 안 나온다”고 말했지만, 실제 핵심 문제는 더 복합적이었습니다. 브랜드명 질문에서는 회사가 언급됐지만, `스타트업 고객지원 툴 추천`, `Zendesk 대안`, `보안 인증 있는 고객지원 솔루션` 같은 비브랜드/비교/리스크 질문에서는 경쟁사만 반복됐습니다.

이때 제안의 초점은 “AI에 노출되게 하겠다”가 아니라 다음처럼 바뀌어야 합니다.

| 고객의 말 | 실제 진단 질문 | 제안서에 들어갈 표현 |
|---|---|---|
| ChatGPT에 안 나와요 | 어떤 질문군에서 빠지는가? | 비브랜드/비교/리스크 질문 기준선 진단 |
| 콘텐츠를 더 써야 하나요? | 기존 페이지가 답변 근거로 읽히는가? | 핵심 페이지 Answer-first 리라이트 |
| 언론 보도를 더 내야 하나요? | AI가 참고할 제3자 근거가 부족한가? | source map 기반 PR/기고/디렉터리 설계 |
| 성과는 어떻게 보나요? | mention/source/citation이 분리 측정되는가? | 월간 GEO 리포트와 실행 티켓 |

이렇게 바꾸면 GEO는 막연한 신사업이 아니라 기존 PR/SEO/콘텐츠 역량을 재조합한 설명 가능한 서비스가 됩니다.

## 제안서에 넣을 수 있는 진단 문장

> 현재 고객사는 브랜드명 질문에서는 일부 언급되지만, 구매 전 비교 질문과 보안/도입 리스크 질문에서는 경쟁사 source가 반복됩니다. 이번 PoC는 30개 질문셋을 기준으로 mention/source/citation을 측정하고, 공식 페이지/외부 근거/기술 상태를 나눠 개선하는 방식으로 진행합니다.

이 문장의 장점은 노출 보장을 약속하지 않는다는 점입니다. 대신 무엇을 측정하고 무엇을 고칠지 분명히 말합니다.

## 실제로 팔 수 있는 GEO 서비스 패키지

| 패키지 | 대상 | 포함 작업 | 산출물 |
|---|---|---|---|
| GEO 기준선 진단 | 처음 시작하는 브랜드 | 질문셋 30개, 3개 플랫폼 측정, 경쟁사/source/citation 기록 | 기준선 리포트 |
| Answer-first 리라이트 | 콘텐츠는 있지만 답변성이 약한 브랜드 | 핵심 페이지 3~5개 구조 개선, FAQ/표/schema 제안 | 리라이트 원고와 체크리스트 |
| Source map 구축 | PR/외부 언급이 필요한 브랜드 | 언론/기고/커뮤니티/디렉터리 후보 설계 | 오프사이트 실행표 |
| 월간 GEO 운영 | 반복 관리가 필요한 브랜드 | 기준선 재측정, 변경점 해석, 다음 액션 티켓 | 월간 리포트 |

![PR 에이전시 GEO 서비스 패키지 맵](../assets/images/body-figures/halox-geo-90-01-pr-agency-geo-service-package-map-codex-only.png)

*PR/마케팅 에이전시는 GEO를 콘텐츠 발행량이 아니라 질문셋, 기준선 측정, source map, 월간 리포트로 설명해야 한다.*

## 4주 실행 흐름

| 주차 | 실행 | 고객에게 보여줄 것 |
|---|---|---|
| 1주차 | SEO 키워드/SERP/AI 질문셋 수집 | 질문군별 기준선 표 |
| 2주차 | 핵심 페이지 리라이트와 기술 점검 | Before/After 원고, schema/robots/sitemap 체크 |
| 3주차 | PR/source map 설계 | 어떤 외부 출처가 어떤 질문에 필요한지 표로 제시 |
| 4주차 | 재측정과 리포트 | mention/source/citation 변화, 다음 30일 액션 |

## 제안서 1페이지 구조

| 섹션 | 포함 내용 |
|---|---|
| 고객의 문제 | AI 답변에서 브랜드가 어떤 질문에 빠지거나 오해되는지 |
| 측정 방식 | 플랫폼, 질문셋, 날짜, 위치/로그인 조건, source/citation 기록 방식 |
| 실행 범위 | 콘텐츠/출처/기술/리스크 중 이번 달에 다룰 영역 |
| 기대 산출물 | 기준선 리포트, 리라이트 원고, source map, 실행 티켓 |
| 제외 범위 | 노출 보장, 모든 AI 플랫폼 통제, 단기 순위 보장 |

|---|---|

| 브랜드가 언급되는 것과 답변 근거로 쓰이는 것 중 무엇이 더 중요한가요? | mention/source/citation 우선순위 |

## 역할별 실행 분담

| 역할 | 할 일 | 산출물 |
|---|---|---|
| 전략/PM | 질문셋과 측정 범위 결정 | 프로젝트 브리프 |
| SEO 담당 | 키워드/SERP/기술 상태 점검 | 검색 의도 맵, technical checklist |
| PR 담당 | 외부 출처 후보와 메시지 정리 | source map, 보도자료/기고 방향 |
| 콘텐츠 담당 | Answer-first 페이지 작성 | 리라이트 원고, FAQ, 비교표 |
| 데이터/리포트 담당 | 플랫폼별 결과 기록 | 월간 GEO 리포트 |
| 법무/브랜드 | 위험 표현 검수 | 승인/수정 로그 |

## 월간 리포트 목차 예시

1. 이번 달 핵심 변화 3개
2. 질문군별 브랜드 언급률
3. source/citation 변화
4. 경쟁사 반복 등장 질문
5. 콘텐츠/출처/기술 원인 분석
6. 리스크 표현과 수정 필요 항목
7. 다음 30일 실행 티켓

## 고객에게 하지 말아야 할 약속

- 특정 AI 답변에 항상 노출된다고 보장하지 않습니다.
- 모든 플랫폼의 답변을 통제할 수 있다고 말하지 않습니다.
- 보도자료 배포만으로 GEO가 해결된다고 말하지 않습니다.
- 콘텐츠 발행량을 성과로 포장하지 않습니다.
- source와 citation을 구분하지 않은 캡처 보고서로 끝내지 않습니다.
