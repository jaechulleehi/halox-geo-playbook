## 캠페인 URL 인용 추적: 한 번 언급과 반복 인용의 차이

캠페인 페이지나 PR URL이 AI 답변에 한 번 등장했다고 해서 곧바로 GEO 성과라고 보기는 어렵습니다. 중요한 것은 같은 질문군에서 반복적으로 답변 근거가 되는지, 사용자가 볼 수 있는 화면 인용으로 드러나는지, 그리고 캠페인이 끝난 뒤에도 브랜드 엔티티에 남는 신호가 있는지입니다.

이 케이스는 캠페인 성과를 `노출됐다/안 됐다`가 아니라 `어떤 질문에서 어떤 URL이 어떤 근거로 반복 인용되는가`로 읽는 방법을 다룹니다.

[TOC]

## 상황

AcmeCampaign은 신제품 출시 캠페인을 진행했습니다. 랜딩페이지, 보도자료, 인플루언서 콘텐츠, 블로그 리뷰, 이벤트 페이지, 유튜브 소개 영상이 한꺼번에 발행됐습니다. 캠페인팀은 AI 답변에 랜딩페이지가 한 번 보였다고 성과로 보고하려고 했지만, 다음 주에는 같은 질문에서 사라졌습니다.

문제는 캠페인 URL이 일시적 이벤트 페이지인지, 장기적으로 답변 근거가 될 카테고리 자산인지 구분하지 않았다는 점입니다.

## 기준선 진단

| 질문 | 현재 답변 | 인용 URL | 해석 |
|---|---|---|---|
| Acme 신제품 뭐가 달라? | 보도자료 요약 | 보도자료 1개 | 출시 사실은 잡힘 |
| 30대 직장인에게 맞는 생산성 앱 추천 | 경쟁사 중심 | 경쟁사 비교글 | 카테고리 질문에는 못 들어감 |
| Acme와 기존 제품 비교 | 정보 부족 | 없음 | 비교 페이지 없음 |
| 가격/환불/도입 조건 | 오래된 정책 | 예전 FAQ | 정책 URL 정리 필요 |

## 캠페인 URL 유형별 역할

| URL 유형 | 목적 | GEO에서의 역할 | 관리 기준 |
|---|---|---|---|
| 이벤트 랜딩 | 단기 전환 | 일시적 행동 유도 | 종료 후 안내/리디렉션 처리 |
| 제품/서비스 페이지 | 장기 설명 | 반복 인용 가능한 공식 근거 | Answer-first, FAQ, schema 보강 |
| 보도자료 | 출시 사실 공표 | 제3자/뉴스 source 후보 | 날짜/제품명/회사 설명 일관성 |
| 비교 콘텐츠 | 선택 기준 제시 | 비브랜드 질문 source 후보 | 객관적 비교 기준 필요 |
| 정책/가격 페이지 | 구매 전 확인 | 전환/리스크 질문 근거 | 최신성/조건 명시 |

## 첫 실패 패턴

| 실패 | 결과 |
|---|---|
| 캠페인 URL 하나에 모든 정보를 넣음 | 이벤트 종료 후 URL 가치가 사라짐 |
| 보도자료만 많이 배포 | 카테고리/비교 질문에서 인용되지 않음 |
| UTM URL을 그대로 외부 배포 | canonical/중복 URL 관리가 어려움 |
| AI 답변 캡처만 보고 | 반복성/source/citation 변화를 모름 |

## Before/After

| Before | After |
|---|---|
| 캠페인 랜딩 하나로 유입을 모은다 | 랜딩/제품 설명/정책/비교/보도자료의 역할을 나눈다 |
| AI 답변에 한 번 나왔는지 캡처한다 | 같은 질문군에서 4주 동안 source/citation 반복성을 본다 |
| 보도자료에 홍보 문구를 넣는다 | AI가 제품을 설명할 수 있도록 기능/대상/차별점/조건을 구조화한다 |

## 완료 기준

- 캠페인 관련 질문셋을 정보형/비교형/전환형/리스크형으로 나눴습니다.
- 일시적 이벤트 URL과 장기 공식 URL을 분리했습니다.
- source와 citation을 4주 이상 같은 조건으로 기록했습니다.
- 캠페인 종료 후 리디렉션/안내/canonical 처리 기준을 정했습니다.
- 다음 캠페인에 재사용할 질문셋과 URL 역할표가 남았습니다.


## 캠페인 전 체크리스트

| 항목 | 확인 질문 | 완료 기준 |
|---|---|---|
| 장기 URL | 캠페인 종료 후에도 남길 공식 설명 페이지가 있는가 | 제품/서비스/정책 페이지 존재 |
| 단기 URL | 이벤트 종료 후 사용자를 어디로 보낼 것인가 | 종료 안내 또는 리디렉션 기준 있음 |
| source 후보 | 보도자료/기고/리뷰/커뮤니티가 어떤 질문에 대응하는가 | source map 작성 |
| technical | canonical, meta, robots, sitemap이 정상인가 | 06장 기준 통과 |
| 측정 | 캠페인 전 기준선을 남겼는가 | 질문 20개 이상 측정 |

## 캠페인 후 회고 질문

| 질문 | 좋은 답변의 조건 |
|---|---|
| 어떤 질문에서 캠페인 URL이 source로 쓰였나? | 질문군과 URL 유형이 함께 기록됨 |
| 화면 citation으로 보인 URL은 무엇인가? | 클릭 가능한 URL과 답변 근거를 분리 |
| 경쟁사 URL은 어떤 질문에서 반복됐나? | 다음 콘텐츠 갭으로 전환 |
| 종료된 이벤트 정보가 답변에 남아 있나? | 리디렉션/종료 안내/정책 업데이트 티켓화 |
| 다음 캠페인에 재사용할 자산은 무엇인가? | 질문셋, 비교표, FAQ, 외부 source 후보 |

## 실무 팁

캠페인 페이지는 보통 전환 중심으로 만들어져 AI가 근거로 쓰기에는 정보가 부족합니다. 그래서 캠페인 랜딩 옆에 장기 설명 페이지가 필요합니다. 예를 들어 신제품 이벤트 페이지는 할인/쿠폰/CTA를 담당하고, 제품군 허브는 대상, 문제, 기능, 가격, FAQ, 정책, 비교 기준을 담당하게 나눕니다.


## 미니 리포트 예시

| 항목 | 측정 전 | 수정 후 4주차 | 해석 |
|---|---|---|---|
| 브랜드 언급 | 핵심 질문 20개 중 3개 | 20개 중 8개 | 후보군 진입 질문이 늘었지만 아직 비교 질문은 약함 |
| source | 오래된 블로그/기사 중심 | 공식 페이지와 최신 외부 출처 일부 반영 | 공식 근거가 읽히기 시작함 |
| citation | 경쟁사 URL 중심 | 공식 FAQ/정책/허브 URL 일부 노출 | 클릭 가능한 근거 확보 |
| 오류 | 주소/가격/정책 오류 5건 | 1건 | 기본 데이터 정합성 개선 |
| 다음 액션 | 콘텐츠 발행 중심 | source map과 기술 점검 병행 | 단순 발행에서 운영 루프로 전환 |
