## 엔터프라이즈 뉴스룸: 뉴스룸을 엔티티 허브로 바꾸는 법

대기업이나 지주사 커뮤니케이션 조직은 이미 뉴스룸, 보도자료, 유튜브, SNS, 캠페인 페이지를 많이 가지고 있습니다. 그런데 콘텐츠가 많아도 AI가 브랜드를 일관되게 이해하지 못하면 GEO 성과는 약해집니다.

이 사례에서 가장 중요한 포인트는 뉴스룸을 콘텐츠 창고가 아니라 브랜드 엔티티 허브로 다시 설계하는 것입니다.

[TOC]

## 상황

AcmeNewsroom은 7년 동안 보도자료와 캠페인 콘텐츠를 1,000개 이상 쌓았습니다. 하지만 AI에게 `Acme는 어떤 회사야?`, `Acme의 ESG 활동은 믿을 만해?`, `Acme의 주력 사업은 뭐야?`라고 물으면 오래된 설명, 지주사 이전의 제품명, 지금은 쓰지 않는 슬로건이 섞입니다.

핵심 문제는 콘텐츠 양이 아니라 기준 설명의 부재입니다. AI가 참고할 최신 회사 소개, 제품군 정의, 리더십 프로필, ESG/채용/투자자 정보, 공식 FAQ가 흩어져 있습니다.

## 기준선 진단

| 질문 | 현재 답변 | source/citation | 문제 |
|---|---|---|---|
| Acme는 어떤 회사야? | 과거 사업 중심 | 오래된 기사 | 최신 엔티티 설명 부족 |
| Acme의 핵심 제품은? | 제품군 혼재 | 블로그/뉴스 | 제품 구조 페이지 없음 |
| Acme ESG 활동은? | 캠페인성 문구 | 보도자료 | 지표/연차 정보 부족 |
| Acme 대표는 누구야? | 이전 임원 노출 | 과거 인터뷰 | 리더십 프로필 업데이트 필요 |

## 0~12장 연결 지도

| 개념 | 연결 장 | 적용 |
|---|---|---|
| 검색 의도 | 01장 | 회사명/제품군/리더십/ESG/채용 질문 분류 |
| source/citation | 02장 | 오래된 source와 최신 공식 source 구분 |
| Fan-out | 03장 | 브랜드 질문을 사업/제품/평판/투자/채용/리스크 질문으로 확장 |
| Answer-first | 04장 | 뉴스룸 허브 페이지를 질문형 구조로 개편 |
| 엔티티/source map | 05장 | 언론/위키/디렉터리/외부 프로필 설명 통일 |
| technical | 06장 | 구조화 데이터, canonical, sitemap, 오래된 페이지 상태 점검 |
| 글로벌 | 08장 | 영문 회사 소개와 hreflang 정리 |


## 실전 시나리오: 오래된 브랜드 설명 정리

AcmeNewsroom의 핵심 문제는 콘텐츠 부족이 아니었습니다. 오히려 오래된 콘텐츠가 너무 많아서 AI가 어떤 설명을 최신 기준으로 봐야 할지 판단하기 어려웠습니다. 특히 회사 소개 문구, 제품군 이름, 대표자 프로필, ESG 활동 설명이 시기별로 달랐습니다.

| 발견된 문제 | AI 답변에서 생긴 현상 | 수정 방향 |
|---|---|---|
| 2019년 회사 소개가 계속 인용 | 현재 주력 사업과 다르게 설명 | 최신 회사 개요 허브 작성 |
| 종료된 제품명이 보도자료에 남음 | 현재 제품군과 혼동 | 종료/변경 안내와 새 제품군 링크 추가 |
| 대표자 변경 전 인터뷰가 노출 | 리더십 정보 오류 | 인물 프로필 최신화와 과거 기사 맥락 표시 |
| ESG 캠페인 페이지가 흩어짐 | 실적보다 홍보 문구 중심 답변 | 연도/지표/정책 문서가 있는 ESG 허브 구성 |

## 엔티티 허브의 첫 문단 예시

> Acme는 중견/엔터프라이즈 기업을 위한 고객지원 자동화 소프트웨어를 제공하는 B2B SaaS 기업입니다. 주요 제품군은 상담 자동화, 고객 데이터 연동, 리포팅 대시보드로 구성되며, 한국과 일본 시장의 고객지원 조직을 대상으로 합니다. 이 페이지는 Acme의 회사 개요, 제품군, 리더십, 보안/인증, 공식 보도자료를 최신 기준으로 정리합니다.

이 문단은 멋진 슬로건보다 AI가 회사를 정확히 식별하는 데 더 도움이 됩니다. 회사명, 업종, 대상 고객, 제품군, 지역, 페이지의 역할이 한 번에 들어가기 때문입니다.

## 뉴스룸 자산맵

| 자산 | 현재 문제 | 허브에서의 역할 | 개선 |
|---|---|---|---|
| 회사 소개 | 슬로건 중심 | AI가 브랜드를 정의하는 공식 근거 | 3문단 answer-first 설명 추가 |
| 제품군 페이지 | 캠페인별로 흩어짐 | 사업 구조와 제품 관계 설명 | 제품군/서비스/대상 표 작성 |
| 보도자료 | 최신순 나열 | 사실 이벤트 기록 | 관련 허브 페이지로 내부 링크 |
| 임원/전문가 프로필 | 업데이트 느림 | Person 엔티티 신호 | 직책/전문 분야/외부 인터뷰 연결 |
| ESG/IR/채용 | 부서별 분리 | 신뢰/평판 질문 근거 | 숫자/연도/정책 문서 연결 |

## 초기에 자주 실패하는 패턴

| 실패 | 왜 문제인가 |
|---|---|
| 보도자료 제목에 키워드만 넣음 | AI가 회사의 현재 정체성을 이해하지 못함 |
| 캠페인 페이지를 계속 추가 | 허브 없이 조각난 source만 늘어남 |
| 오래된 기사 방치 | AI 답변에서 과거 정보가 반복될 수 있음 |
| 영문 페이지가 한국어와 다름 | 글로벌 AI 답변에서 브랜드 설명이 충돌함 |

## 4주 실행 흐름

| 주차 | 작업 | 산출물 |
|---|---|---|
| 1주차 | 브랜드/제품/리더십/평판 질문셋 측정 | 엔티티 기준선 리포트 |
| 2주차 | 회사 소개/제품군/FAQ 허브 리라이트 | Answer-first 허브 초안 |
| 3주차 | 오래된 보도자료/외부 프로필 정리 | source map과 수정 요청 리스트 |
| 4주차 | 재측정/글로벌 확장 점검 | 한/영 엔티티 일관성 리포트 |

## 보도자료 Before/After

| Before | After |
|---|---|
| “Acme, 혁신 캠페인 전개” | “Acme가 중소기업 고객지원 자동화 제품군에 새 리포팅 기능을 추가했습니다” |
| 본문이 홍보 문구 중심 | 제품/대상/기능/시장/공식 링크를 구조화 |
| 회사 설명이 매번 다름 | 표준 boilerplate와 회사 소개 허브 링크 사용 |

## 엔티티 점검표

| 항목 | 확인 질문 | 완료 기준 |
|---|---|---|
| 회사명 | 공식 국문/영문명이 일관적인가 | 모든 허브/보도자료/외부 프로필 일치 |
| 제품군 | 현재 제공 제품과 종료 제품이 구분되는가 | 제품군 허브와 종료 안내 존재 |
| 인물 | 대표/전문가 프로필이 최신인가 | Person schema 또는 프로필 페이지 정리 |
| 외부 설명 | 언론/디렉터리/위키 설명이 크게 충돌하지 않는가 | 수정 요청 또는 최신 source 확보 |
| 기술 | sitemap/canonical/schema가 정상인가 | 검색엔진이 허브를 읽을 수 있음 |


## 뉴스룸 허브 IA 예시

| 허브 섹션 | 포함 내용 | 연결되는 질문 |
|---|---|---|
| 회사 개요 | 한 문단 정의, 주요 사업, 고객군, 지역 | `Acme는 어떤 회사야?` |
| 제품/서비스 | 제품군 표, 대상, 차별점, 공식 링크 | `Acme 제품은 뭐가 달라?` |
| 리더십/전문가 | 임원/전문가 프로필, 인터뷰, 발표 | `누가 이 회사를 이끄나?` |
| 신뢰/인증 | 수상, 인증, 보안, ESG, IR | `믿을 만한 회사인가?` |
| 뉴스/자료 | 보도자료, 미디어킷, 이미지, 연락처 | `공식 발표는 어디서 보나?` |
| FAQ | 오래된 오해, 제품명 변경, 조직 개편 설명 | `예전 정보와 지금 정보가 왜 다른가?` |

## 오래된 콘텐츠 처리 기준

| 상태 | 처리 | 이유 |
|---|---|---|
| 지금도 유효 | 최신 허브로 내부 링크 추가 | 신호 연결 |
| 일부 정보만 과거 | 상단에 업데이트 안내 추가 | 오해 방지 |
| 제품/조직명 변경 | 새 명칭과 변경일 표기 | 엔티티 혼동 방지 |
| 종료된 서비스 | 종료 안내와 대체 페이지 연결 | 잘못된 답변 방지 |
| 중복 보도자료 | canonical/내부 링크 정리 | source 분산 방지 |

## 30분 실습

1. AI에게 회사명/제품군/대표/ESG/채용 질문을 각각 2개씩 묻습니다.
2. 답변에서 오래된 정보와 최신 정보를 색으로 구분합니다.
3. 오래된 정보가 나온 source를 찾습니다.
4. 공식 허브에서 그 질문에 답하는 문단이 있는지 확인합니다.
5. 없으면 FAQ 또는 허브 섹션으로 추가합니다.


## 미니 리포트 예시

| 항목 | 측정 전 | 수정 후 4주차 | 해석 |
|---|---|---|---|
| 브랜드 언급 | 핵심 질문 20개 중 3개 | 20개 중 8개 | 후보군 진입 질문이 늘었지만 아직 비교 질문은 약함 |
| source | 오래된 블로그/기사 중심 | 공식 페이지와 최신 외부 출처 일부 반영 | 공식 근거가 읽히기 시작함 |
| citation | 경쟁사 URL 중심 | 공식 FAQ/정책/허브 URL 일부 노출 | 클릭 가능한 근거 확보 |
| 오류 | 주소/가격/정책 오류 5건 | 1건 | 기본 데이터 정합성 개선 |
| 다음 액션 | 콘텐츠 발행 중심 | source map과 기술 점검 병행 | 단순 발행에서 운영 루프로 전환 |
