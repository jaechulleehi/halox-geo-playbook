## 산업별 GEO 케이스북

GEO는 개념만으로 이해하기 어렵습니다. 같은 `브랜드 언급률`, `답변 근거(source)`, `화면 인용(citation)`을 보더라도 업종마다 질문의 모양, 필요한 출처, 실행 순서가 달라지기 때문입니다.

90장은 앞에서 배운 개념을 업종별 상황에 적용하는 케이스북입니다. 실제 회사나 브랜드를 특정하기보다, 여러 현장에서 반복적으로 나타나는 문제 유형을 기준으로 정리합니다. 독자는 자기 업종과 가까운 사례를 고른 뒤 SEO 키워드, AI 질문셋, 콘텐츠/출처/기술 과제를 어떻게 나눌지 확인하면 됩니다.

## 이 장을 읽는 법

먼저 우리 업종과 가장 가까운 사례를 하나 고릅니다. 그다음 해당 사례의 `SEO 키워드 관점`, `AI 질문/GEO 관점`, `실행 순서`, `측정 지표`를 자기 브랜드에 맞게 바꿉니다.

다만 이 장은 업종명을 바꿔 끼우는 예시집이 아닙니다. 산업별 GEO는 `누가 의사결정하는가`, `AI가 어떤 단위를 답변 후보로 삼는가`, `어떤 출처를 신뢰 근거로 보는가`, `답변 뒤에 어떤 행동이 이어지는가`, `실패했을 때 어떤 리스크가 생기는가`에 따라 달라집니다. 이 다섯 가지를 먼저 잡아야 케이스를 자기 브랜드에 맞게 옮길 수 있습니다.

| 구분 기준 | 봐야 할 질문 | 예시 |
|---|---|---|
| 의사결정자 | 누가 최종 선택을 하는가 | PR 담당자, 구매위원회, 환자, 투자자, 커머스 고객 |
| 답변 단위 | AI가 무엇을 후보로 묶는가 | 브랜드, 캠페인 URL, 뉴스룸, 지점, 상품, 전문가 |
| 신뢰 출처 | 어떤 근거가 반복 인용되는가 | 공식 사이트, 언론, 리뷰, 지도 프로필, 정책 문서, 상품 feed |
| 전환 행동 | 답변 뒤 사용자가 무엇을 하는가 | 상담 요청, 데모 신청, 예약, 가입, 구매, 다운로드 |
| 실패 리스크 | 틀린 답변이 어떤 문제를 만드는가 | 과장된 평판, 오래된 출처, 잘못된 가격, 규제 표현, 예약 정보 오류 |

## 07장, 11장, 12장과 다른 점

[07장 산업별 GEO 전략](https://wikidocs.net/346335)은 업종별 차이를 이해하는 장입니다. [11장 커머스 GEO](https://wikidocs.net/346596)와 [12장 병원/오프라인 매장 SEO/GEO](https://wikidocs.net/346606)는 특정 산업을 깊게 다루는 장입니다.

90장은 이론을 다시 설명하지 않습니다. 대신 `이런 상황이면 어떤 질문을 만들고, 어떤 콘텐츠를 고치고, 어떤 지표를 봐야 하는가`를 사례형으로 보여줍니다. 앞 장들이 개념과 기준이라면, 90장은 적용 예시입니다.

## 사례를 자기 브랜드에 적용하는 순서

1. 가장 가까운 업종 사례를 고릅니다.
2. 사례 안의 SEO 키워드를 우리 업종 키워드로 바꿉니다.
3. SEO 키워드를 AI 질문셋으로 확장합니다.
4. 현재 답변에서 브랜드 언급률, 답변 근거, 화면 인용을 확인합니다.
5. 콘텐츠/출처/기술/리뷰/상품 데이터 중 어느 과제가 먼저인지 나눕니다.
6. 2주 또는 4주 실행 단위로 다시 측정합니다.

## 케이스를 읽을 때 기록할 표

각 사례를 읽을 때는 아래 표를 자기 브랜드 기준으로 채우면 좋습니다. 이 표가 있어야 케이스북이 단순 읽을거리에서 실행 계획으로 바뀝니다.

| 항목 | 기록할 내용 | 예시 |
|---|---|---|
| 대표 질문셋 | 브랜드/비브랜드/비교/검증/전환 질문 | `GEO 대행사 추천`, `OO 브랜드 신뢰도`, `A와 B 비교` |
| 현재 답변 상태 | mention/source/citation/경쟁사/오해 | 브랜드는 언급되지만 화면 인용은 경쟁사 블로그 |
| 핵심 자산 | owned/earned/third-party/technical 자산 | 뉴스룸, 보도자료, 리뷰, Product schema, 지도 프로필 |
| 첫 번째 병목 | 콘텐츠/출처/기술/데이터/리뷰 중 우선 문제 | FAQ는 있으나 schema와 본문이 다름 |
| 30일 액션 | 고칠 페이지와 담당, 완료 기준 | 팩트시트 작성, 캠페인 URL canonical 정리, 상품 feed 동기화 |
| 재측정 기준 | 같은 질문으로 언제 다시 볼지 | D+14, D+30, 월간 리포트 |
