## 산업별 GEO 익명 케이스북

GEO는 개념만으로 이해하기 어렵습니다. 실제 현장에서는 업종마다 질문 구조, 인용되는 출처, 운영 리포트, 의사결정자가 모두 다릅니다. 그래서 이 페이지는 HaloX가 스터디와 미팅에서 반복적으로 확인한 패턴을 `산업별 익명 사례`로 정리합니다.

실제 고객명, 계정, 비공개 URL, 계약 조건은 넣지 않습니다. 대신 강의와 이북에서 바로 쓸 수 있도록 문제 상황, 봐야 할 지표, 필요한 콘텐츠 구조, 연결되는 HaloX 분석 프레임을 남깁니다.

## 사례 1. PR 에이전시형: GEO를 제안 상품으로 만들어야 하는 경우

PR 에이전시는 고객에게 새로운 마케팅 키워드를 설명해야 합니다. 하지만 GEO를 “AI 검색에 잘 나오게 해드립니다” 정도로 말하면 기존 SEO 제안과 다르게 보이지 않습니다.

이 유형의 핵심 문제는 상품화 언어입니다. 고객은 복잡한 모델 작동 원리보다 다음 세 가지를 먼저 알고 싶어 합니다.

| 고객 질문 | 제안서에서 답해야 할 것 |
|---|---|
| 지금 우리 브랜드는 AI 답변에 나오나 | baseline mention/source/citation 진단 |
| 어떤 채널이 AI 답변 근거로 쓰이나 | 뉴스룸/언론/블로그/리뷰/커뮤니티 source map |
| 다음 달에 무엇을 바꾸면 되나 | 콘텐츠 리라이트/출처 보강/측정 리포트 |

이 사례는 05장과 09장에서 특히 중요합니다. Source/Citation/Entity를 설명할 때 “출처가 많다”가 아니라 “고객 제안서에서 어떤 출처를 개선 과제로 보여줄 것인가”로 바꿔야 합니다.

HaloX 기능으로 설명할 때는 `GEO 대행사 검증 체크리스트`, `source/citation 분리 추적`, `1페이지 진단 리포트` 흐름이 자연스럽습니다.

## 사례 2. 캠페인 URL 인용 추적형: 한 번 언급된 것과 반복 인용은 다르다

인플루언서 캠페인이나 PR 캠페인을 운영하면 특정 URL이 AI 답변에 들어가는지가 궁금해집니다. 하지만 한 번 인용됐다는 사실만으로 성과라고 말하기는 어렵습니다.

GEO 리포트에서 봐야 할 것은 단발성 노출이 아니라 반복성입니다.

| 측정 항목 | 봐야 하는 이유 |
|---|---|
| 프롬프트별 출처 포함 여부 | 어떤 질문에서 URL이 답변 근거가 되는지 확인 |
| 반복 측정 시 인용 안정성 | 우연한 1회 노출과 안정적 인용을 구분 |
| 검색 TOP10과 AI 답변 교차율 | 검색에는 있는데 AI 답변에는 빠지는 이유 파악 |
| 브랜드/경쟁사 동시 언급 | 캠페인이 카테고리 내 비교 맥락에 들어갔는지 확인 |

이 사례는 02장의 측정 프레임을 강화하는 데 적합합니다. 특히 네이버 AI 브리핑, Google AI Overview, ChatGPT Search, Perplexity를 한 화면에서 비교하는 설명으로 확장할 수 있습니다.

HaloX 기능으로는 `AI 브리핑 인용 안정성 리포트`와 `네이버/구글 AI 교차율 카드`가 연결됩니다.

## 사례 3. 엔터프라이즈 뉴스룸형: 뉴스룸은 콘텐츠 창고가 아니라 엔티티 허브다

대기업이나 지주사 커뮤니케이션 조직은 이미 뉴스룸, 유튜브, 보도자료, SNS, 캠페인 페이지를 많이 가지고 있습니다. 문제는 콘텐츠가 많아도 AI가 브랜드를 일관되게 이해하지 못할 수 있다는 점입니다.

이 유형은 “글을 더 쓰자”보다 “엔티티 허브를 다시 설계하자”에 가깝습니다.

| 점검 축 | 질문 |
|---|---|
| 회사/브랜드 소개 | AI가 회사의 핵심 사업과 카테고리를 일관되게 설명하는가 |
| 주요 인물/현안 | 민감한 질문에서 어떤 출처와 문장을 참고하는가 |
| 뉴스룸 구조 | 보도자료가 answer-first/FAQ/팩트시트 구조를 갖는가 |
| 온드미디어 범위 | 뉴스룸 외에 유튜브/블로그/채널 콘텐츠가 답변 근거로 쓰이는가 |
| 벤치마크 | 글로벌 기업 뉴스룸과 비교해 어떤 구조가 부족한가 |

이 사례는 04장 콘텐츠 구조, 05장 Entity/오프사이트, 09장 운영 체계에서 함께 다뤄야 합니다.

HaloX 기능으로 설명할 때는 `전략 프롬프트 세트`, `뉴스룸 점수화`, `경쟁사/벤치마크 비교 리포트`가 자연스럽습니다.

## 사례 4. 로컬 병원/지점형: 지역 질문과 내원 전환을 연결해야 하는 경우

로컬 병원, 지점형 서비스, 오프라인 전문 서비스는 일반적인 브랜드 GEO와 다릅니다. 사용자는 “브랜드명”보다 “지역 + 증상/시술/문제”로 질문합니다.

예를 들어 이런 질문이 중요합니다.

| 질문 유형 | 예시 |
|---|---|
| 지역 추천형 | 강남 피부과 추천 기준은 무엇인가 |
| 시술 비교형 | 보톡스와 필러는 어떤 경우에 다르게 선택하나 |
| 가격/후기형 | 시술 전 확인해야 할 비용과 후기 기준은 무엇인가 |
| 외국인 고객형 | 한국에서 피부과를 찾는 해외 고객은 무엇을 먼저 확인하나 |

이 유형에서는 지점 URL, 지역 키워드, 블로그 클러스터, 리뷰/지도/FAQ 구조가 함께 움직여야 합니다. 사이트 하나를 고치는 것보다 지점별 질문셋과 콘텐츠 묶음을 만드는 일이 먼저일 수 있습니다.

HaloX 기능으로는 `병원 지점형 GEO 운영 설계`, `지역 질문셋`, `지점/URL/콘텐츠 클러스터 점검표`가 연결됩니다.

## 사례 5. 금융/거래소형: 브랜드 검색과 비브랜드 전환 키워드를 동시에 봐야 하는 경우

금융/거래소/규제 산업은 브랜드 신뢰와 전환 의도가 함께 움직입니다. 브랜드명으로 검색했을 때 AI가 어떤 이미지를 설명하는지도 중요하지만, 비브랜드 질문에서 후보로 들어가는지도 중요합니다.

| 질문 축 | 예시 |
|---|---|
| 브랜드 신뢰 | 이 브랜드는 안전한가, 어떤 이슈가 있었나 |
| 입문/방법 | 처음 거래를 시작하려면 무엇을 확인해야 하나 |
| 비교/선택 | 어떤 거래소를 선택해야 하나 |
| 정책/리스크 | 수수료, 인증, 보안, 고객보호 기준은 무엇인가 |

이 유형의 실패 패턴은 권위가 약한 외부 도메인에 콘텐츠를 많이 쌓는 것입니다. AI가 신뢰할 만한 출처로 보지 않으면 글 수가 많아도 인용 근거가 되기 어렵습니다.

HaloX 기능으로는 `키워드/프롬프트 클러스터`, `authority 진단`, `기술 SEO/GEO 점검`, `브랜드/비브랜드 시계열 리포트`가 연결됩니다.

## 사례 6. 커머스/플랫폼형: AI 고객이 상품을 읽을 수 있어야 한다

커머스 GEO는 단순히 블로그 글을 잘 쓰는 문제가 아닙니다. AI가 상품명, 가격, 재고, 옵션, 배송, 반품, 리뷰, 추천 대상을 명확하게 읽을 수 있어야 합니다.

| 정보 구조 | 확인 질문 |
|---|---|
| 상품명/카테고리 | AI가 상품을 같은 카테고리 후보로 분류할 수 있는가 |
| 가격/재고/옵션 | 실시간성이 필요한 정보가 구조화되어 있는가 |
| 리뷰/평점 | 제3자 신뢰 신호가 연결되어 있는가 |
| 비교 기준 | 어떤 사용자가 이 상품을 선택해야 하는지 명확한가 |
| 정책 정보 | 배송/반품/교환 조건이 파싱 가능한가 |

이 사례는 07장의 커머스/AIO 전략과 08장의 글로벌/영문 확장 전략으로 이어집니다.

HaloX 기능으로는 `Agent-Readable Commerce Score`, `상품 정보 구조화 체크리스트`, `Product schema/merchant feed/llms.txt 점검`이 자연스럽습니다.
