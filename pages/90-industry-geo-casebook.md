## 산업별 GEO 케이스북

![산업별 GEO 케이스북 실행 패턴](../assets/images/page-heroes/halox-geo-90-industry-casebook-hero.png)

이 케이스북은 GEO 개념을 업종별 상황으로 옮기는 장입니다. 같은 mention/source/citation 지표라도 PR 에이전시, 캠페인, 뉴스룸, 로컬 서비스, 금융, 커머스에서는 먼저 봐야 할 질문이 다릅니다.

각 사례는 실제 고객명을 드러내지 않는 익명 패턴으로 읽으면 됩니다. 중요한 것은 어느 업종이든 “AI 답변에 이름이 나왔는가”보다 왜 나왔고, 어떤 근거가 붙었고, 다음에 무엇을 고칠지입니다.

[TOC]

## 읽는 순서

| 기준 | 읽는 법 |
|---|---|
| 서비스화 | GEO를 제안서와 월간 리포트로 설명해야 하면 90-01을 본다 |
| 캠페인 | 단기 URL이나 이벤트 페이지를 추적해야 하면 90-02를 본다 |
| 업종별 운영 | 뉴스룸/로컬/금융/커머스는 해당 사례를 먼저 읽는다 |

## 활용 흐름

1. 우리 업종과 가장 가까운 사례를 고른다
2. 사례의 핵심 질문군을 우리 브랜드 질문으로 바꾼다
3. mention/source/citation 중 약한 지표를 표시한다
4. 콘텐츠/source/기술/리포트 중 먼저 고칠 층을 정한다
5. 30일 뒤 같은 질문으로 변화를 확인한다

## 케이스북 사용 예시

AcmeGEO 팀이 병원 고객을 맡았다면 90-04와 12장을 함께 읽습니다. PR 고객이라면 90-01과 90-03을 먼저 보고, 제안서에는 “질문셋/근거/수정 액션/재측정”의 네 줄을 남깁니다.

## 케이스를 리포트로 바꾸는 법

사례는 “성공했다”는 이야기보다 재사용 가능한 판단 기준을 남겨야 합니다. 기준선에서 무엇이 약했는지, 어떤 source/citation이 반복됐는지, 어떤 실행 티켓을 만들었는지, 30일 뒤 무엇을 다시 봤는지를 남기면 다른 업종에도 적용할 수 있습니다.

| 케이스 항목 | 남길 내용 |
|---|---|
| 기준선 | mention/source/citation, 경쟁 URL, 질문군 |
| 실패 원인 | 콘텐츠 공백, 외부 source 부족, 기술 이슈, 평판 리스크 |
| 실행 티켓 | 공식 페이지 수정, FAQ/비교표, PR/source, schema/robots 점검 |
| 재측정 | 같은 질문셋으로 30일 뒤 확인할 지표 |

## 보고서에 남길 문장

```text
이 사례의 핵심은 업종명이 아니라 기준선과 실행 티켓입니다. 어떤 질문에서 빠졌는지, 어떤 출처가 대신 쓰였는지, 어떤 URL을 보강했는지, 언제 같은 조건으로 다시 볼지를 기록합니다.
```
