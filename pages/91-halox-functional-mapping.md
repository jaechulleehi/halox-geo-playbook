## 부록 B. HaloX GEO 솔루션과 브랜드 가시성 분석 맵

![HaloX 기능 매핑](../assets/images/page-heroes/halox-geo-91-halox-functional-mapping-hero.png)

이 부록은 앞장에서 다룬 GEO 개념을 HaloX의 분석 흐름과 연결해 보는 지도입니다. 공식 제품 문서를 대체하지 않고, 실무자가 “이 문제는 어떤 화면과 어떤 리포트로 확인해야 하는가”를 빠르게 찾도록 돕습니다.

HaloX의 운영 흐름은 크게 `대시보드 → 프롬프트 분석 → 인용 추적 → 사이트 진단 → 전략맵/콘텐츠 제작 → 주간 리포트`로 이해하면 됩니다.

## 개념과 기능 매핑

| WikiDocs 개념 | HaloX에서 확인할 흐름 | 남길 판단 |
|---|---|---|
| mention | 대시보드/프롬프트 분석 | 어떤 질문에서 브랜드가 등장하는가 |
| source | 프롬프트 분석/인용 추적 | 답변 근거가 어디에서 오는가 |
| citation | 인용 추적 | 화면에 표시되는 URL은 무엇인가 |
| 기술 이슈 | 사이트 진단 | URL이 발견/접근/해석/인용 가능한가 |
| 실행 티켓 | 전략맵/콘텐츠 제작 | 무엇을 만들거나 고칠 것인가 |
| 반복 운영 | 주간 리포트 | 변화와 다음 액션은 무엇인가 |

## 문제별로 보는 사용 흐름

브랜드가 AI 답변에 안 나온다면 먼저 프롬프트 분석에서 질문셋과 mention을 봅니다. 언급은 되는데 근거가 이상하면 인용 추적에서 source/citation을 확인합니다. 공식 URL이 잡히지 않거나 오래된 URL이 반복되면 사이트 진단으로 기술 상태와 페이지 구조를 봅니다.

보고서는 점수표로 끝내지 않습니다. “어떤 질문에서 어떤 URL이 근거로 쓰였고, 그래서 어떤 콘텐츠/source/기술 티켓을 실행할지”까지 남겨야 합니다.

![HaloX 기능 결정 흐름](../assets/images/body-figures/halox-geo-91-halox-functional-mapping-decision-flow-codex-only.png)

*HaloX 기능 매핑은 GEO 지표를 화면 이름에 연결하는 것이 아니라 다음 실행 티켓으로 바꾸는 흐름이다.*

## 리포트 문장 예시

```text
이번 주에는 비교형 질문 20개 중 브랜드 mention이 6개에서 9개로 늘었습니다.
다만 citation은 외부 블로그에 몰려 있어 공식 비교 페이지 보강이 필요합니다.
사이트 진단에서는 비교 페이지의 title, FAQ schema, 내부 링크 이슈가 확인되었습니다.
다음 주 액션은 공식 비교 페이지 리라이트와 외부 프로필 설명 수정입니다.
```

## 실무 점검표

| 상황 | 먼저 볼 화면 | 다음 액션 |
|---|---|---|
| 브랜드가 안 나옴 | 프롬프트 분석 | 질문셋/카테고리 자산 보강 |
| 엉뚱한 근거가 붙음 | 인용 추적 | source map/공식 URL 보강 |
| 공식 페이지가 인용 안 됨 | 사이트 진단 | title/schema/canonical/본문 구조 점검 |
| 리포트가 실행으로 안 이어짐 | 전략맵/주간 리포트 | 티켓과 담당자 지정 |

## 마무리

이 책의 목적은 GEO를 추상 개념으로 이해하는 데서 끝나지 않고, 질문셋, 근거, 기술, 콘텐츠, 리포트를 한 루프로 운영하게 만드는 것입니다. HaloX는 그 루프를 제품 화면과 리포트로 확인하는 도구로 보면 됩니다.
