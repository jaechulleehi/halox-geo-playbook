# HaloX GEO WikiDocs 전체 리뷰 요약

## 결론

현재 책은 목차/페이지 수/링크/외부 링크 상태는 안정적입니다. `TOC.md` 기준 95개 페이지가 모두 존재하고, 누락 이미지/누락 TOC 링크/깨진 외부 링크은 발견되지 않았습니다.

다만 독자 경험 관점에서는 아직 “좋은 위키독스 책”이라기보다 “워크시트와 생성 이미지가 많이 붙은 강의/운영 패키지”처럼 보이는 문제가 큽니다. 특히 모든 페이지에 대표 이미지와 본문 이미지가 동시에 들어가 있고, 부모 장 페이지 일부는 대표 이미지가 두 종류로 중복되어 첫 화면이 무겁습니다. 표 비중이 높은 페이지가 많아 본문 설명보다 표/체크리스트가 먼저 보이는 문제도 있습니다.

## 가장 큰 문제 5개

### 1. 이미지 정책이 과합니다

모든 페이지가 최소 2개의 이미지를 갖고 있습니다.

- `page-heroes/` 대표 이미지
- `body-figures/` 본문 설명 이미지

부모 장 페이지 10개는 여기에 `chapter-heroes/` 이미지까지 들어가 3개입니다.

우선 정리 대상:

| 페이지 | 이미지 수 | 문제 |
|---|---:|---|
| `00-chapter-0.md` | 3 | page hero + chapter hero + body figure |
| `02-chapter-2.md` | 3 | page hero + chapter hero + body figure |
| `03-chapter-3.md` | 3 | page hero + chapter hero + body figure |
| `04-chapter-4.md` | 3 | page hero + chapter hero + body figure |
| `05-chapter-5.md` | 3 | page hero + chapter hero + body figure |
| `06-chapter-6.md` | 3 | page hero + chapter hero + body figure |
| `07-chapter-7.md` | 3 | page hero + chapter hero + body figure |
| `08-chapter-8.md` | 3 | page hero + chapter hero + body figure |
| `09-chapter-9.md` | 3 | page hero + chapter hero + body figure |
| `10-chapter-10.md` | 3 | page hero + chapter hero + body figure |

권장 정책:

- 각 페이지 상단에는 대표 이미지 1개만 둡니다.
- 본문 설명 이미지는 정말 설명력이 있는 페이지에만 남깁니다.
- 부모 장 페이지에서는 `chapter-heroes/`와 `page-heroes/` 중 하나만 남깁니다.
- 썸네일 목적 이미지는 본문 중복 노출보다 WikiDocs/외부 공유용 메타 자산으로 관리하는 편이 낫습니다.

### 2. 독자용 본문보다 워크시트/표가 앞서는 페이지가 많습니다

특히 02~12장, 90장, 91장은 표 비중이 높습니다. 실무형 책이라는 장점은 있지만, 처음 보는 독자에게는 “왜 이 표를 채워야 하는지”보다 “표를 채워라”가 먼저 보일 수 있습니다.

우선 보강 방향:

- 표 앞에 2~4문단짜리 개념/판단 기준을 추가합니다.
- `이 표를 쓰는 순간` / `이 표로 결정하는 것`을 먼저 설명합니다.
- 표는 요약용보다 판단/실행용으로만 남깁니다.

### 3. 용어 통일성이 깨져 있습니다

현재 혼용이 보입니다.

| 혼용 | 권장 기준 |
|---|---|
| `엔터티` / `엔티티` | `엔티티`로 통일 |
| `source` / `답변 근거(source)` / `답변 근거` | 첫 등장만 `답변 근거(source)`, 이후 `답변 근거` |
| `citation` / `화면 인용(citation)` / `화면 인용` | 첫 등장만 `화면 인용(citation)`, 이후 `화면 인용` |
| `Fan-out` / `fan-out` / `Query Fan-out` | 제목/고유 개념은 `Query Fan-out`, 일반 설명은 `fan-out` 또는 `질문 확장 패턴` 중 하나로 정리 |
| `Answer-first` / `answer-first` | `Answer-first`로 통일 |
| `프롬프트` / `prompt` | 한국어 본문은 `프롬프트`, 코드/검색어 맥락만 `prompt` |

특히 `엔터티/엔티티` 혼용은 다음 파일에서 직접 확인됐습니다.

### 4. AI스럽거나 메타적인 표현이 남아 있습니다

자동 스캔 기준으로 다음 표현들이 반복 감지됐습니다.

- `중요합니다`
- `도움이 됩니다`
- `다양한`
- `활용할 수 있습니다`
- `알아보겠습니다`
- `작성 예시`
- `제출 템플릿`
- `워크숍`
- `강의`

`작성 예시`는 실습형 페이지에서는 필요할 수 있지만, 모든 페이지에 반복되면 책이 “독자에게 설명하는 글”보다 “과제 양식”처럼 보입니다.

권장 방향:

- `작성 예시`는 유지하되 페이지별 실제 산출물 이름으로 바꿉니다. 예: `AcmeGEO 예시`, `리포트 작성 예`, `제안서 비교 예`.
- `제출 템플릿`은 공개 독자용에서는 `복사해서 쓰는 양식`, `내 브랜드에 적용해 보기`처럼 바꿉니다.
- `강의/워크숍`은 10장처럼 의도된 장에서만 남기고, 일반 본문에서는 `따라 해보는 순서`, `실행 흐름`으로 바꿉니다.
- `중요합니다/도움이 됩니다`는 이유와 판단 기준이 드러나는 문장으로 바꿉니다.

### 5. 부모 장 페이지의 역할이 흐려진 곳이 있습니다

부모 장 페이지는 독자에게 “이 장에서 무엇을 얻고, 어느 순서로 읽을지”를 알려줘야 합니다. 현재 일부 부모 페이지는 이미지/표/링크가 많아 장의 관문이라기보다 축약본처럼 보입니다.

부모 페이지 개선 기준:

- 첫 문단: 이 장의 독자 문제 1개
- 두 번째 문단: 이 장이 남기는 산출물 1개
- 읽는 순서: child page를 3~6개 묶음으로 안내
- 표는 1개 이하로 제한
- 중복 대표 이미지는 제거

## 우선순위별 수정안

### P0. 바로 고칠 것

1. 부모 장 페이지 10개의 중복 hero 이미지 제거
2. 헤딩/이미지 간격 이슈 60건 수정

4. `Answer-first`, `Fan-out`, `source/citation` 표기 기준 문서화 후 적용
5. `알아보겠습니다`, `중요합니다`, `도움이 됩니다` 같은 AI투 표현 1차 치환

### P1. 독자 경험 개선

1. 표 비중 높은 장부터 표 앞 설명 보강
2. `작성 예시/제출 템플릿/완료 기준` 반복 구조를 페이지별 산출물 중심 표현으로 변경
3. 부모 장 페이지를 “축약본”이 아니라 “읽는 길잡이”로 재정리
4. 02장/03장/05장/06장/09장/10장은 독자가 바로 막히는 핵심 장이므로 우선 점검

### P2. 콘텐츠 완성도 개선

1. 01장 SEO 기본기와 06장 기술 파트는 깊이는 있으나 용어와 표가 많아 초보자용 설명 흐름을 더 부드럽게 조정
2. 07/12/90장 산업 사례는 사례 간 표현 패턴이 반복되므로 각 업종의 실패/판단/실행 차이를 더 드러냄
3. 91장 부록은 제품 연결 맵 성격이 강하므로 홍보처럼 보이지 않게 “판단 기준 → 기능 연결” 순서로 재정리
